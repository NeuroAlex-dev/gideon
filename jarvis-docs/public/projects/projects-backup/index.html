Бэкапы -что сохранять и как не потерять

4 мин · обновлено 2026-05-10
Сервер -как рабочий стол. Всё важное с рабочего стола должно быть скопировано в сейф. Сейф = GitHub.

Всё, что Ваш Агент знает, помнит и создал -живёт на VPS. Если с сервером что-то случится (авария, случайное удаление, переустановка) -данные могут пропасть. Бэкапы -единственная страховка. Эта статья -что и как сохранять.

Главное сообщение

Что не в GitHub -того не существует

Память, знания и проекты на VPS -это Ваш единственный экземпляр. Подключите GitHub и регулярно делайте git push. Восстановление из GitHub -одна команда. Восстановление без бэкапа -невозможно.

Что бэкапить

ЧтоГде на сервереВажностьПочему
MEMORY.md~/workspace/КритичноВся долгосрочная память о Вас
SOUL.md, USER.md~/workspace/КритичноЛичность и профиль Агента
GOALS.md~/workspace/ВажноТекущие цели
knowledge/~/workspace/ВажноНакопленная база знаний
memory/~/workspace/ЖелательноДневники (последние 30 дней)
.claude/skills/~/workspace/ВажноВсе скиллы
Проекты~/projects/КритичноКод, сайты, боты
.env~/.agent/КритичноAPI-ключи (запишите отдельно!)

Что НЕ нужно бэкапить

  • Daily notes старше 30 дней -автоочистка удалит их сама
  • /tmp/ -временные файлы
  • node_modules/ -восстанавливается через npm install
  • .agent/bot/ -код бота обновляется командой /new в @jarchitect_bot

Простой бэкап через git

Если GitHub уже подключён, скажите Агенту:

«Сделай бэкап workspace в GitHub»

Агент выполнит:

Команды бэкапаcd ~/workspace git add -A git commit -m "backup: 2026-05-10" git push

Для каждого проекта -аналогично:

«Запуши проект my-website в GitHub»

Автоматический бэкап

Агент коммитит изменения в процессе работы -формат [agent] feat: .... Это уже локальная история. Чтобы она уехала в GitHub:

«Каждый вечер предлагай мне запушить все изменения за день в GitHub»

Агент запомнит и будет напоминать. Или настройте расписание через /settings:

«Присылай напоминание о бэкапе каждый день в 20:00»

Особый случай: ключи API

Файл .env (~/.agent/.env) содержит Ваши API-ключи. Его нельзя пушить в GitHub (это небезопасно). Но его нужно помнить.

Рекомендация: запишите список Ваших ключей (Deepgram, GitHub, Vercel и т.д.) в надёжное место -менеджер паролей, заметки телефона, бумажка в ящике стола. Если сервер пропадёт -ключи придётся вводить заново через /settings.

Восстановление из бэкапа

Если пришлось переустановить Агента (новый VPS, авария):

  1. Установите Агента через @jarchitect_bot как обычно
  2. Пройдите первичную настройку (bootstrap)
  3. Скажите Агенту:

«Склонируй мой workspace из GitHub: https://github.com/username/workspace»

Агент скачает все файлы -память, знания, скиллы -и вернётся в рабочее состояние. Восстановление проектов -аналогично, по одному:

«Склонируй проект my-website из GitHub»

Мой Вам совет

Минимум, который спасёт Вас от потери данных:

  1. Подключите GitHub (статья «GitHub -бэкап и совместная работа»)
  2. Раз в неделю говорите Агенту: «Запуши workspace в GitHub»
  3. Запишите API-ключи в безопасное место вне сервера

Три действия, пять минут в неделю. Если сервер упадёт -восстановитесь за 10 минут вместо «всё пропало».
