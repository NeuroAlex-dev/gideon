## Сессия 1 (jarvis-docs — воссоздание базы знаний)

Тема: воссоздать сайт-базу знаний по Джарвису из выгрузки 40 HTML-файлов и задеплоить на Vercel.

### Сделано
- Разобрал структуру выгрузки `jarvis-website/jarvis/` (40 HTML, всё в плоской папке, разделы зашиты в навигации/breadcrumb каждой страницы)
- Извлёк карту разделов из главной: 11 разделов (start, architecture, bot, memory, skills, projects, mcp, vscode, monitoring, security, troubleshooting). В выгрузке 39 статей покрывают эти разделы (часть статей из исходного сайта не выгружена — счётчики на главной показывают реальное количество в выгрузке)
- Создал проект `jarvis-docs/` рядом с `parser/` и `sales-manager/`
- Написал `scripts/build.js` + `scripts/templates.js` (чистый Node.js, без npm-зависимостей):
  - парсер исходных HTML через regex
  - резолв slug по siblings-навигации (URL себя ищет в siblings других статей)
  - cleanup артефактов оригинального markdown→HTML (`<p>&lt;/div&gt;</p>` → `</div>`)
  - переписывание ссылок `/docs/...` → `/...` (корень нового домена)
  - clean-логика: перед билдом удаляет всё сгенерированное, оставляет `style.css` и `script.js`
- Шаблоны: home (4 featured-карточки start + 10 категорий), section-index (грид статей раздела), article (breadcrumb + h1 + page-meta + metaphor + body + siblings)
- Написал `public/style.css` (~600 строк) под все 52 уникальных класса из выгрузки: BEM-структура + кириллические legacy-классы (callout-важно, callout-осторожно, callout-перевод, callout-совет), светофор (access-card --green/--yellow/--red), copy-блоки (copy-block/codeblock/prompt-block), tree-diagram, breadcrumb, siblings. Дизайн с поддержкой dark mode через `prefers-color-scheme`
- `public/script.js` — copy-кнопки из оригинала (clipboard API)
- Структура файлов: `<section>/<slug>/index.html` (а не `<slug>.html`) — работает на любом сервере без `cleanUrls`-конфига
- Локальный smoke на Python http.server :4400, все URL 200
- Установил Vercel CLI глобально (`npm i -g vercel`). Нюанс: `node` не в системном PATH (лежит в `C:\Users\Administrator\nodejs\`), пришлось ставить через `powershell -Command '$env:PATH = "C:\Users\Administrator\nodejs;" + $env:PATH; npm i -g vercel'` — иначе esbuild post-install падает
- Залогинился в Vercel как `neuroalex-dev` через device-code OAuth
- Деплой: `vercel deploy --prod --yes`. Имя `jarvis-docs` оказалось занято кем-то в Vercel — автоматически выдан `jarvis-docs-two.vercel.app`
- Production проверен curl: главная и статьи отдают правильный контент

### Решения
- **Стек: чистый HTML+CSS** (не Astro/Next) — у нас уже почти готовый HTML на руках, нужен только парсинг + шаблоны + CSS
- **Корень сайта `/`, не `/docs/`** — для отдельного домена чище. Build.js переписывает все `/docs/...` ссылки в `/...`
- **Файлы как `<slug>/index.html`** вместо `<slug>.html` — работает без `cleanUrls`, проще локально тестировать
- **CSS с нуля** под существующие BEM-классы (исходный `style.css` в выгрузке отсутствовал) — современная документация, читаемая типографика, dark mode
- **Footer удалён** — Александр попросил убрать `Поддержка: @aibasis_bot`
- **Имя `jarvis-docs-two`** — оставляем как есть (не пересоздаём проект)
