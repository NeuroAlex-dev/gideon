## Сессия 2 (vibecoder-platform — зеркало lesson.viberost.ru)

Тема: воссоздать собственный курс Александра «Профессия: Вайбкодер» (lesson.viberost.ru) и задеплоить на vercel без логина по паролю.

### Сделано
- Разведка: сайт на Next.js + Vercel. Логин через Server Action POST на /login, в лоб curl'ом не взять (500). Часть страниц публичные (/solutions, /glossary, /guides), часть за cookie (/cheatsheets, /courses)
- Краулер на Playwright (`scripts/crawl.js`): headless chromium, BFS-обход с сохранением рендеренного HTML и всех ассетов через context.request (sharing cookies)
- Первый проход — публичные 103 страницы (без логина)
- Headed-логин (`scripts/login-interactive.js`): открываем chromium на сервере в видимом режиме, Александр через RDP вводит пароль сам, скрипт сохраняет storageState в `scripts/storage-state.json`
- Cookie `vibecoding_session` от Iron Session **привязан к IP** — попытка использовать cookie с домашнего ноута Александра на VPS-сервере падала (307→/login). Только headed-логин с того же IP, где будет идти краулер, сработал
- Полный перекауринг с авторизованной сессией → 125 страниц в 5 разделах
- Восстановил 3 недостающих JS-чанка (43911..., 981315..., a6dad97...) из бэкапа raw-source-prev + один скачал с оригинала
- Refetch glossary (`scripts/refetch-glossary.js`): отдельная страница падала на networkidle timeout, перекачал с waitUntil:'load' + явное ожидание hydration
- Build-pipeline (`scripts/build.js`): копирует ассеты + страницы как `<slug>/index.html`, переписывает `<a href="/login">`, выпиливает `<form>` с lucide-log-out (кнопка «Выйти»)
- Anti-FOUC: синхронный inline-скрипт в начале `<head>` читает localStorage.theme и ставит data-theme на html до paint. MutationObserver пинит атрибут против сброса при React hydration (некоторые страницы вроде /solutions и /glossary сбрасывали тему через useEffect)
- CSS-override для header.header-dark под data-theme=light (server отдаёт `class="header-dark"`, React меняет на light при mount — между этим мигание)
- CSS+JS защита от «Выйти»: `:has(svg.lucide-log-out){display:none!important}` + MutationObserver удаляющий вставленные кнопки
- `.vercelignore` + outputDirectory=public + пустой build/install command — Vercel выкладывает только public/, не тащит node_modules/raw-source/scripts (9.9 МБ деплой)
- Деплой: https://vibecoder-platform.vercel.app — имя не было занято, чистый домен без -two

### Решения
- **Стек: чистый статический HTML+CSS+скачанные Next.js чанки**, без Next.js-сервера. Сервер не нужен — сайт превращается в статику с переходами через обычные `<a href>` (а не SPA-router) и работающим переключателем темы из оригинальных JS-чанков
- **Краулер скачивает RENDERED HTML после hydration**, не raw markup — это позволяет получить контент даже если он рендерится клиентом. Платится за это hydration FOUC, который потом приходится тушить inline-скриптами
- **storage-state.json в .gitignore** — содержит cookie сессии Александра, секрет
- **raw-source/ В РЕПЕ** — для воспроизводимости. Если позже надо пересобрать сайт, не надо снова логиниться/краулить. Размер 16 МБ — приемлемо
- **Имя `vibecoder-platform`** не было занято в Vercel — повезло, получили чистый домен (в отличие от jarvis-docs где было `-two`)

### Подводные камни для будущих рипов Next.js сайтов
- Server HTML после login часто отдаётся **с дефолтной темой** даже если у пользователя в localStorage другая → FOUC при каждой навигации. Лечится синхронным inline-скриптом в начале head + MutationObserver на data-theme
- Логин-форма Next.js Server Action **не берётся curl'ом** даже с правильными Next-Action хедерами. Только headed Playwright или ручной login через DevTools и копирование cookie
- Iron Session cookie **часто bind к IP** — экспортированный из домашнего Chrome cookie не сработает с VPS-сервера. Логиниться надо с того же IP, что и краулер
- RSC payload (`<script>self.__next_f.push([...])`) дублирует структуру компонентов. Если выпиливать UI-элементы через regex по HTML — React при hydration их **восстанавливает из payload**. Нужна или модификация RSC payload, или CSS-скрытие, или MutationObserver-удаление
- Не каждая страница ссылается на все нужные JS-чанки. Например глоссарий запрашивал чанк `981315b0101c85f7.js` которого не было в скачанных ассетах главной → переключатель темы на нём не работал. После рип-а всегда проверять что для всех ссылок в HTML есть файлы на диске
- Vercel CLI на Windows требует node в PATH (сложности с глобальной установкой) → trick через PowerShell: `$env:PATH = "C:\Users\Administrator\nodejs;" + $env:PATH; npm i -g vercel`
