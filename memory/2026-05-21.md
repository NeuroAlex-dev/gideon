### Решения
- **Универсальный конструктор кампаний** — не один продукт, а параметризованный движок (оффер, ЦА, ИКР, тон — поля кампании)
- **С личного аккаунта Александра**, 10-20 первых сообщений в день, тёплый аутрич, не спам — стратегия минимального риска бана
- **4 режима автономии** на выбор для каждой кампании: full_auto, qualify+handoff, draft_approval, hybrid
- **Claude Code CLI по подписке** как AI-ядро (как в боте) — без API-ключа
- **Брифинг кампании через @flash_gideon_bot** (discovery-interview стиль), правки и метрики — в веб-UI парсера (gideon-bay.vercel.app)
- **Архитектура A** — отдельный сервис `sales-manager/`, своя SQLite БД, два процесса PM2 (server + worker), HTTP API на `:3001`
- **MVP scope:** режимы full_auto и draft (qualify/hybrid — фаза 2), без вкладки переписок (только список + детали кампании)

## Сессия 2 (реализация MVP)

### Сделано
- Phase 0 (Foundation): `sales-manager/` scaffold, package.json, npm install (147 deps), SQLite-схема в `lib/db.js`, 4 группы query-helpers (campaigns, leads, conversations+messages, drafts+events+stats) — 16 тестов
- Phase 1 (Core libs): `lib/safety.js` (рабочие часы, лимиты, рандом, TG error classifier, unsubscribe-детектор), `lib/ai.js` (обёртка над claude CLI с DI-runner), `lib/prompts.js` (system + first-message), `lib/dialog-engine.js` (full_auto + draft_approval) — все тесты
- Phase 2 (I/O): `lib/telegram.js` (GramJS adapter, шарит сессию парсера), `lib/outbound.js` (scheduler tick + processApprovedDrafts), `lib/inbound.js` (batch-окно 60s + dialog-engine integration)
- Phase 3 (Server + Worker): `server.js` (Express :3001, HMAC auth + POST /api/auth, все endpoints campaigns/leads/lifecycle/stats/drafts/conversations/events), `worker.js` (tick + NewMessage listener + alerts на flood_wait), `lib/bot-notifier.js` (TG Bot API → Александру), `ecosystem.config.cjs` (два процесса PM2)
- Phase 4 (Bot): `bot/sales-menu.js` (команда `/sales`, мастер брифинга 7 полей, выбор режима inline, manual загрузка лидов, callback-кнопки `sm:approve/reject/edit:N`), подключение в `bot/index.js`
- Phase 5 (Web UI): `parser/public/sales.html` + `sales.js` (список кампаний, login по паролю), `parser/public/sales-campaign.html` + `sales-campaign.js` (3 таба: бриф с inline-правкой, лиды, метрики)
- Phase 6 (Deploy): обновлён `vercel.json` (rewrite `/api/sales/*` → sales.138-16-178-94.nip.io)

### Что осталось руками сделать Александру (вне рамок кода)
1. Создать `sales-manager/.env` — секреты: TG_API_ID/HASH (те же что у парсера), TG_BOT_TOKEN, OWNER_CHAT_ID, SM_PASSWORD, SM_SECRET, CLAUDE_CLI_PATH
2. Добавить в `C:\Users\Administrator\projects\voice-input\Caddyfile` блок `sales.138-16-178-94.nip.io { reverse_proxy localhost:3001 }`, выполнить `caddy reload`
3. PM2 start: `cd sales-manager && pm2 start ecosystem.config.cjs && pm2 save`. Обновить start-bot.bat для автозапуска
4. Перезапустить бот (`pm2 restart agent-bot`) — он подхватит новый `sales-menu.js`
5. Smoke-тест: создать кампанию `/sales`, добавить 1 лид (второй TG аккаунт), запустить, проверить полный цикл
