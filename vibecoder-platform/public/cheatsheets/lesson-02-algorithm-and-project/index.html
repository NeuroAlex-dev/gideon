Пошаговый алгоритм + Личный проект

На этом уроке вы получили систему из 5 шагов — ваш «GPS-навигатор» для создания любых цифровых продуктов. И сформулировали идею личного проекта, который будете развивать весь курс.

1. Почему без системы ничего не работает

Как обычно создают продукты: «Хочу сделать приложение» → сразу открывают ИИ → пишут промпт → получают что-то непонятное → переделывают 15 раз → бросают через 3 дня.

 

Три причины, почему так происходит:

  • Нет чёткой идеи — «хочу приложение» это мечта, а не идея
  • Не изучил, что уже есть — изобретает велосипед
  • Не спланировал — начинает делать всё сразу и тонет в деталях
Факт

70% IT-стартапов закрываются в первый год. Главная причина — не технологии, а отсутствие плана. Это одинаково работает что для стартапов с бюджетами, что для новичков с ИИ.

Аналогия

Без системы — купил кирпичи, начал класть стены. Забыл фундамент. Всё развалилось. С системой — сначала чертёж, потом фундамент, потом стены. Дом стоит.

2. Пять шагов — ваш GPS-навигатор

Универсальный процесс создания любого цифрового продукта. Работает для лендинга, бота, приложения — для чего угодно. Всегда в этом порядке.

 

ШагНазваниеЧто делаем
1ИдеяФормулируем ЧТО создаём, ДЛЯ КОГО и ЗАЧЕМ
2ИсследованиеСмотрим существующие примеры, берём лучшее
3КритикаПросим ИИ честно оценить идею
4ПланРазбиваем на маленькие шаги
5РеализацияДаём задачи ИИ по одной
Аналогия

Это как GPS-навигатор: вы ввели адрес — он построил маршрут. Без навигатора — блуждаете наугад.

3. Шаг 1 — Идея

Формулируем ЧТО, ДЛЯ КОГО и ЗАЧЕМ. Используйте шаблон:

 

Я создаю ЧТО для КОГО, чтобы решить проблему КАКУЮ

 

Примеры:

  • «Я создаю калькулятор стоимости ремонта для владельцев квартир, чтобы они за 2 минуты узнали примерную стоимость»
  • «Я создаю лендинг для психологов, чтобы у них появился сайт для записи клиентов»
  • «Я создаю Telegram-бота для владельцев кофеен, чтобы они могли отслеживать расходы»
  • «Я создаю квиз для репетиторов, чтобы клиенты понимали, какой формат обучения подходит»
Совет

Идея без формулировки — это мечта. Мечта без плана — это фантазия. Запишите свою идею по шаблону прямо сейчас.

 

Готовый промпт — попросить ИИ помочь с идеей:

Я хочу выбрать идею для своего цифрового продукта.
Моя сфера: {впишите свою нишу или интерес}
Предложи 5 идей продуктов (лендинг / квиз / калькулятор / бот / приложение),
которые я могу создать без кода с помощью ИИ.
Для каждой идеи напиши:
- Что создаём (тип продукта)
- Для кого (аудитория)
- Какую проблему решает
Выбери из 5 идей одну лучшую и объясни почему.

4. Шаг 2 — Исследование

Смотрим, что уже существует. Изучаем 5–10 примеров: что работает, что нет. Выписываем лучшие идеи — это ваш фундамент.

Аналогия

Шеф-повар не придумывает борщ заново каждый раз. Он изучает 10 лучших рецептов и собирает свой. Мы делаем так же.

 

Зачем исследовать:

  • Понять, что уже работает
  • Вдохновиться удачными решениями
  • Избежать типичных ошибок
  • Собрать референсы для ИИ

 

Три способа исследовать:

 

Способ 1 — через ИИ (Claude / ChatGPT / Gemini):

Покажи мне 5-7 примеров успешных проектов для {ваша ниша}.
Для каждого опиши:
- Какие секции есть
- Что делает его эффективным
- Какие элементы можно взять для моего проекта

 

Способ 2 — через Google. Ищете «лучшие сайты [ваша ниша]» и изучаете, как они устроены.

 

Способ 3 — через галерею Lovable. Находите похожие проекты и смотрите, из каких блоков они собраны.

5. Шаг 3 — Критика

Находим слабые места ДО начала работы. Даём ИИ свою идею вместе с результатами исследования и просим раскритиковать максимально честно.

Аналогия

Критика — это не нападение. Это рентген вашей идеи. Лучше увидеть трещину ДО стройки, чем когда стены уже стоят.

 

Готовый промпт:

Я хочу создать {ваша идея из Шага 1}.
Проанализируй мою идею и ответь:
- Что может не сработать?
- Чего не хватает для успеха?
- Какие секции/элементы обязательно должны быть?
- Какие типичные ошибки делают в таких проектах?
- Как улучшить мою идею?

 

Что вы получите:

  • Список что добавить
  • Список чего избегать
  • Улучшенное понимание проекта
Совет

Лучше найти проблему за 3 минуты в чате с ИИ, чем через 5 часов работы обнаружить, что ваш продукт никому не нужен.

6. Шаг 4 — План

Разбиваем проект на маленькие шаги. Описываем итоговый результат, делим на 3–5 этапов. Каждый этап — одна задача для ИИ.

Аналогия

Собирать LEGO по инструкции — легко. Высыпать все детали на пол и надеяться, что само соберётся — нет.

 

Пример плана: калькулятор ремонта

  1. Создать форму ввода (площадь, комнаты, тип ремонта)
  2. Добавить расчёт стоимости
  3. Добавить красивый дизайн
  4. Добавить кнопку «Получить смету на email»
  5. Опубликовать по ссылке

 

Каждый пункт — отдельная задача. Не пытайтесь сделать всё за один промпт.

7. Шаг 5 — Реализация

Даём ИИ задачи по одной. Одна задача — один промпт. Берём этап из плана → пишем промпт → проверяем результат → следующий этап.

Аналогия

Реализация — как конвейер на заводе. Одна деталь за другой. Не пять одновременно.

 

Подробно разберём реализацию с Урока 3. Сейчас главное — запомнить принцип: по одной задаче за раз.

8. Личный проект

Личный проект — это продукт, который вы развиваете до защиты на последней неделе курса. На каждом модуле применяете новые навыки к СВОЕМУ продукту.

 

3 критерия хорошей идеи:

#КритерийПроверка
1Вам лично интересноВы будете работать над этим 7 недель
2Кому-то это нужноЕсть аудитория, которая заплатит или будет пользоваться
3Это реально сделатьНе «построить новый Яндекс», а конкретный продукт

 

Если все три галочки стоят — идея хорошая. Если хотя бы одной нет — пересмотрите.

 

Шаблон формулировки:

Я создаю {ЧТО} для {КОГО}, чтобы решить проблему {КАКУЮ}

 

Проверьте свою идею:

  • Есть конкретный тип продукта? (не «приложение», а «квиз» или «калькулятор»)
  • Есть конкретная аудитория? (не «все люди», а «репетиторы» или «владельцы кофеен»)
  • Есть конкретная проблема? (не «хочу помочь», а «нет сайта для записи»)
Что дальше с идеей?

В Уроке 5 вы создадите полный паспорт проекта — с документацией, правилами для ИИ и планом работ. Сегодняшняя идея станет фундаментом для этого паспорта.

9. Алгоритм одной картинкой

Весь процесс, который вы будете использовать на каждом модуле курса:

 

ИДЕЯ → ИССЛЕДОВАНИЕ → КРИТИКА → ПЛАН → РЕАЛИЗАЦИЯ

  💡          🔍           🤔        📋         ⚡
Что, для    Смотрим      Находим    Разбиваем   Даём задачи
кого,       лучших,      дыры ДО    на шаги     ИИ по одной
зачем       берём        начала
            за основу    работы

Домашнее задание

Пропустите свою идею через полный цикл исследования и критики. Используйте готовый промпт ниже — скопируйте, подставьте свою идею и отправьте Claude.

 

Промпт для домашки:

Ты — опытный продуктовый менеджер.
Вот моя идея проекта: {вставьте свою формулировку идеи}

Выполни 3 задачи:

1. ИССЛЕДОВАНИЕ:
   Найди 5 похожих продуктов, которые уже существуют.
   Для каждого напиши: что хорошо, что плохо, что можно взять.

2. КРИТИКА:
   Раскритикуй мою идею честно:
   - 3 слабых места
   - Что я забыл учесть
   - Кому это точно НЕ нужно

3. УЛУЧШЕНИЕ:
   Предложи улучшенную версию моей идеи.
   Сформулируй в одном предложении:
   Я создаю [ЧТО] для [КОГО] чтобы решить проблему [КАКУЮ]

 

Результат: обновлённая формулировка идеи, прошедшая проверку.

Готово

После этого урока вы знаете 5-шаговый алгоритм создания цифровых продуктов и умеете формулировать идею так, чтобы она была конкретной, проверенной и готовой к реализации. Этот алгоритм — ваш фундамент на весь курс.
