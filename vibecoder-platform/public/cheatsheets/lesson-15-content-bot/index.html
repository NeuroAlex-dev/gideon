Шпаргалки / Урок 15: Бот-копирайтер

Создаём бота-копирайтера

Мультиагентный подход — Claude Code — Telegram Bot — OpenRouter

Пошаговый алгоритм

Повторяется для любого проекта.

  1. Создать новую папку проекта. В Claude Code → New session → Choose a different folder → Новая папка

  2. Создать команду агентов. Отправить промпт с описанием задачи → Claude создаёт agents.md с ролями, моделями, температурами

  3. Запустить архитектуру. «Запускай команду в работу, полностью спроектируй и создай сервис» → Claude строит всю структуру проекта

  4. Добавить API-ключи в .env. Telegram Bot Token + OpenRouter API Key. Если файл не виден — ⌘ ⇧ . для показа скрытых файлов на Mac. Убрать расширение .example из имени файла.

  5. Протестировать бота в Telegram. Нажать /start → проверить работу кнопок → при ошибке сделать скриншот и закинуть в Claude Code

  6. Загрузить обучающий контент. Выгрузить посты из Telegram Desktop (app с сайта telegram.org, целиком синяя иконка) → загрузить JSON или MD файл прямо в чат бота

  7. Докрутка и доработка. Тестируем → что не нравится, описываем словами → Claude Code правит → повторяем. Можно добавлять новые кнопки и функции бесконечно.

Ключевые промпты урока

Промпт 1 — Создание команды агентов

Создаём копирайтера в Telegram, который будет писать моим языком на основе
выгруженных файлов из моего Telegram-канала.
Файлы могут быть загружены в виде MD или JSON.

Он должен уметь:
- Создавать и читать контент-план
- Редактировать посты: делать более человечными, короткими, хлёсткими
- Писать в моём стиле на основе загруженных примеров

В первую очередь создай команду агентов в файле agents.md:
- Оркестратор — координирует всех
- Архитектор — планирует структуру проекта
- Декомпозитор — разбивает задачи на мелкие подзадачи
- Тестировщик — проверяет работу всех агентов

Для каждого агента укажи: роль, модель, температуру.
Также выдай структуру агентов с описанием.

Промпт 2 — Запуск разработки

Отлично, теперь запускай команду в работу.
Полностью спроектируй и создай сервис.
Скажи, куда добавить ключи для OpenRouter и для бота Telegram.

Промпт 3 — Контент-план с тремя ветками

Создай файл контент-план.md, подвязанный к кнопке «Контент-план».
Сохрани его в папку проекта.

В контент-плане:
- Проанализируй все посты, которые я загрузил как примеры стиля
- Разбей контент-план на три ветки:
  1. Экспертная — взгляд на рынок, тренды, экспертные мнения
  2. Продуктовая — про мои продукты и услуги
  3. Личная — истории, ошибки, процессы, закулисье
- Продающая ветка — в меньшей пропорции, чем остальные
- Проанализируй целевые аудитории, разбей на подсегменты
- Исходя из их болей, мечт, проблем, хотелок и страхов (БМПХС)
  создай план на ближайшие две недели

По кнопке «Контент-план» в Telegram показывай 3 ближайших темы.

Зачем столько агентов?

ИИ плохо понимает многосоставные задачи. Когда каждый агент отвечает за одну задачу — результат точнее. Декомпозитор разбивает сложное на мелкое, а Оркестратор собирает всё обратно.

OpenRouter — агрегатор моделей

Подключить один раз → использовать любые модели (Claude, GPT-4.1, Gemini, Llama и др.). Ключ берётся в openrouter.ai → Settings → API Keys. Задайте лимит $5–10, чтобы контролировать расходы.

Где взять API-ключи

Telegram — Bot Token

  1. Открыть @BotFather в Telegram
  2. /newbot → задать имя → задать username (должен заканчиваться на bot)
  3. Скопировать токен — закрыть окно можно только после копирования!
  4. Вставить в .env → TELEGRAM_BOT_TOKEN=...

OpenRouter — API Key

  1. Зайти на openrouter.ai
  2. Settings → API Keys → Create Key
  3. Задать лимит (например, $5) и срок действия
  4. Скопировать ключ сразу — потом недоступен!
  5. Вставить в .env → OPENROUTER_API_KEY=...
Файл .env скрытый

Чтобы увидеть скрытые файлы на Mac: ⌘ + ⇧ + .. Если Claude создал .env.example — переименуйте, убрав .example.

Аналогия

БМПХС — 5 слоёв аудитории, которые закрывает контент: Боли · Мечты · Проблемы · Хотелки · Страхи. Каждый пост должен бить хотя бы в один слой.

Лайфхаки урока

  • Говорите промпты голосом. Super Whisper или Handy — назначьте горячую клавишу. Нажали → сказали → отпустили. В разы быстрее печати.

  • Ошибка = скриншот в Claude. Получили ошибку в боте → скриншот → вставить в Claude Code. Он сам разберётся и починит.

  • Мелкие задачи = лучший код. ИИ плохо понимает сложные многосоставные задачи. Декомпозитор-агент разбивает их на маленькие — результат точнее.

  • Выгружайте свои посты из Telegram. Telegram Desktop с сайта telegram.org → Настройки → Экспорт данных. Получите JSON с постами канала для обучения бота.

  • Tone of Voice из голосовых. Транскрибируйте голосовые из Telegram → отдайте Claude. Он найдёт ваши настоящие паттерны речи точнее, чем вы сами.

  • Сначала agents.md — потом всё остальное. Создание команды агентов в самом начале — залог управляемого проекта. Пропустить этот шаг = хаотичный код.
