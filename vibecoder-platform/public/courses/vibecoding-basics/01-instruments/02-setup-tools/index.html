Устанавливаем инструменты

Готовы? Поехали! Сейчас установим всё, что нужно для работы. Каждый шаг — 2-5 минут. К концу урока у вас будет полный рабочий набор.

Аналогия

Аналогия: ремонт квартиры. Сначала завозят материалы, потом работают. Мы сейчас «завозим» программы. Один раз поставили — и забыли. Дальше только работаем.

Что вас ждёт в этом уроке

  • Шаг 1 — Google Chrome (2 мин)
  • Шаг 2 — Редактор кода (5 мин)
  • Шаг 3 — Node.js (5 мин)
  • Шаг 4 — Git (5 мин)
  • Проверка — всё работает?

Блок 1: Google Chrome (2 мин)

Если Chrome уже стоит — пропускайте этот шаг. Если нет:

  1. Откройте google.com/chrome
  2. Нажмите «Скачать Chrome»
  3. Запустите скачанный файл
  4. Дождитесь установки. Готово!
Почему именно Chrome?

У Chrome лучшие инструменты разработчика. Нажмите F12 в Chrome — увидите панель, где можно смотреть код любого сайта. Мы будем этим пользоваться.

Готово

Что поняли: Chrome — не просто браузер, а рабочий инструмент разработчика.

Блок 2: Редактор кода (5 мин)

Редактор кода — это место, где вы будете проводить 90% времени. Выбирайте один из двух:

Вариант А: Cursor (рекомендуем)

Аналогия

Аналогия: Cursor = машина с автопилотом. VS Code — обычная машина, вы всё делаете сами. Cursor — та же машина, но с ИИ-помощником, который подсказывает дорогу и помогает парковаться.

  1. Откройте cursor.com
  2. Нажмите «Download»
  3. Установите программу
  4. Войдите в аккаунт (можно через Google)
  5. Выберите тему оформления — рекомендуем тёмную

Вариант Б: VS Code (бесплатная альтернатива)

  1. Откройте code.visualstudio.com
  2. Нажмите «Download»
  3. Установите программу
  4. При первом запуске — установите русский язык (Extensions → Russian Language Pack)
Внимание

Не устанавливайте оба! Выберите один редактор. Cursor — если хотите встроенный ИИ. VS Code — если хотите полностью бесплатно.

Готово

Что поняли: редактор кода — ваш главный инструмент. Cursor удобнее для вайбкодинга, но VS Code тоже подойдёт.

Блок 3: Node.js (5 мин)

Node.js — это «движок», который запускает ваши проекты на компьютере.

Аналогия

Аналогия: электричество. Вы купили блендер (написали сайт). Но он не работает без розетки. Node.js — это ваша розетка. Без него проект не запустится.

  1. Откройте nodejs.org
  2. Скачайте версию LTS (зелёная кнопка слева). LTS = стабильная версия
  3. Установите — просто нажимайте «Далее» на каждом шаге

Проверка — всё ли работает:

Откройте Терминал (на Mac: Spotlight → Terminal, на Windows: нажмите Win+R, введите cmd):

node --version

Если видите что-то вроде v20.11.0 — значит, Node.js установлен. Цифры могут отличаться — это нормально.

Внимание

Если терминал пишет «command not found» — перезагрузите компьютер и попробуйте снова. Иногда Node.js не виден до перезагрузки.

Готово

Что поняли: Node.js — движок для запуска проектов. Ставится один раз, работает в фоне.

Блок 4: Git (5 мин)

Git — система контроля версий. Звучит сложно, но на практике — это ваша страховка.

Аналогия

Аналогия: Google Docs. Помните, как в Google Docs можно смотреть историю изменений и вернуться к прошлой версии? Git делает то же самое, но для кода. Если что-то сломали — откатили назад.

  1. Откройте git-scm.com
  2. Скачайте и установите (нажимайте «Далее», настройки по умолчанию подходят)

Проверка:

git --version

Если видите git version 2.43.0 (или любую другую версию) — всё работает.

Для Mac

На Mac Git часто уже установлен. Сначала проверьте команду выше. Если Git есть — устанавливать не нужно.

Готово

Что поняли: Git — страховка для кода. Любые изменения можно откатить.

Финальная проверка: всё установлено?

Откройте терминал и выполните обе команды:

node --version

git --version
Что проверяемОжидаемый результат
ChromeОткрывается браузер
Cursor или VS CodeОткрывается редактор
node --versionv18+ или v20+
git --versiongit version 2.x
Готово

Все 4 пункта работают? Поздравляю! Вы полностью готовы к вайбкодингу. 🎉

📝 Домашнее задание

Установите все 4 инструмента и сделайте финальную проверку из таблицы выше.

Образ идеального результата

После выполнения ДЗ у вас:

  • Установлен Chrome (или уже был)
  • Работает Cursor или VS Code
  • Команда node --version показывает версию
  • Команда git --version показывает версию
  • Все 4 пункта проверки пройдены ✅
Внимание

Пока не пройдёте все 4 проверки — к следующему уроку переходить нет смысла. Без инструментов мы не сможем работать!

  • Установлен Google Chrome
  • Установлен Cursor или VS Code
  • Установлен Node.js (проверен через терминал)
  • Установлен Git (проверен через терминал)
  • Все 4 проверки пройдены успешно
