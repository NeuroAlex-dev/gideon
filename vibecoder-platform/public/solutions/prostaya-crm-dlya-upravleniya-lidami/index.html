Простая CRM для управления лидами

Заявки приходят из разных каналов: сайт, Telegram, WhatsApp, сарафанка. Вы записываете в блокнот, потом теряете. Знакомо? Создадим CRM-систему, где каждый лид виден на доске. Статус, сумма, источник — всё на одном экране.

Что понадобится

  • Lovable — AI-помощник, который создаёт веб-приложения за вас
  • Список источников лидов — откуда приходят заявки (сайт, мессенджеры, звонки)
  • Стадии воронки — как заявка движется от первого контакта до сделки
  • Домен (по желанию) — если хотите свой адрес вместо бесплатного .vercel.app

Код писать не нужно. Lovable соберёт всё по вашему описанию.

Аналогия

CRM — это пульт управления вашей воронкой продаж. Как диспетчерская в аэропорту: каждый самолёт на радаре, ни один не пропадёт. Каждый клиент виден, каждая сделка под контролем. Никто не провалится сквозь щели.

Шаг 1: Подготовка

Прежде чем открыть Lovable, определите структуру воронки продаж.

Выпишите в блокнот:

  1. Стадии воронки — через какие этапы проходит заявка
  2. Источники лидов — откуда приходят клиенты
  3. Поля карточки — какая информация нужна по каждому лиду
  4. Средний цикл сделки — сколько времени от заявки до оплаты

Пример воронки для агентства:

СтадияЧто происходитЦвет
НовыйЗаявка только поступилаСиний
В работеСвязались, обсуждаем задачуЖёлтый
ПереговорыОтправили коммерческое предложениеОранжевый
ВыиграноКлиент оплатилЗелёный
ПроиграноКлиент отказалсяКрасный

Пример полей карточки лида:

ПолеПример
ИмяАлексей Петров
Контакт+7 999 123-45-67
ИсточникTelegram
Сумма сделки150 000 руб.
ЗаметкиХочет лендинг + настройку рекламы
Совет

Начните с 5 стадий. Больше — запутаетесь. Воронка должна быть простой: новый — в работе — переговоры — выиграно — проиграно. Добавить стадии легко, убрать лишние — сложнее.

Шаг 2: Создание CRM в Lovable

Откройте lovable.dev, создайте новый проект и вставьте этот промпт:

Создай простую CRM-систему для управления лидами на React.

Требования:
1. Kanban-доска с колонками (drag-and-drop):
   - "Новый" (синий)
   - "В работе" (жёлтый)
   - "Переговоры" (оранжевый)
   - "Выиграно" (зелёный)
   - "Проиграно" (красный)
2. Карточка лида на доске: имя, сумма, источник, дата создания
3. При клике на карточку — модальное окно с полной информацией:
   - Имя клиента
   - Телефон / email
   - Источник (выпадающий список: Сайт, Telegram, WhatsApp, Звонок, Рекомендация)
   - Сумма сделки
   - Заметки (текстовое поле)
   - Дата создания (автоматически)
4. Кнопка "Добавить лид" — открывает форму создания
5. Drag-and-drop перемещение карточек между колонками
6. Счётчик лидов и сумма в каждой колонке
7. Адаптивный дизайн (мобильные + десктоп)
8. Данные сохраняются в localStorage
9. Используй Tailwind CSS, shadcn/ui и dnd-kit для drag-and-drop

Замените стадии и источники на свои. Нажмите Enter.

Lovable соберёт CRM за 2–3 минуты.

Важно

Не закрывайте вкладку во время генерации. Если закроете — процесс прервётся и придётся начинать заново.

Ожидаемый результат: Kanban-доска с колонками, карточками и формой добавления лида. Перетаскивание работает.

Шаг 3: Поиск и фильтры

Лидов будет много. Нужен быстрый поиск и удобные фильтры.

Добавь поиск и фильтрацию лидов:

1. Поле поиска вверху страницы — ищет по имени, телефону, заметкам
2. Фильтр по источнику: выпадающий список (Все, Сайт, Telegram, WhatsApp, Звонок, Рекомендация)
3. Фильтр по сумме: "Все", "до [СУММА_1] руб.", "[СУММА_1]–[СУММА_2] руб.", "от [СУММА_2] руб."
4. Фильтр по дате: "Сегодня", "Эта неделя", "Этот месяц", "Все"
5. Сортировка: по дате создания, по сумме (по возрастанию / убыванию)
6. При поиске и фильтрации — карточки на доске обновляются в реальном времени
7. Кнопка "Сбросить фильтры"
Зачем фильтры

Когда лидов 5 — фильтры не нужны. Когда их 50 — без фильтров вы потеряете половину. Настройте фильтры заранее, пока лидов мало.

Ожидаемый результат: Быстрый поиск по имени. Фильтры по источнику, сумме и дате. Доска обновляется мгновенно.

Шаг 4: Статистика и аналитика

Данные без аналитики — просто список. Добавим цифры, которые помогают принимать решения.

Добавь панель статистики над kanban-доской:

1. KPI-карточки в ряд:
   - Всего лидов (общее количество)
   - В работе (количество активных лидов)
   - Конверсия (процент "Выиграно" от общего числа)
   - Сумма выигранных сделок (общая выручка)
2. Мини-график: количество новых лидов по дням за последние 30 дней
3. Круговая диаграмма: распределение лидов по источникам
4. Средний чек выигранных сделок
5. Средний цикл сделки: дни от создания до "Выиграно"
6. Используй Recharts для графиков
Про конверсию

Конверсия — это КПД вашей воронки. Как расход топлива в машине. Если из 100 лидов выигрываете 10 — конверсия 10%. Знать эту цифру важнее, чем знать количество заявок.

Ожидаемый результат: Панель с KPI: лиды, конверсия, выручка. Графики показывают источники и динамику.

Шаг 5: Дизайн и удобство

Сделаем CRM приятной в ежедневном использовании.

Улучши дизайн и UX:

1. Тёмная тема (по умолчанию):
   - Фон: #0f172a, карточки: #1e293b с лёгкой тенью
   - Заголовки колонок с цветными полосками сверху
2. Переключатель тем: тёмная / светлая
3. Шапка: "[НАЗВАНИЕ БИЗНЕСА] — CRM", дата, количество лидов
4. Карточки лидов:
   - Иконка источника (Telegram, WhatsApp, телефон, глобус для сайта)
   - Сумма сделки крупным шрифтом
   - Дата создания мелким текстом
   - Цветная метка приоритета
5. Мобильная версия: колонки переключаются табами вместо горизонтального скролла
6. Уведомление при перемещении в "Выиграно" — конфетти-анимация
7. Быстрое добавление: кнопка "+" в каждой колонке
Совет про мобильную версию

Вы будете открывать CRM с телефона чаще, чем с компьютера. Клиент позвонил — вы тут же создаёте лид. Мобильная версия должна работать идеально.

Ожидаемый результат: Стильный интерфейс с иконками источников. Мобильная версия с табами. Конфетти при закрытии сделки.

Шаг 6: Деплой (публикация)

CRM готова. Публикуем, чтобы открывать с любого устройства.

  1. В Lovable нажмите "Deploy" (вверху справа)
  2. Выберите "Deploy to Vercel"
  3. Подключите GitHub аккаунт (если нет — создайте бесплатно)
  4. Lovable загрузит приложение автоматически
  5. Вы получите ссылку вроде: https://my-crm-abc.vercel.app
  6. По желанию — подключите свой домен в настройках Vercel
Про безопасность данных

Данные хранятся в localStorage вашего браузера. Никто кроме вас их не видит. Но если очистите кэш — данные пропадут. Экспортируйте в CSV регулярно.

Как использовать ежедневно:

  • Утром: откройте CRM, проверьте новые лиды
  • После каждого звонка: обновите статус и заметки
  • Вечером: переместите карточки по воронке
  • Раз в неделю: проверьте конверсию и средний чек

Что дальше

Базовая CRM готова. Вот как её развивать:

1. История действий — лог изменений по каждому лиду. Кто звонил, когда, о чём договорились.

2. Напоминания — задачи привязаны к лидам. «Перезвонить Алексею в среду» — уведомление в Telegram.

3. Импорт / экспорт — загрузка лидов из CSV. Выгрузка базы для рассылок.

4. Командная работа — несколько менеджеров. Каждый видит своих лидов. Руководитель видит всех.

5. Интеграция с мессенджерами — заявка из Telegram автоматически появляется на доске.

6. Воронка по услугам — отдельная доска для каждого продукта или направления.

Итог

Вы создали CRM за 2–4 часа без единой строчки кода. Все лиды в одном месте. Статус каждого клиента виден на доске. Конверсия и выручка — в цифрах. Больше никаких потерянных заявок.

✏️Упражнение: Заполните CRM первыми лидами

Добавьте 5–10 реальных лидов из последних двух недель. Распределите их по колонкам. Ответьте на три вопроса: какой источник даёт больше заявок? Сколько лидов «зависли» без движения? Какая общая сумма сделок в работе?

  1. Лучший источник — посмотрите круговую диаграмму. Самый большой сегмент — ваш главный канал. Вкладывайте в него больше.
  2. Зависшие лиды — карточки в колонке «В работе» старше 7 дней. Свяжитесь с ними сегодня или переместите в «Проиграно».
  3. Сумма в работе — число под заголовком колонки «В работе» + «Переговоры». Это ваш потенциальный доход. Если мало — нужно больше лидов на входе.

Мало реальных лидов? Создайте тестовых. Главное — привыкнуть к системе.

Готово! Ваша CRM работает. Заявки из всех каналов на одной доске, воронка наглядная, статистика считается автоматически. Управляйте продажами системно, а не по памяти.
