ИИ-агент клиентского сервиса

Клиенты пишут — а ответа нет часами. Нанять оператора стоит 60К+ в месяц. Создайте ИИ-агента, который отвечает 24/7, квалифицирует заявки и передаёт горячих менеджерам. Экономия от 80К/мес.

Что понадобится

  • Lovable — AI-помощник для создания веб-приложений. Код писать не нужно.
  • Список частых вопросов — 20–40 вопросов, которые задают ваши клиенты.
  • Критерии квалификации — по каким признакам заявка считается «горячей».
  • Домен (по желанию) — свой адрес вместо бесплатного .vercel.app.

Время на создание: 3–5 часов. Навыков программирования не нужно.

Аналогия

ИИ-агент — это неутомимый сотрудник, который работает в ночную смену и не жалуется. Отвечает на типовые вопросы, записывает контакты, а утром передаёт менеджеру только горячих клиентов с полным досье.

Шаг 1: Подготовка — база знаний и критерии квалификации

Агент хорош настолько, насколько хороша его подготовка.

Что сделать:

  1. Соберите 20–40 частых вопросов из переписок и звонков
  2. Напишите короткие ответы на каждый (2–3 предложения)
  3. Разбейте на категории: оплата, доставка, услуги, гарантии, сроки
  4. Определите критерии горячей заявки

Критерии квалификации (пример):

КритерийГорячий клиентХолодный клиент
БюджетНазывает сумму«Просто интересуюсь»
Сроки«Нужно на этой неделе»«Когда-нибудь потом»
ПотребностьКонкретная задачаОбщие вопросы
Решение«Выбираю подрядчика»«Пока думаю»
Совет

Добавьте 3–5 квалификационных вопросов. Агент задаёт их в чате. По ответам определяет — горячий клиент или нет.

Шаг 2: Создание чат-виджета в Lovable

  1. Откройте lovable.dev
  2. Создайте новый проект → "Blank project"
  3. Вставьте промпт:
Создай ИИ-агента клиентского сервиса на React.

Интерфейс чат-виджета:
- Круглая кнопка (60px) в правом нижнем углу с иконкой чата
- При клике — окно чата 400x550px с заголовком "[НАЗВАНИЕ КОМПАНИИ]"
- Поле ввода сообщения + кнопка отправки
- История сообщений: бот слева (серый фон), клиент справа (синий)
- Аватарка бота — иконка робота. Индикатор "Печатает..."
- Кнопка "Начать заново" в шапке

Приветствие:
- "Здравствуйте! Я помощник [КОМПАНИЯ]. Чем могу помочь?"
- Быстрые кнопки: "[КАТЕГОРИЯ_1]", "[КАТЕГОРИЯ_2]", "[КАТЕГОРИЯ_3]"

Логика ответов:
- База знаний — массив {category, question, answer, keywords}
- Поиск по ключевым словам в сообщении
- Нашёл совпадение → ответ из базы
- Не нашёл → "Сейчас уточню и вернусь с ответом"

База знаний (заполни 10 примеров):
- [ВОПРОС_1]: [ОТВЕТ_1] (ключевые слова: [СЛОВО_1, СЛОВО_2])
- [ВОПРОС_2]: [ОТВЕТ_2] (ключевые слова: [СЛОВО_1, СЛОВО_2])
- [ВОПРОС_3]: [ОТВЕТ_3] (ключевые слова: [СЛОВО_1, СЛОВО_2])

Дизайн: минималистичный, цвет акцента [ВАШ_ЦВЕТ].
Адаптивный, Tailwind CSS, shadcn/ui.
Важно

Заполните минимум 10 пар «вопрос-ответ» в промпте. Чем больше база — тем умнее агент с первого дня.

Ожидаемый результат: рабочий чат-виджет. Агент отвечает на типовые вопросы.

Шаг 3: Квалификация лидов — бюджет, сроки, потребность

Главная ценность агента — он отсеивает «просто любопытных» от реальных клиентов.

Добавь систему квалификации лидов:

1. После 2-3 ответов из базы знаний агент спрашивает:
   - "Подскажите, какую задачу хотите решить?" (потребность)
   - "На какой бюджет рассчитываете?" (бюджет)
   - "Когда планируете начать?" (сроки)

2. Оценка лида по ответам (автоматическая):
   - Горячий (красный): назвал бюджет + сроки до 2 недель
   - Тёплый (жёлтый): есть потребность, но нет сроков
   - Холодный (синий): общие вопросы, нет конкретики

3. Горячий лид → "Отлично! Оставьте контакт — менеджер
   свяжется в течение [ВРЕМЯ]". Форма: имя, телефон, email.

4. Тёплый → продолжает консультировать из базы знаний.

5. Холодный → отвечает на вопросы + предлагает подписаться
   на рассылку или скачать полезный материал.

6. Все ответы клиента сохраняются в профиле лида.
Аналогия с приёмным покоем

Квалификация — как сортировка в приёмном покое. Тяжёлый случай — сразу к врачу. Лёгкий — подождёт в очереди. Агент определяет «тяжесть» заявки и направляет к нужному специалисту.

Ожидаемый результат: агент задаёт квалификационные вопросы. Горячие заявки собирают контакт. Холодные получают консультацию.

Шаг 4: Лог диалогов и админ-панель

Менеджеры должны видеть все диалоги и управлять базой знаний.

Добавь админ-панель (/admin, защищённую паролем [ПАРОЛЬ]):

1. "Дашборд" — сводка за сегодня:
   - Всего диалогов, горячих лидов, среднее время ответа
   - График: диалоги за 7 дней (столбчатая диаграмма)

2. "Диалоги" — полная история переписок:
   - Фильтры: дата, статус (горячий/тёплый/холодный)
   - Каждый диалог: сообщения + оценка лида + контакт
   - Цветные метки: красный, жёлтый, синий

3. "Лиды" — таблица горячих и тёплых заявок:
   - Имя, контакт, потребность, бюджет, сроки, дата
   - Экспорт в CSV. Сортировка по температуре.

4. "База знаний" — CRUD для вопросов-ответов:
   - Таблица: категория, вопрос, ответ, ключевые слова
   - Кнопки: добавить, редактировать, удалить
   - Поиск по базе

5. "Необработанные" — вопросы, на которые агент не нашёл
   ответа. Кнопка "Создать ответ" → добавить в базу.

Данные в localStorage.
Совет

Раздел «Необработанные» — золотая жила. Он показывает, о чём спрашивают клиенты, а вы не покрываете. Добавляйте ответы каждую неделю.

Про хранение данных

LocalStorage хранит данные в браузере — для MVP этого достаточно. Для рабочего проекта подключите Supabase через Lovable.

Ожидаемый результат: админка с дашбордом, историей диалогов, лидами и базой знаний.

Шаг 5: Дизайн и брендирование

Агент должен выглядеть как часть вашего бренда.

Улучши дизайн:

1. Виджет: плавная анимация появления, скруглённые углы,
   тень. Цвет: [ВАШ_ЦВЕТ]. На мобильном — на весь экран.
   Быстрые кнопки категорий — горизонтальный скролл.

2. Админ-панель: тёмный сайдбар с иконками, карточки
   с тенями, зебра-полоски в таблицах, цветные бейджи
   для температуры лидов.

3. Общее: шрифт Inter, плавные переходы между страницами.

Ожидаемый результат: профессиональный виджет и удобная админка.

Шаг 6: Деплой

Публикуем агента в интернете.

  1. В Lovable нажмите "Deploy" вверху справа
  2. Выберите "Deploy to Vercel"
  3. Подключите GitHub аккаунт (если нет — создайте бесплатно)
  4. Получите ссылку: https://ваш-агент.vercel.app

Хотите свой домен? Купите на reg.ru и подключите в настройках Vercel.

Встраивание на сайт

Попросите Lovable сгенерировать код встраивания (iframe или скрипт). Одна строка кода — и виджет появится на каждой странице вашего сайта.

Ожидаемый результат: агент опубликован и готов к встраиванию на ваш сайт.

Что дальше

Базовый агент работает. Вот как его развить.

1. Подключите OpenAI API. Агент будет понимать вопросы «своими словами», а не только по ключевым словам.

2. Уведомления в Telegram. Горячий лид — мгновенное сообщение менеджеру.

3. Режим нерабочего времени. Ночью агент собирает контакты: «Менеджер ответит в 9:00».

4. Интеграция с CRM. Лиды автоматически попадают в базу клиентов с историей переписки.

Результат

ИИ-агент отвечает клиентам 24/7 за секунды. Квалифицирует заявки: горячие — менеджеру, остальные — консультирует сам. Вы экономите 80К+ в месяц на операторе. Клиенты довольны скоростью. Менеджеры работают только с горячими лидами.

✏️Упражнение: Составьте сценарий квалификации для своего бизнеса

Напишите 3 квалификационных вопроса и критерии горячего/тёплого/холодного лида для вашей ниши.

Пример для веб-студии:

Вопросы агента:

  1. «Какой сайт вам нужен?» → лендинг / интернет-магазин / корпоративный
  2. «На какой бюджет рассчитываете?» → до 50К / 50–150К / от 150К
  3. «Когда нужен готовый сайт?» → срочно / месяц / без сроков

Горячий: Конкретный тип + бюджет от 50К + сроки до месяца. Действие: «Оставьте контакт — менеджер свяжется за 15 минут».

Тёплый: Знает тип сайта, но без бюджета или сроков. Действие: Продолжает консультировать + предлагает кейсы.

Холодный: «Сколько стоит сайт?» без конкретики. Действие: Отправляет прайс-лист + приглашает на бесплатную консультацию.
