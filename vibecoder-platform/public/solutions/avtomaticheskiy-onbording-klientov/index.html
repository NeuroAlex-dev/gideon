Автоматический онбординг клиентов

Новый клиент оплатил услугу — и тишина. Не знает, что делать дальше. Менеджер объясняет одно и то же каждому. Создадим систему, которая встречает клиента автоматически. Приветствие, инструкции, доступы — без участия человека.

Что понадобится

  • Lovable — AI-конструктор сайтов и приложений
  • Список шагов клиента — что он должен сделать после оплаты
  • Тексты приветствий — письмо или сообщение для нового клиента
  • Домен (по желанию) — свой адрес вместо бесплатного .vercel.app

Код писать не нужно. Lovable создаст всё по описанию.

Аналогия

Онбординг — это стюардесса в самолёте. Вы сели в кресло — а вам уже объясняют, где выходы, когда еда и что делать при турбулентности. Без стюардессы пассажиры паникуют. Без онбординга клиенты теряются.

Шаг 1: Подготовка — маршрут нового клиента

Опишите путь клиента после оплаты. Что он получает и когда.

ЭтапДействиеКогда
1Приветственное письмо с контактамиСразу
2Доступ к рабочему пространствуЧерез 1 час
3Бриф на заполнениеДень 1
4Первый звонок-знакомствоДень 2
5Стратегия на согласованиеДень 5
Совет

Не перегружайте первый день. 2–3 действия максимум. Остальное — на второй и третий день.

Шаг 2: Создание проекта в Lovable

Откройте Lovable, создайте новый проект и вставьте промпт:

Создай систему автоматического онбординга клиентов на React.

Требования:
1. Главная панель (дашборд):
   - KPI-карточки: всего клиентов, на онбординге сейчас,
     завершили онбординг, среднее время прохождения
   - Список новых клиентов с прогресс-баром (% выполнения)
   - Уведомления: кто застрял на шаге больше 2 дней

2. Конструктор онбординг-маршрута:
   - Вертикальная цепочка шагов, соединённых стрелками
   - Каждый шаг: номер, название, описание, тип действия
   - Типы: "Отправить письмо", "Выдать доступ",
     "Назначить задание", "Запланировать звонок"
   - Кнопка "Добавить шаг", drag & drop для перестановки

3. Карточка клиента:
   - Имя, email, телефон, дата старта, тариф
   - Прогресс по шагам (чеклист с галочками)
   - Кнопка "Отметить выполнено" для каждого шага

4. Данные в localStorage, адаптивный дизайн
5. Tailwind CSS, shadcn/ui
Важно

Замените типы шагов на свои. У каждого бизнеса онбординг свой. Промпт — шаблон, подстройте под себя.

Ожидаемый результат: дашборд с клиентами, конструктор маршрута и карточки клиентов с прогрессом.

Шаг 3: Автоматические уведомления и письма

Клиент не должен гадать, что делать. Система сама напомнит.

Добавь систему уведомлений и шаблонов писем:

1. Шаблоны писем для каждого шага:
   - "Приветствие" — имя, контакты менеджера, ссылки
   - "Доступы" — логин, пароль, ссылка на платформу
   - "Первое задание" — описание + дедлайн
   - "Напоминание" — если шаг не выполнен за N дней
   - Редактор: тема, текст, переменные {{имя}}, {{тариф}}

2. Триггеры отправки:
   - При переходе на новый шаг — автоматическое письмо
   - При задержке больше 2 дней — напоминание
   - При завершении онбординга — финальное письмо

3. Лог уведомлений:
   - Кому отправлено, когда, какой шаблон, статус
   - Фильтры: по клиенту, по типу письма, по дате
Про переменные

Переменные и подставляются автоматически. «Привет, Анна!» работает лучше, чем «Уважаемый клиент».

Ожидаемый результат: шаблоны писем для каждого этапа, триггеры и лог отправок.

Шаг 4: Клиентский портал с чеклистом

Клиент должен видеть свой прогресс. Добавим портал.

Добавь клиентский портал с чеклистом:

1. Страница клиента (публичная, по ссылке):
   - Приветствие с именем клиента
   - Чеклист шагов с описанием каждого
   - Прогресс-бар: "3 из 7 шагов выполнено"
   - Текущий шаг подсвечен и развёрнут

2. Для каждого шага:
   - Описание (что нужно сделать)
   - Ссылка или файл (инструкция, доступ)
   - Дедлайн и кнопка "Готово"

3. Синхронизация:
   - Клиент отмечает шаг — менеджер видит в панели
   - Шаг не выполнен за 3 дня — уведомление
Аналогия с навигатором

Клиентский портал — как навигатор. Клиент видит маршрут, текущую точку и сколько осталось. Без навигатора он звонит вам: «А куда ехать?»

Ожидаемый результат: публичная страница с чеклистом, прогрессом и кнопками выполнения.

Шаг 5: Аналитика онбординга

Нужно видеть, где клиенты застревают.

Добавь аналитику онбординга:

1. Дашборд аналитики:
   - Среднее время прохождения онбординга (дни)
   - Конверсия: начали → завершили (%)
   - Шаг с наибольшим отвалом (где застревают)

2. Воронка по шагам:
   - Горизонтальные полоски: сколько клиентов на каждом шаге
   - Красный цвет — шаг с потерей больше 20%

3. Таблица клиентов:
   - Имя, дата старта, текущий шаг, дней на онбординге
   - Статус: "В процессе", "Завершён", "Застрял"
Совет

Если на одном шаге теряется больше 20% клиентов — переписывайте инструкцию. Проблема не в клиентах. Проблема в объяснении.

Ожидаемый результат: аналитика с воронкой и выявлением проблемных шагов.

Шаг 6: Деплой (публикация)

Система готова. Публикуем в интернете.

Улучши дизайн системы онбординга:

1. Цвета: фон #f8fafc, выполненные #10b981 (зелёный),
   текущий #3b82f6 (синий), просроченные #ef4444 (красный)
2. Боковая панель: "Дашборд", "Клиенты", "Маршрут",
   "Уведомления", "Аналитика"
3. Клиентский портал: минималистичный, крупный шрифт,
   иконки для каждого типа шага
4. Мобильная версия: бургер-меню, карточки на всю ширину

Вариант 1 — бесплатный деплой:

  1. В Lovable нажмите "Deploy" (вверху справа)
  2. Выберите "Deploy to Vercel"
  3. Получите ссылку: https://onboarding-abc.vercel.app

Вариант 2 — свой домен:

  1. Купите домен (например, start.mybusiness.ru)
  2. В настройках Vercel добавьте свой домен
  3. Настройте DNS по инструкции Vercel
Про безопасность

Клиентский портал доступен по ссылке без пароля. Не размещайте там конфиденциальные данные. Для защиты добавьте авторизацию по паролю — попросите Lovable.

Что дальше

Базовый онбординг работает. Вот как его развить:

1. Интеграция с CRM. Новый клиент в CRM — автоматически стартует онбординг.

2. Telegram-уведомления. Менеджер получает сообщение, когда клиент завершил шаг или застрял.

3. Видео-инструкции. 30 секунд экранной записи заменяют страницу текста.

4. Оценка онбординга. После завершения — опрос: «Понятны ли инструкции?»

5. Шаблоны маршрутов. Разные тарифы — разные маршруты.

Итог

Вы создали систему онбординга за 2–4 часа. Каждый новый клиент получает приветствие, инструкции и чеклист автоматически. Менеджер видит прогресс. Клиент не теряется. Время на ввод нового клиента сократилось с дней до минут.

✏️Упражнение: Создайте маршрут онбординга для своего бизнеса

Опишите 5–7 шагов для нового клиента в первую неделю.

Пример для онлайн-школы английского:

Шаг 1 (День 0): «Добро пожаловать!» — приветствие, контакты куратора, чат. Шаг 2 (День 0): «Доступ к платформе» — логин, пароль, инструкция. Шаг 3 (День 1): «Тест уровня» — результат определяет группу. Шаг 4 (День 2): «Первое занятие» — расписание, выбор времени. Шаг 5 (День 5): «Обратная связь» — куратор звонит. Шаг 6 (День 7): «Онбординг завершён» — письмо с прогрессом.

Готово! Автоматический онбординг создан. Каждый клиент получает чёткий маршрут. Менеджер видит прогресс. Клиент доволен — а вы не тратите время на объяснения.
