Трекер привычек и продуктивности

Привычки формируют жизнь. Но без трекера они забываются к среде. Создадим приложение, где пользователь отмечает привычки, видит прогресс и получает мотивацию. Готовый продукт для продажи в wellness-нише. Подписка или разовая покупка — вы решаете.

Что понадобится

  • Lovable — AI-конструктор сайтов и приложений
  • Список привычек — 5–10 привычек для демо-версии
  • Механика мотивации — как поощрять пользователя за прогресс
  • Домен (по желанию) — если хотите свой адрес вместо бесплатного .vercel.app

Код писать не нужно. Lovable создаст всё по описанию.

Аналогия

Трекер привычек — это фитнес-браслет для поведения. Браслет считает шаги — трекер считает выполненные привычки. Вы не забываете пить воду, когда браслет напоминает. Вы не бросаете медитацию, когда трекер показывает серию в 14 дней.

Шаг 1: Подготовка

Прежде чем открыть Lovable, продумайте структуру продукта.

Выпишите в блокнот:

  1. Целевая аудитория — кто будет пользоваться трекером
  2. Привычки по умолчанию — какие привычки предложить новому пользователю
  3. Механика мотивации — что удерживает пользователя (серии, баллы, бейджи)
  4. Модель монетизации — бесплатный тариф + платный, или разовая покупка

Пример набора привычек для wellness-продукта:

Совет

Начните с 6–8 привычек по умолчанию. Дайте пользователю добавлять свои. Слишком много привычек сразу — человек бросит через два дня. Лучше 5 стабильных, чем 20 забытых.

Шаг 2: Создание проекта в Lovable

Откройте Lovable, создайте новый проект и вставьте этот промпт:

Создай трекер привычек и продуктивности на React.

Требования:
1. Главный экран — сегодняшний день:
   - Список привычек с чекбоксами (отметить выполненное)
   - Прогресс-бар дня (выполнено 4 из 7 — 57%)
   - Мотивационная фраза, меняется каждый день
   - Текущая серия (streak): "5 дней подряд без пропусков"
2. Привычки по умолчанию: вода, медитация, зарядка, чтение, сон, прогулка
3. Кнопка "Добавить привычку":
   - Название, иконка (эмоджи), категория, частота (ежедневно / X раз в неделю)
   - Цвет привычки (выбор из палитры)
4. Календарь-тепловая карта (как на GitHub):
   - Каждый день — квадрат, цвет зависит от % выполнения
   - При клике на день — детали: что сделано, что пропущено
5. Данные сохраняются в localStorage
6. Адаптивный дизайн (мобильные + десктоп)
7. Используй Tailwind CSS и shadcn/ui

Замените привычки на свои. Нажмите Enter.

Lovable создаст трекер за 2–3 минуты.

Важно

Не закрывайте вкладку во время генерации. Если закроете — процесс прервётся и придётся начинать заново.

Ожидаемый результат: Экран с привычками на сегодня, чекбоксы, прогресс-бар и тепловая карта календаря.

Шаг 3: Серии и мотивация

Пользователь возвращается, когда не хочет прервать серию. Добавим геймификацию.

Добавь систему мотивации и серий:

1. Серии (streaks):
   - Текущая серия: "12 дней подряд" с огненной иконкой
   - Лучшая серия: "рекорд — 30 дней"
   - При пропуске — серия сбрасывается, показать сообщение поддержки
2. Бейджи достижений:
   - "Первый день" — отметил хотя бы одну привычку
   - "Неделя" — 7 дней подряд без пропусков
   - "Месяц" — 30 дней подряд
   - "Идеальный день" — все привычки за день выполнены
   - "Сотня" — 100 дней в трекере
3. Уровень пользователя:
   - Очки за каждую отмеченную привычку (+10)
   - Бонус за полный день (+50)
   - Бонус за серию (+5 за каждый день серии)
   - Уровни: Новичок, Практик, Мастер, Легенда
4. Экран достижений: сетка бейджей, полученные — цветные, остальные — серые
Про серии

Серия — это цепочка из скрепок. Каждый день вы добавляете одну. Через неделю — цепочка. Через месяц — гирлянда. Разорвать жалко. Именно это удерживает пользователей в приложении.

Ожидаемый результат: Счётчик серий с иконкой огня. Бейджи достижений. Уровень пользователя с очками.

Шаг 4: Статистика и аналитика

Пользователь хочет видеть прогресс. Покажем цифры и графики.

Добавь экран статистики:

1. KPI-карточки:
   - Всего дней в трекере
   - Процент выполнения за последнюю неделю
   - Текущая серия
   - Лучшая привычка (самый высокий % выполнения)
2. График выполнения по неделям (линейный, Recharts):
   - Ось Y: процент выполнения (0–100%)
   - Ось X: последние 8 недель
3. Статистика по каждой привычке:
   - Название, % выполнения за месяц, текущая серия
   - Мини-календарь: 30 дней, зелёный — сделано, серый — пропущено
4. Самый продуктивный день недели (столбчатая диаграмма)
5. Фильтр по периоду: неделя, месяц, всё время
Зачем статистика в wellness-продукте

Числа мотивируют. «87% выполнения за неделю» звучит лучше, чем «я вроде стараюсь». Графики показывают рост. Пользователь видит прогресс и не бросает.

Ожидаемый результат: Экран с графиками, статистикой по каждой привычке и мини-календарями.

Шаг 5: Напоминания и настройки

Привычка без напоминания — просто список. Добавим уведомления.

Добавь напоминания и настройки пользователя:

1. Напоминания для каждой привычки:
   - Время напоминания (выбор часов и минут)
   - Дни напоминания (чекбоксы: Пн, Вт, Ср, Чт, Пт, Сб, Вс)
   - Включить/выключить переключателем
   - Использовать браузерные уведомления (Notification API)
2. Настройки профиля:
   - Имя пользователя
   - Цель: "Хочу выработать привычку за 66 дней"
   - Начало дня: во сколько обнуляется прогресс (по умолчанию 00:00)
3. Экспорт данных в JSON (для бэкапа)
4. Импорт данных из JSON
5. Кнопка "Сбросить прогресс" с подтверждением
6. Переключатель тем: тёмная / светлая
Совет про напоминания

Браузерные уведомления работают только если пользователь разрешил их. Добавьте красивое окно с объяснением: «Разрешите уведомления, чтобы не забывать о привычках».

Ожидаемый результат: Настройки напоминаний для каждой привычки. Профиль. Экспорт и импорт данных.

Шаг 6: Деплой (публикация)

Приложение готово. Опубликуем его в интернете.

Улучши дизайн приложения:

1. Цвета:
   - Основной фон: мягкий градиент (#f0fdf4 → #ecfdf5)
   - Акцент: #10b981 (зелёный wellness)
   - Серии: оранжевый #f59e0b
   - Карточки: белые с лёгкой тенью
2. Нижняя навигация (мобильная):
   - "Сегодня", "Календарь", "Статистика", "Достижения", "Настройки"
   - Иконки + подписи
3. Анимации: плавная отметка привычки (чекбокс с зелёной волной)
4. Мобильная версия как приоритет (PWA-стиль)

Вариант 1 — бесплатный деплой:

  1. В Lovable нажмите "Deploy" (вверху справа)
  2. Выберите "Deploy to Vercel"
  3. Получите ссылку: https://habit-tracker-abc.vercel.app

Вариант 2 — свой домен:

  1. Купите домен (например, myhabits.ru)
  2. В настройках Vercel добавьте свой домен
  3. Настройте DNS по инструкции Vercel
Про данные пользователей

Сейчас данные в localStorage — каждый пользователь хранит свои данные в своём браузере. Для полноценного продукта подключите базу данных (Supabase). Тогда пользователи смогут войти с любого устройства.

Что дальше

Базовый трекер готов. Вот как развивать продукт:

1. Авторизация — регистрация через email. Данные в облаке, доступ с любого устройства.

2. Социальные функции — группы поддержки. Друзья видят прогресс друг друга.

3. Монетизация — бесплатно: 3 привычки. Платный тариф: безлимит + статистика + бейджи.

4. Push-уведомления — мобильные пуши через PWA или Telegram-бот.

5. Шаблоны привычек — готовые наборы: «Утренняя рутина», «ЗОЖ», «Продуктивность».

6. Интеграция с календарём — привычки как события в Google Calendar.

Итог

Вы создали трекер привычек за 2–4 часа. Чекбоксы, серии, бейджи, статистика — всё работает. Это готовый продукт для wellness-ниши. Добавьте авторизацию, подключите оплату — и запускайте продажи.

✏️Упражнение: Протестируйте трекер на себе

Добавьте 5 своих привычек. Отмечайте их 7 дней. После недели ответьте: какая привычка далась легче всего? Какую пропускали чаще? Что нужно изменить в интерфейсе?

  1. Самая лёгкая — привычка с самым высоким процентом выполнения в статистике. Обычно это то, что уже встроено в распорядок (например, вода).
  2. Самая сложная — самый низкий процент. Подумайте: время напоминания неудобное? Привычка слишком амбициозная? Разбейте на шаги помельче.
  3. Что улучшить — если забывали открыть трекер, нужны напоминания. Если отмечали, но не чувствовали мотивации — усильте геймификацию. Добавьте звук при отметке или анимацию конфетти за идеальный день.

Ваш опыт — лучший UX-тест. Исправляйте то, что мешает лично вам.
