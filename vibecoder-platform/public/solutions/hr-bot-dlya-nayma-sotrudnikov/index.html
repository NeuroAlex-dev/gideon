HR-бот для найма сотрудников

Каждая вакансия — это 50+ откликов. Вы читаете резюме, задаёте одни и те же вопросы, тратите часы на первичный отсев. Создадим HR-бота, который проводит первичное собеседование и фильтрует кандидатов. Вам приходят только подходящие.

Что понадобится

  • Lovable — AI-конструктор сайтов и приложений. Код писать не нужно.
  • Список вопросов для отсева — 5–10 вопросов, которые вы задаёте каждому кандидату.
  • Критерии оценки — что считается хорошим ответом на каждый вопрос.
  • Домен (по желанию) — свой адрес вместо бесплатного .vercel.app.

Время на создание: 2–4 часа. Навыков программирования не нужно.

Аналогия

HR-бот — это охранник на входе в клуб. Он вежливо задаёт вопросы и пропускает только тех, кто подходит по дресс-коду. Вы не тратите время на случайных прохожих. Внутрь попадают только те, с кем стоит общаться.

Шаг 1: Подготовка — вопросы и критерии оценки

Бот хорош настолько, насколько точны ваши вопросы.

Что сделать:

  1. Выпишите 5–10 вопросов для первичного отсева
  2. Для каждого вопроса определите хороший и плохой ответ
  3. Назначьте вес каждому вопросу (важность от 1 до 5)
  4. Определите проходной балл

Пример для вакансии «менеджер по продажам»:

ВопросХороший ответПлохой ответВес
Опыт в продажах?1+ год, конкретные цифры«Без опыта, но готов учиться»5
Работали с CRM?Называет систему«Нет, не работал»3
Почему хотите к нам?Знает о компании«Ищу любую работу»4
Зарплатные ожидания?В рамках вилкиСильно выше вилки3
Когда готовы начать?В течение 2 недель«Через пару месяцев»2

Проходной балл: 60% от максимума. Если набрал меньше — вежливый отказ.

Совет

Первый вопрос — самый важный. Он должен отсеивать 50% неподходящих. Например: «Сколько лет опыта в продажах?» Если нужен опыт, а его нет — дальше не спрашиваем.

Шаг 2: Создание проекта в Lovable

  1. Откройте Lovable
  2. Создайте новый проект — "Blank project"
  3. Вставьте промпт:
Создай веб-приложение "HR-бот для найма" на React.

Страница кандидата (/apply):
1. Заголовок: "[КОМПАНИЯ] — Вакансия: [ДОЛЖНОСТЬ]"
2. Краткое описание вакансии (2-3 предложения)
3. Условия: зарплата, график, формат (офис/удалёнка)
4. Чат-интерфейс с ботом:
   - Бот приветствует: "Здравствуйте! Я помощник [КОМПАНИЯ]. Давайте познакомимся. Ответьте на несколько вопросов."
   - Бот задаёт вопросы по одному. Ждёт ответа. Потом следующий.
   - После каждого ответа — подтверждение: "Спасибо, записал. Следующий вопрос:"
   - В конце: "Спасибо за ответы! Мы рассмотрим вашу анкету в течение [СРОК]."
5. Вопросы бота (заполни 5 примеров):
   - "Как вас зовут?" (обязательно)
   - "Ваш email для связи?" (обязательно)
   - "[ВОПРОС_1]"
   - "[ВОПРОС_2]"
   - "[ВОПРОС_3]"
6. Прогресс-бар: "Вопрос 3 из 7"
7. Ответы сохраняются в localStorage
8. Минималистичный дизайн, Tailwind CSS, shadcn/ui
Важно

Замените [КОМПАНИЯ], [ДОЛЖНОСТЬ] и вопросы на свои. Первые два вопроса (имя и email) — обязательные. Остальные — ваши критерии отсева.

Ожидаемый результат: страница с описанием вакансии и чат-ботом. Бот задаёт вопросы по одному и сохраняет ответы.

Шаг 3: Скоринг кандидатов — автоматическая оценка

Бот не просто записывает ответы. Он оценивает каждого кандидата по баллам.

Добавь систему скоринга кандидатов:

1. Каждый вопрос имеет "правильные" ключевые слова.
   Пример:
   - Вопрос: "Опыт в продажах?"
   - Ключевые слова для высокого балла: "лет", "года", "руководил", "выполнял план", "CRM"
   - Ключевые слова для низкого: "нет опыта", "стажёр", "не работал"
   - Вес вопроса: 5

2. Подсчёт баллов:
   - Нашёл ключевое слово высокого балла → +вес вопроса
   - Нашёл низкого → 0 баллов
   - Ничего не нашёл → половина веса (нейтрально)
   - Итоговый балл: сумма / максимум × 100%

3. Категории кандидатов:
   - Зелёный (80%+): "Отличный кандидат — рекомендуем собеседование"
   - Жёлтый (50-79%): "Средний — на усмотрение менеджера"
   - Красный (ниже 50%): "Не подходит — автоматический отказ"

4. После завершения анкеты:
   - Зелёный → "Отлично! Мы свяжемся с вами для собеседования."
   - Жёлтый → "Спасибо! Мы рассмотрим вашу анкету."
   - Красный → "Спасибо за интерес! К сожалению, ваш опыт
     не совсем подходит. Мы сохраним контакт на будущее."
Аналогия со светофором

Скоринг — это светофор для найма. Зелёный — проезжайте, берём на собеседование. Жёлтый — подождите, посмотрим внимательнее. Красный — не по нашей дороге, но без обид.

Ожидаемый результат: бот автоматически оценивает ответы. Кандидат получает вежливый ответ по результату.

Шаг 4: Админ-панель для просмотра кандидатов

HR-менеджер должен видеть всех кандидатов и управлять процессом.

Добавь админ-панель (/admin, защищённую паролем [ПАРОЛЬ]):

1. "Дашборд":
   - Всего откликов, зелёных, жёлтых, красных (за сегодня / всего)
   - График: отклики по дням за 30 дней
   - Средний балл кандидатов

2. "Кандидаты" — таблица всех откликов:
   - Имя, email, дата, балл (с цветным индикатором)
   - Статус: Новый / На собеседовании / Принят / Отказ
   - Фильтры: по категории (зелёный/жёлтый/красный), дате, статусу
   - Сортировка: по баллу, по дате
   - При клике — полная анкета: все вопросы и ответы

3. "Детали кандидата" (модальное окно):
   - Все ответы кандидата с баллами за каждый вопрос
   - Общий балл и категория
   - Кнопки: "Пригласить на собеседование", "Отказать", "Отложить"
   - Поле для заметок HR-менеджера
   - Кнопка "Скопировать email" для быстрой связи

4. "Вакансии" — управление несколькими вакансиями:
   - Список вакансий: название, дата создания, количество откликов
   - Добавление новой вакансии с вопросами и критериями
   - Отдельная ссылка /apply/[id] для каждой вакансии

5. Экспорт в CSV: имя, email, балл, категория, дата

Данные в localStorage.
Про несколько вакансий

Одна админка — много вакансий. Каждая со своими вопросами и критериями. Каждая со своей ссылкой. Разместите ссылку в описании вакансии на hh.ru или в Telegram-канале.

Ожидаемый результат: админка с дашбордом, списком кандидатов и управлением вакансиями.

Шаг 5: Дизайн и удобство

Сделаем бот профессиональным, а админку — удобной.

Улучши дизайн:

1. Страница кандидата (/apply):
   - Логотип компании вверху (заглушка — цветной круг с инициалами)
   - Чат: сообщения бота слева (серый фон), кандидата справа (синий)
   - Аватарка бота — иконка робота
   - Индикатор "Печатает..." перед каждым вопросом (задержка 1 сек)
   - Плавная анимация появления сообщений
   - Прогресс-бар сверху: зелёный, заполняется по мере ответов
   - Мобильная версия — на весь экран

2. Админ-панель:
   - Тёмная тема: фон #0f172a, карточки #1e293b
   - Боковое меню с иконками: Дашборд, Кандидаты, Вакансии
   - Цветные бейджи баллов: зелёный, жёлтый, красный
   - Карточки KPI с иконками и тенями
   - Таблицы с зебра-полосками
   - Шрифт Inter, плавные переходы
Совет про первое впечатление

Кандидат оценивает компанию по форме отклика. Красивый бот с прогресс-баром говорит: «Здесь всё по-взрослому». Гугл-форма говорит: «Тут экономят на всём».

Ожидаемый результат: профессиональный чат-бот для кандидатов. Удобная тёмная админка для HR.

Шаг 6: Деплой (публикация)

HR-бот готов. Публикуем и рассылаем ссылку кандидатам.

Бесплатный вариант:

  1. В Lovable нажмите "Deploy" вверху справа
  2. Выберите "Deploy to Vercel"
  3. Подключите GitHub аккаунт (если нет — создайте бесплатно)
  4. Получите ссылку: https://ваш-hr-бот.vercel.app/apply

Со своим доменом:

  1. Купите домен на reg.ru (~200 руб/год за .ru)
  2. В настройках Vercel добавьте свой домен
  3. Пропишите DNS-записи по инструкции Vercel
Где разместить ссылку

Добавьте ссылку на бота в описание вакансии. На hh.ru, в Telegram-каналах, в соцсетях. Вместо «откликнуться» — «пройти короткое интервью». Конверсия откликов вырастет.

Ожидаемый результат: бот опубликован. Кандидаты переходят по ссылке и проходят интервью.

Что дальше

Базовый HR-бот работает. Вот как его развивать:

1. Подключите OpenAI API. Бот будет понимать ответы «своими словами», а не только по ключевым словам. Оценка станет точнее.

2. Уведомления в Telegram. Зелёный кандидат — мгновенное сообщение HR-менеджеру с резюме и баллом.

3. Видео-вопросы. Кандидат записывает ответ на камеру. HR смотрит 30-секундное видео вместо текста.

4. Тестовое задание. После вопросов — мини-задача по профессии. Менеджер продаж пишет скрипт звонка. Дизайнер делает мудборд.

5. Шаблоны писем. Автоматическое приглашение на собеседование или вежливый отказ. Один клик — письмо ушло.

6. Аналитика найма. Воронка: отклики — прошли бота — собеседование — оффер — вышел на работу. Где теряете кандидатов?

Результат

HR-бот проводит первичное собеседование 24/7. Кандидаты отвечают в удобное время. Бот оценивает и фильтрует автоматически. Вы видите только зелёных и жёлтых. Экономия — 5–10 часов в неделю на каждую вакансию.

✏️Упражнение: Составьте скоринговую карту для своей вакансии

Выберите одну вакансию. Напишите 5 вопросов для бота. Для каждого — ключевые слова хорошего и плохого ответа. Назначьте вес от 1 до 5. Определите проходной балл.

Пример для вакансии «SMM-менеджер»:

Вопросы бота:

  1. «Какие соцсети вы вели? Покажите примеры» (вес: 5)

    • Высокий: «Instagram», «Telegram», «ВКонтакте», «портфолио», «ссылка»
    • Низкий: «нет опыта», «личный блог», «не веду»

  2. «Сколько подписчиков было у лучшего проекта?» (вес: 4)

    • Высокий: «тысяч», «к», «рост», «увеличил», «вырос»
    • Низкий: «не помню», «немного», «пару сотен»

  3. «Какие инструменты используете для работы?» (вес: 3)

    • Высокий: «Canva», «Figma», «планировщик», «аналитика», «таргет»
    • Низкий: «телефон», «ничего», «не знаю»

  4. «Как вы измеряете результат SMM?» (вес: 4)

    • Высокий: «охват», «вовлечённость», «конверсия», «ROI», «лиды»
    • Низкий: «лайки», «не измеряю», «подписчики»

  5. «Зарплатные ожидания и когда готовы начать?» (вес: 2)

    • Высокий: «в рамках вилки», «сразу», «неделя», «две недели»
    • Низкий: «через два месяца», «намного выше»

Максимум: 5+4+3+4+2 = 18 баллов Проходной (60%): 11 баллов

Зелёный: 15–18 (80%+). Жёлтый: 11–14 (50–79%). Красный: ниже 11.

Готово! HR-бот для найма работает. Кандидаты проходят интервью в удобное время. Бот оценивает и фильтрует автоматически. Вы тратите время только на лучших кандидатов.
