Парсер для поиска клиентов

Вы ищете клиентов вручную. Листаете чаты, группы, комментарии. Копируете контакты в таблицу. На это уходит 2–3 часа в день. Создадим парсер, который находит потенциальных клиентов по ключевым словам. До 50 лидов в день — без ручного поиска.

Что понадобится

  • Lovable — AI-конструктор сайтов и приложений. Код писать не нужно.
  • Список ключевых слов — по каким фразам люди ищут ваши услуги.
  • Источники данных — ссылки на группы, чаты, форумы, где сидят ваши клиенты.
  • Домен (по желанию) — свой адрес вместо бесплатного .vercel.app.

Время на создание: 2–4 часа. Навыков программирования не нужно.

Аналогия

Парсер — это золотоискатель с металлоискателем. Вы раньше промывали песок руками. Теперь прибор сканирует огромную площадь и пищит, когда находит золото. Вы только собираете самородки.

Шаг 1: Подготовка — ключевые слова и источники

Парсер ищет по словам. Чем точнее слова — тем качественнее лиды.

Что сделать:

  1. Выпишите 15–20 фраз, которые используют ваши клиенты
  2. Разделите на категории: прямой запрос, проблема, обсуждение
  3. Составьте список источников: группы, форумы, каталоги
  4. Определите критерии «хорошего лида»

Пример ключевых слов для веб-студии:

КатегорияКлючевые слова
Прямой запрос«нужен сайт», «ищу разработчика», «сделать лендинг»
Проблема«сайт не работает», «мало заявок с сайта», «нужна переделка»
Обсуждение«кто делал сайт», «посоветуйте студию», «сколько стоит сайт»

Пример источников:

ИсточникТипПримерный охват
Бизнес-чаты в TelegramГрупповой чат1 000–50 000 участников
Профильные форумыФорум10 000+ сообщений
Комментарии в соцсетяхПубличный пост100–5 000 комментариев
Совет

Добавьте «стоп-слова» — фразы, которые отсеивают нецелевые сообщения. Например: «бесплатно», «сам сделаю», «не нужно». Это сэкономит время на фильтрации.

Шаг 2: Создание проекта в Lovable

  1. Откройте Lovable
  2. Создайте новый проект — "Blank project"
  3. Вставьте промпт:
Создай веб-приложение "Парсер лидов" на React.

Главная страница — дашборд:
1. Заголовок "[НАЗВАНИЕ] — Парсер лидов"
2. Блок KPI-карточек в ряд:
   - Найдено сегодня (число)
   - Всего лидов (число)
   - Горячих (число)
   - Конверсия в ответ (процент)
3. Таблица найденных лидов:
   - Имя / никнейм
   - Текст сообщения (обрезанный до 100 символов)
   - Источник (группа, форум)
   - Ключевое слово, по которому найден
   - Дата и время
   - Статус: Новый / Написали / Ответил / Нецелевой
   - Кнопки: "Отметить горячим", "Нецелевой", "Подробнее"
4. Фильтры над таблицей: по источнику, ключевому слову, статусу, дате
5. Кнопка "Добавить лид вручную"
6. Данные в localStorage
7. Адаптивный дизайн, Tailwind CSS, shadcn/ui
Важно

Замените [НАЗВАНИЕ] на название вашего бизнеса. Не закрывайте вкладку во время генерации.

Ожидаемый результат: дашборд с таблицей лидов, KPI-карточками и фильтрами.

Шаг 3: Система ключевых слов и мониторинг

Добавим настройку ключевых слов и источников для поиска.

Добавь страницу "Настройки поиска" (/settings):

1. Блок "Ключевые слова":
   - Таблица: слово, категория (прямой/проблема/обсуждение), приоритет (высокий/средний/низкий)
   - Добавление, редактирование, удаление слов
   - Цветовая индикация приоритета

2. Блок "Источники":
   - Таблица: название, ссылка, тип (чат/форум/соцсеть), статус (активный/пауза)
   - Добавление, редактирование, удаление источников
   - Переключатель активности для каждого источника

3. Блок "Стоп-слова":
   - Список слов для исключения
   - Добавление и удаление

4. Блок "Симуляция поиска":
   - Кнопка "Запустить поиск"
   - Генерирует 5-15 тестовых лидов на основе ключевых слов
   - Показывает, как лиды появляются в таблице на дашборде

Навигация: боковое меню с иконками — Дашборд, Настройки, Шаблоны, Аналитика.
Про симуляцию

Симуляция генерирует тестовые данные. Для реального парсинга нужен API. Но сначала постройте интерфейс. Потом подключите данные.

Ожидаемый результат: страница настроек с управлением ключевыми словами, источниками и стоп-словами. Симуляция создаёт тестовых лидов.

Шаг 4: Шаблоны сообщений и быстрый ответ

Нашли лида — нужно написать ему. Шаблоны ускоряют ответ в 5 раз.

Добавь страницу "Шаблоны сообщений" (/templates):

1. Список шаблонов. Каждый шаблон:
   - Название (например: "Первое сообщение", "Ответ на запрос")
   - Текст сообщения с переменными: {имя}, {ключевое_слово}, {источник}
   - Категория шаблона: первый контакт / ответ на вопрос / follow-up
   - Кнопки: редактировать, копировать, удалить

2. Предустановленные шаблоны (3 штуки):
   - "Здравствуйте, {имя}! Увидел ваше сообщение про {ключевое_слово}. Мы как раз этим занимаемся. Могу помочь?"
   - "Привет! Заметил, что вы ищете {ключевое_слово}. У нас есть решение. Рассказать подробнее?"
   - "{имя}, добрый день! По вашему вопросу — у нас есть опыт. Можем обсудить?"

3. На дашборде: при клике "Подробнее" у лида — модальное окно:
   - Полный текст сообщения лида
   - Блок "Быстрый ответ": выбор шаблона из выпадающего списка
   - Превью сообщения с подставленными переменными
   - Кнопка "Копировать" — копирует текст в буфер обмена
   - История действий: когда нашли, когда написали, статус
Совет

Персонализация решает. Шаблон с именем и упоминанием конкретного запроса получает ответ в 3 раза чаще, чем стандартная рассылка.

Ожидаемый результат: библиотека шаблонов. Один клик — и персональное сообщение готово к отправке.

Шаг 5: Аналитика и дизайн

Добавим статистику и сделаем интерфейс приятным.

1. Страница "Аналитика" (/analytics):
   - График: найдено лидов по дням за 30 дней (линейный)
   - Круговая диаграмма: лиды по источникам
   - Столбчатая: лиды по ключевым словам (топ-10)
   - Воронка: Найдено → Написали → Ответили → Стали клиентом
   - Конверсия на каждом этапе воронки (проценты)
   - Лучший день недели для поиска
   - Используй Recharts

2. Дизайн всего приложения:
   - Тёмная тема по умолчанию
   - Фон: #0f172a, карточки: #1e293b
   - Акцентный цвет: зелёный (#22c55e) для горячих лидов
   - Боковое меню с иконками, сворачиваемое
   - Таблицы с зебра-полосками
   - Шрифт Inter, плавные анимации
   - Мобильная версия: нижняя навигация вместо бокового меню
Воронка лидов

Воронка — это сито с разным размером ячеек. Сверху сыплете 100 лидов. Написали 50-ти. Ответили 20. Стали клиентами 5. Аналитика показывает, где теряете больше всего. Там и чините.

Ожидаемый результат: страница аналитики с графиками. Тёмная тема. Удобная мобильная версия.

Шаг 6: Деплой (публикация)

Парсер готов. Публикуем для доступа с любого устройства.

Бесплатный вариант:

  1. В Lovable нажмите "Deploy" вверху справа
  2. Выберите "Deploy to Vercel"
  3. Подключите GitHub аккаунт (если нет — создайте бесплатно)
  4. Получите ссылку: https://ваш-парсер.vercel.app

Со своим доменом:

  1. Купите домен на reg.ru (~200 руб/год за .ru)
  2. В настройках Vercel добавьте свой домен
  3. Пропишите DNS-записи по инструкции Vercel
Про безопасность

Данные хранятся в localStorage вашего браузера. Экспортируйте лидов в CSV регулярно. Если очистите кэш браузера — данные пропадут.

Ожидаемый результат: парсер доступен по ссылке. Работает с телефона и компьютера.

Что дальше

Базовый парсер работает. Вот как его развивать:

1. Подключите Telegram API. Парсер будет сканировать группы автоматически и добавлять лидов в реальном времени.

2. Автоответ. Нашёл горячего лида — сразу отправляет шаблонное сообщение. Вы подключаетесь, когда клиент ответил.

3. Скоринг лидов. ИИ оценивает «температуру» по тексту сообщения. Горячие — сверху списка.

4. Уведомления в Telegram. Нашёл лида с высоким приоритетом — мгновенное сообщение вам.

5. Экспорт в CRM. Лиды автоматически попадают в вашу CRM-систему с полным контекстом.

6. Мультиканальность. Поиск одновременно в Telegram, ВКонтакте, форумах и Avito.

Результат

Парсер находит до 50 лидов в день по вашим ключевым словам. Шаблоны ускоряют первый контакт в 5 раз. Аналитика показывает лучшие источники и конверсию. Вы тратите 30 минут вместо 3 часов на поиск клиентов.

✏️Упражнение: Составьте карту ключевых слов для своей ниши

Напишите 15 ключевых слов в трёх категориях: прямой запрос (5 штук), проблема (5 штук), обсуждение (5 штук). Добавьте 5 стоп-слов.

Пример для фитнес-тренера:

Прямой запрос:

  1. «ищу тренера»
  2. «нужен фитнес-тренер»
  3. «персональные тренировки»
  4. «онлайн-тренер»
  5. «программа тренировок»

Проблема:

  1. «не могу похудеть»
  2. «болит спина от сидячей работы»
  3. «нет мотивации заниматься»
  4. «не знаю какие упражнения делать»
  5. «набрала вес после родов»

Обсуждение:

  1. «кто занимается с тренером»
  2. «посоветуйте тренера»
  3. «сколько стоят тренировки»
  4. «онлайн или в зале»
  5. «какой результат за месяц»

Стоп-слова: «бесплатно», «сам разберусь», «не нужен тренер», «шутка», «мем»

Готово! Парсер для поиска клиентов работает. Ключевые слова настроены, шаблоны готовы, аналитика считается. Тратьте время на переговоры, а не на ручной поиск.
