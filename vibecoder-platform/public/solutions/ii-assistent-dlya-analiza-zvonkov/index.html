ИИ-ассистент для анализа звонков

Руководитель отдела продаж не может прослушать 50 звонков в день. А менеджеры делают одни и те же ошибки снова и снова. Без анализа — нет роста.

Сейчас мы построим ассистента, который анализирует звонки. Загрузил запись — получил разбор: что хорошо, что плохо, что исправить.

Что понадобится

  • Lovable — AI-конструктор для создания интерфейса
  • Google AI Studio или Claude — для ИИ-анализа транскриптов звонков
Аналогия

ИИ-аналитик звонков — это как видеоповтор в спорте. Футболист не видит свои ошибки в игре — адреналин, скорость. Но на записи тренер показывает: «Вот тут ты потерял мяч, а тут — отличный пас». Наш ИИ делает то же самое с переговорами.

Быстрый старт

Если нет записей звонков — начните с текстовых переписок. Скопируйте диалог в Google AI Studio или Claude с промптом из Шага 3 — и получите анализ за 30 секунд.

Шаг 1: Подготовка

Определите:

  1. Источники звонков: IP-телефония, мобильные записи, Zoom/Google Meet
  2. Скрипт продаж — с чем сравнивать (если есть)
  3. Критерии оценки:
    • Приветствие и представление
    • Выявление потребностей (вопросы)
    • Презентация решения
    • Работа с возражениями
    • Закрытие (следующий шаг)
  4. Формат: аудио (mp3/wav) или текстовый транскрипт

Шаг 2: Создание ассистента в Lovable

  1. Откройте Lovable
  2. Создайте новый проект
  3. Вставьте промпт:
Создай ИИ-ассистента для анализа звонков на React с Tailwind CSS.

Главная страница:
— Загрузка транскрипта: textarea (вставить текст) или загрузка TXT-файла
— Выбор типа звонка: холодный, тёплый, входящий, повторный
— Имя менеджера (для истории)
— Кнопка «Анализировать»

Страница результата:
1. Общая оценка: 0-100 баллов (круговая диаграмма с цветом)
2. Оценка по критериям — 5 шкал:
   — Приветствие (0-10)
   — Вопросы (0-10)
   — Презентация (0-10)
   — Возражения (0-10)
   — Закрытие (0-10)
3. Радар-диаграмма навыков
4. Блок «Что хорошо» — 2-3 цитаты из звонка с пояснением
5. Блок «Что улучшить» — 2-3 момента + как надо было сказать
6. Ключевые фразы: что клиент говорил о боли, бюджете, сроках

Дизайн: аналитический. Светлая тема, акцентный синий. Шрифт Inter.

Ожидаемый результат: загрузка транскрипта → оценка по критериям → рекомендации.

Шаг 3: Промпт для анализа

Этот промпт работает в Google AI Studio, Claude или внутри приложения:

Ты — эксперт по анализу продающих звонков. Проанализируй транскрипт переговоров.

Оцени по шкале 0-10 каждый критерий:

1. ПРИВЕТСТВИЕ: представился ли менеджер, назвал компанию, создал позитивный контакт
2. ВОПРОСЫ: задавал ли открытые вопросы, выявил боль и потребность
3. ПРЕЗЕНТАЦИЯ: говорил выгодами (не характеристиками), привёл кейс или пример
4. ВОЗРАЖЕНИЯ: услышал возражение, не спорил, дал аргументированный ответ
5. ЗАКРЫТИЕ: предложил конкретный следующий шаг, назначил дату

Формат ответа:
— Общий балл (среднее × 10)
— Оценка каждого критерия с обоснованием
— 2 сильных момента (процитируй фразу менеджера)
— 2 слабых момента (процитируй + предложи лучший вариант)
— Ключевые сигналы клиента: боль, бюджет, сроки, возражения
— Итоговая рекомендация менеджеру (1 предложение)
Совет

Этот промпт можно вставить прямо в Google AI Studio или Claude вместе с транскриптом звонка. Результат получите за 30 секунд — без интерфейса.

Шаг 4: Статистика менеджеров

Добавь дашборд по менеджерам:

1. Таблица менеджеров:
— Имя, количество проанализированных звонков, средний балл, тренд (↑↓)

2. Профиль менеджера:
— Радар-диаграмма навыков (среднее по всем звонкам)
— График прогресса: балл по неделям
— Слабое место: «Ваш главный рост — в работе с возражениями (средний 4/10)»
— Топ-3 рекомендации для развития

3. Сравнение менеджеров:
— Bar chart: средний балл каждого
— Кто лучше всех закрывает, у кого лучшие вопросы

Шаг 5: Библиотека лучших практик

Добавь библиотеку лучших звонков:

1. «Золотые фразы» — лучшие цитаты из звонков с высоким баллом:
— Категория: приветствие / вопросы / возражения / закрытие
— Фраза менеджера
— Контекст: в какой ситуации использовал
— Результат: привело к продаже? (да/нет)

2. «Антипаттерны» — типичные ошибки:
— Фраза менеджера → Почему плохо → Как лучше

3. Поиск по библиотеке: «как ответить на 'дорого'» → подборка лучших ответов из реальных звонков.
Важно

Анализ звонков — инструмент роста, а не наказания. Используйте для обучения, не для штрафов. Иначе менеджеры перестанут записывать звонки.

Шаг 6: Деплой (публикация)

Проверь дизайн:
— Радар-диаграмма читаема
— Цитаты из звонков выделены визуально
— Мобильная версия: руководитель смотрит аналитику с телефона
— Пустое состояние: «Загрузите первый транскрипт»

Два варианта публикации:

  1. Бесплатно — на поддомене Lovable
  2. Свой домен — подключите домен
Готово!

У вас есть ИИ-аналитик звонков. Загружайте транскрипты — получайте разбор и рекомендации. Менеджеры растут, конверсия увеличивается.

Что дальше

  • Подключите Speech-to-Text (Whisper API) для автоматической транскрипции аудио
  • Интегрируйте с IP-телефонией для автоматического анализа каждого звонка
  • Добавьте еженедельный отчёт руководителю в Telegram
  • Используйте Google AI Studio для поиска паттернов: какие фразы коррелируют с продажей
  • Настройте автоматические тренировки: после слабого звонка → рекомендация сценария

✏️Упражнение: Практика: проанализируйте звонок

Возьмите транскрипт последнего звонка (или переписку с клиентом). Загрузите в ассистента или вставьте в Google AI Studio с промптом из Шага 3.

Пример результата анализа:

Общий балл: 62/100

КритерийБаллКомментарий
Приветствие8/10Хорошо представился, назвал компанию
Вопросы5/10Только 2 вопроса, оба закрытых
Презентация7/10Упомянул кейс, но без цифр
Возражения4/10На «дорого» ответил скидкой (ошибка!)
Закрытие6/10Предложил «подумать» вместо конкретного шага

Главная рекомендация: Задавайте больше открытых вопросов перед презентацией. На возражение «дорого» — покажите ROI, а не давайте скидку.
