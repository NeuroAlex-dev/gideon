Чат-бот для квалификации лидов

Для отделов продаж

Проблема
Менеджеры тратят время на нецелевых клиентов
Результат
Бот квалифицирует лиды и распределяет по менеджерам. Конверсия отдела +25%
Lovable2-4 часа

Что понадобится

  • Lovable — AI-конструктор сайтов и приложений
Аналогия

Чат-бот для квалификации — как администратор в поликлинике. Он спрашивает, что беспокоит, и направляет к нужному врачу. Без него все толпятся у одного кабинета. С ним — каждый сразу попадает к своему специалисту.

Шаг 1: Подготовка

Перед созданием бота определите критерии квалификации.

Что нужно решить:

  • По каким признакам разделять лидов (бюджет, срочность, размер компании)
  • Сколько категорий лидов вам нужно (горячий, тёплый, холодный)
  • Какой менеджер работает с какой категорией
  • Какие вопросы задать, чтобы определить категорию
Совет

Используйте методику BANT: Budget (бюджет), Authority (полномочия), Need (потребность), Timeline (сроки). Четыре вопроса — и вы знаете, насколько лид горячий.

Пример матрицы квалификации:

КритерийГорячий лидТёплый лидХолодный лид
БюджетОпределёнПримерныйНе определён
СрокиДо 2 недельДо 2 месяцевНе определены
ПолномочияПринимает решениеВлияет на решениеПросто изучает
ПотребностьКонкретная задачаОбщий интересЛюбопытство

Шаг 2: Создание проекта в Lovable

  1. Откройте Lovable
  2. Нажмите «New Project»
  3. Вставьте промпт:
Создай веб-приложение «Квалификация лидов» на русском языке.

Виджет чат-бота:
- Плавающая кнопка в правом нижнем углу
- При клике открывается окно чата
- Бот приветствует: «Здравствуйте! Помогу найти подходящее решение.
  Ответьте на 4 коротких вопроса — это займёт 1 минуту»
- Вопросы задаются по одному, с вариантами ответов (кнопки)
- После ответов — бот показывает рекомендацию
- Форма для контакта: имя, телефон, email

Вопросы бота:
1. Какая у вас задача? (Варианты: Запустить новый проект / Улучшить
   существующий / Нужна консультация / Другое)
2. Какой у вас бюджет? (До 100 тыс / 100-500 тыс / Более 500 тыс /
   Пока не определён)
3. Когда хотите начать? (На этой неделе / В этом месяце / В ближайшие
   3 месяца / Пока просто изучаю)
4. Кто принимает решение? (Я сам / Нужно согласовать / Это групповое
   решение)

Дизайн: минималистичный, сине-белая палитра.
Адаптивная вёрстка для мобильных устройств.
Совет

Замените вопросы и варианты ответов на свои. Каждый бизнес квалифицирует лидов по-разному. Вопросы выше — шаблон для старта.

Ожидаемый результат: Чат-виджет с четырьмя вопросами и формой сбора контактов.

Шаг 3: Логика скоринга

Бот должен ставить баллы за каждый ответ. Это и есть ключевая механика квалификации.

Отправьте в чат Lovable:

Добавь систему скоринга лидов:

- Каждый ответ даёт баллы от 1 до 4
- Первый вариант ответа — 4 балла, последний — 1 балл
- После всех вопросов подсчитай общий балл
- 13-16 баллов — горячий лид (красная метка)
- 9-12 баллов — тёплый лид (жёлтая метка)
- 4-8 баллов — холодный лид (синяя метка)
- Покажи категорию лида после заполнения формы контакта
- Сохрани все ответы и балл в базу данных

Ожидаемый результат: Бот автоматически определяет категорию лида по ответам.

Почему скоринг работает

Без скоринга менеджер тратит 15 минут на первый звонок. С ним — он сразу знает, горячий это лид или холодный. Горячему звонит первым. Холодному отправляет автописьмо.

Шаг 4: Распределение по менеджерам

Лид квалифицирован. Теперь бот должен направить его к нужному менеджеру.

Напишите в чат Lovable:

Добавь автоматическое распределение лидов:

- Горячие лиды — назначить на старшего менеджера
- Тёплые лиды — распределить между менеджерами по очереди
- Холодные лиды — добавить в список для email-рассылки
- При назначении — отправлять уведомление менеджеру
- Показать уведомление как зелёный тост внизу экрана
- Добавь страницу /managers для настройки списка менеджеров
- У каждого менеджера: имя, email, категория лидов, лимит в день

Ожидаемый результат: Лиды автоматически попадают к нужному менеджеру. Никто не простаивает, никто не перегружен.

Важно

Настройте лимит лидов на менеджера в день. Без лимита один менеджер получит все горячие лиды и не успеет их обработать. Качество упадёт.

Шаг 5: Админ-панель и аналитика

Без аналитики вы не узнаете, работает ли бот. Нужна панель с ключевыми метриками.

Добавь панель аналитики по маршруту /admin:

Дашборд:
- Всего лидов за сегодня / неделю / месяц
- Распределение по категориям: горячие, тёплые, холодные (диаграмма)
- Конверсия из чата в заполненную форму (процент)
- Среднее время прохождения опроса
- На каком вопросе чаще всего уходят

Таблица лидов:
- Имя, контакт, категория, балл, назначенный менеджер, дата
- Фильтры по категории, менеджеру и дате
- Сортировка по баллу и дате
- Экспорт в CSV

График воронки:
- Открыли чат → начали опрос → закончили → оставили контакт

Ожидаемый результат: Вы видите полную картину: сколько лидов приходит, как они квалифицируются и где теряются.

Совет

Следите за метрикой «на каком вопросе уходят». Если 40% людей закрывают чат на вопросе о бюджете — переформулируйте его. Возможно, он слишком прямолинейный.

Шаг 6: Деплой (публикация)

Вариант 1 — бесплатный:

В Lovable нажмите «Share» → «Publish». Получите ссылку вида проект.lovable.app. Подходит для тестирования с командой.

Вариант 2 — свой домен:

  1. Купите домен (например, bot.вашсайт.ru)
  2. В настройках проекта добавьте свой домен
  3. Настройте DNS-записи у регистратора домена
Встраивание на сайт

Чат-виджет можно встроить на существующий сайт через iframe. Скопируйте код встраивания из настроек проекта и вставьте на нужные страницы.

Готово

Чат-бот квалифицирует лиды автоматически. Горячие — сразу к старшему менеджеру. Тёплые — в очередь. Холодные — на прогрев. Менеджеры работают только с целевыми клиентами.

Что дальше

  • Подключите Telegram-бота для уведомлений менеджерам в реальном времени
  • Добавьте интеграцию с CRM (amoCRM, Bitrix24) для автоматической передачи лидов
  • Настройте автоматические email-цепочки для холодных лидов
  • Добавьте A/B тестирование разных вопросов и вариантов ответов
  • Создайте отчёт по эффективности каждого менеджера
  • Добавьте второй сценарий бота для повторных обращений

✏️Упражнение: Практика: настройте бота для своего бизнеса

Создайте чат-бота для квалификации лидов вымышленной студии веб-разработки «СайтПро».

Условия:

  • Услуги: лендинг, интернет-магазин, корпоративный сайт
  • Бюджеты: до 50 тыс., 50-200 тыс., более 200 тыс.
  • Два менеджера: Анна (лендинги) и Борис (магазины и корпоративные)

Задача: соберите бота в Lovable, адаптируйте вопросы под эти услуги и настройте распределение между Анной и Борисом.

Проверьте:

  • Бот задаёт вопросы по одному
  • Скоринг корректно считает баллы
  • Лиды распределяются по менеджерам согласно правилам
  • Админ-панель отображает все заявки

Используйте промпты из шагов 2-5. Замените вопрос о задаче на: «Какой сайт вам нужен? (Лендинг / Интернет-магазин / Корпоративный сайт / Не уверен)». Замените вопрос о бюджете на диапазоны «СайтПро». В шаге 4 настройте: лендинги — Анне, магазины и корпоративные — Борису. Проверьте весь цикл от чата до назначения менеджера. Пройдите опрос три раза с разными ответами и убедитесь, что категории лидов определяются верно.
