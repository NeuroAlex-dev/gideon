Дашборд аналитики продаж

Данные разбросаны по кабинетам маркетплейсов. Одна вкладка — Wildberries, другая — Ozon, третья — Excel. Единой картины нет. Создадим дашборд, где все метрики на одном экране: продажи, остатки, маржа, тренды. Решения на основе данных, а не интуиции.

Что понадобится

  • Lovable — AI-помощник, который создаёт веб-приложения за вас
  • Данные о продажах — выгрузка из кабинета маркетплейса (Excel или CSV)
  • Список метрик — что именно хотите отслеживать
  • Домен (по желанию) — если хотите свой адрес вместо бесплатного .vercel.app

Код писать не нужно. Lovable сделает всё по описанию.

Аналогия

Дашборд аналитики — это кабина пилота в самолёте. Все приборы перед глазами: скорость, высота, топливо. Пилот не летает вслепую. И вы не должны продавать вслепую. Данные — ваш штурвал.

Шаг 1: Подготовка

Прежде чем открыть Lovable, определите ключевые метрики бизнеса.

Выпишите в блокнот:

  1. Ключевые метрики (KPI) — что отслеживаете каждый день
  2. Товары — названия, себестоимость, цена продажи
  3. Период — за какой период смотрите данные (день, неделя, месяц)
  4. Источники — откуда берёте данные (маркетплейсы, CRM, Excel)

Пример для продавца на маркетплейсе:

МетрикаЧто показываетПочему важна
ВыручкаОбщая сумма продажМасштаб бизнеса
МаржаВыручка минус себестоимость и комиссииРеальный заработок
Средний чекВыручка / количество заказовЭффективность ассортимента
КонверсияЗаказы / просмотры карточкиКачество карточки товара
ОстаткиСколько товара на складеКогда пора дозаказать
Топ товаровЛидеры по продажамКуда вкладывать деньги
Совет

Начните с 4–6 метрик. Больше — запутаетесь. Дашборд должен давать ответ за 5 секунд. Если нужно думать — метрик слишком много.

Шаг 2: Создание дашборда в Lovable

Откройте lovable.dev, создайте новый проект и вставьте этот промпт:

Создай дашборд аналитики продаж для продавца на маркетплейсах на React.

Требования:
1. Верхняя панель с KPI-карточками (4 штуки в ряд):
   - Общая выручка (с иконкой и процентом изменения за период)
   - Количество заказов (с процентом изменения)
   - Средний чек (с процентом изменения)
   - Маржинальность (с процентом изменения)
2. График выручки по дням (линейный chart за последние 30 дней)
3. Круговая диаграмма: распределение продаж по категориям товаров
4. Таблица топ-10 товаров: название, продажи (шт.), выручка, маржа, остаток
5. Переключатель периода: "Сегодня", "Неделя", "Месяц", "Квартал"
6. Цвета: тёмная тема (тёмный фон, светлый текст) или светлая на выбор
7. Адаптивный дизайн (мобильные + десктоп)
8. Используй Tailwind CSS, shadcn/ui и Recharts для графиков

Замените данные на свои. Нажмите Enter.

Lovable создаст дашборд за 2–3 минуты.

Важно

Не закрывайте вкладку во время генерации. Если закроете — процесс прервётся и придётся начинать заново.

Ожидаемый результат: Страница с KPI-карточками, графиками и таблицей товаров. Данные пока демонстрационные.

Шаг 3: Загрузка данных

Дашборд создан, но с демо-данными. Добавим загрузку реальных данных.

Добавь возможность загрузки данных:

1. Кнопка "Загрузить данные" в правом верхнем углу
2. Drag-and-drop зона для CSV / Excel файла
3. Парсинг CSV с колонками:
   - Дата (формат: ДД.ММ.ГГГГ)
   - Название товара
   - Количество (штук)
   - Цена продажи
   - Себестоимость
   - Комиссия маркетплейса
   - Категория товара
4. После загрузки — автоматический пересчёт всех метрик и графиков
5. Сохранение данных в localStorage (чтобы не загружать каждый раз)
6. Кнопка "Очистить данные" для сброса

Пример строки CSV:
01.01.2026,Чехол для iPhone 16,5,1200,400,180,Аксессуары
Откуда взять CSV

В кабинете Wildberries: Аналитика — Отчёт по продажам — Скачать. В кабинете Ozon: Финансы — Отчёт о реализации. Скачайте и загрузите в дашборд.

Ожидаемый результат: Загружаете CSV — графики и таблицы обновляются реальными данными. Данные сохраняются в браузере.

Шаг 4: Графики и визуализация

Сделаем графики информативнее. Добавим сравнение периодов и тренды.

Улучши визуализацию данных:

1. Линейный график выручки: две линии (текущий и предыдущий период),
   подсказка при наведении, зелёный — рост, красный — падение
2. Столбчатая диаграмма: заказы по дням недели,
   подсветка лучшего и худшего дня
3. Таблица товаров: сортировка по любому столбцу,
   маржа > 30% — зелёный, 15-30% — жёлтый, < 15% — красный,
   остатки < [МИНИМУМ] штук — красный (пора заказать)
4. KPI-карточки: стрелки вверх/вниз, мини-спарклайн внутри,
   зелёный фон при росте, красный при падении
5. Фильтр по категориям: при выборе — все графики обновляются
Про цветовые индикаторы

Цвета на дашборде — как сигналы светофора. Зелёный — всё хорошо, едем дальше. Жёлтый — внимание, присмотрись. Красный — стоп, нужно действовать. Решения принимаются за секунды.

Ожидаемый результат: Графики интерактивные, с подсказками и сравнением. Таблица сортируется, цвета помогают быстро оценить ситуацию.

Шаг 5: Экспорт и отчёты

Данные нужно не только смотреть, но и делиться ими.

Добавь экспорт данных и ручной ввод:

1. Кнопка "Скачать отчёт" — экспорт таблицы товаров в CSV
2. Кнопка "Скачать PDF" — отчёт с KPI, графиком выручки и топ-10 товаров
3. Кнопка "Печать" — открыть диалог печати браузера
4. Форма ручного ввода:
   - Поля: дата, товар, количество, цена, себестоимость, комиссия
   - Кнопка "Добавить запись"
   - Таблица с редактированием и удалением строк
Совет про ручной ввод

Ручной ввод — для начала. 5–10 товаров удобнее вбить руками, чем разбираться с выгрузкой. Когда товаров станет больше — перейдёте на CSV.

Ожидаемый результат: Можно скачать данные в CSV, сгенерировать PDF-отчёт, вручную добавлять записи.

Шаг 6: Дизайн и деплой

Сделаем дашборд красивым и опубликуем в интернете.

Улучши дизайн дашборда:

1. Тёмная тема (по умолчанию):
   - Фон: #1a1a2e, карточки: #16213e с тенью
   - Акцент: #4ade80 (зелёный) для роста, #f87171 (красный) для падения
2. Переключатель тем: тёмная / светлая (иконка солнце/луна)
3. Боковая панель: "Обзор", "Товары", "Загрузка данных", "Отчёты"
4. Шапка: "[НАЗВАНИЕ МАГАЗИНА] — Аналитика", дата обновления, переключатель периода
5. Мобильная версия: KPI-карточки 2 в ряд, графики на полную ширину, горизонтальный скролл таблицы

Деплой:

  1. В Lovable нажмите "Deploy" (вверху справа)
  2. Выберите "Deploy to Vercel"
  3. Подключите GitHub аккаунт (если нет — создайте бесплатно)
  4. Вы получите ссылку вроде: https://sales-dashboard-abc.vercel.app
  5. По желанию — подключите свой домен в настройках Vercel
Про безопасность данных

Данные хранятся в вашем браузере (localStorage). Никто кроме вас их не видит. Но если очистите кэш — данные пропадут. Делайте экспорт в CSV.

Как использовать:

  • Открывайте утром — 5 минут на оценку вчерашнего дня
  • Загружайте свежую выгрузку раз в неделю
  • Скачивайте PDF-отчёт для партнёров или инвесторов

Что дальше

Базовый дашборд готов. Вот как его развивать:

1. Автоматический импорт — подключите API маркетплейса, данные обновляются сами.

2. Прогнозирование — прогноз продаж на основе тренда. Дашборд подскажет, сколько закупать.

3. Уведомления — остатки ниже минимума? Маржа упала? Уведомление в Telegram.

4. Сравнение маркетплейсов — где продаётся лучше: на WB или Ozon?

5. ABC-анализ — классификация товаров. A-лидеры, B-середняки, C-аутсайдеры.

Итог

Вы создали дашборд аналитики за 2–4 часа. Все метрики на одном экране. Больше не нужно переключаться между вкладками и сводить данные в Excel. Решения принимаются на основе цифр.

✏️Упражнение: Соберите свой первый отчёт

Загрузите CSV за последний месяц. Ответьте на три вопроса: какой товар принёс больше всего маржи? В какой день недели больше заказов? Какой товар пора дозаказать?

  1. Топ по марже — отсортируйте таблицу по столбцу «Маржа» по убыванию. Первая строка — лидер.
  2. Лучший день — столбчатая диаграмма заказов по дням. Самый высокий столбик — ответ.
  3. Дозаказ — товары с красным индикатором остатков. Это приоритет на закупку.

Мало данных? Начните вносить вручную. Даже 10 записей покажут картину.

Готово! Ваш дашборд аналитики работает. Данные на одном экране, графики показывают тренды, таблица подсвечивает проблемы. Управляйте продажами по цифрам, а не по ощущениям.
