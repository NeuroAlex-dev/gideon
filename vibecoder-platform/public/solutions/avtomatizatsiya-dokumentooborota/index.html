Автоматизация документооборота

Каждый договор — ручная работа. Скопировать шаблон, заменить реквизиты, проверить даты, исправить ошибки. Два часа на пакет документов. Создадим систему автоматической генерации: выбрал шаблон, заполнил поля, получил готовый документ. 10 минут вместо двух часов.

Что понадобится

  • Lovable — AI-конструктор сайтов и приложений
  • Шаблоны документов — ваши типовые договоры, акты, счета
  • Реквизиты компании — ИНН, ОГРН, адрес, банковские данные
  • Домен (по желанию) — если хотите свой адрес вместо бесплатного

Код писать не нужно. Lovable соберёт всё по вашему описанию.

Аналогия

Автоматизация документов — это печатный станок вместо переписчика. Раньше монах копировал книгу месяцами. Станок печатает за минуты. Вы не копируете шаблон вручную — система заполняет его сама. Одна кнопка — готовый документ.

Шаг 1: Подготовка

Прежде чем открыть Lovable, соберите все типовые документы.

Выпишите в блокнот:

  1. Типы документов — что создаёте чаще всего
  2. Переменные поля — что меняется от документа к документу
  3. Реквизиты — данные вашей компании и типовых контрагентов
  4. Нумерация — как нумеруете документы

Пример типовых документов:

Совет

Начните с одного документа — самого частого. Обычно это счёт или договор. Отладьте его, потом добавьте остальные. Один работающий шаблон лучше пяти недоделанных.

Шаг 2: Создание проекта в Lovable

Откройте Lovable, создайте новый проект и вставьте промпт:

Создай систему автоматической генерации документов на React.

Требования:
1. Дашборд:
   - Количество документов за месяц
   - Последние созданные документы (список)
   - Быстрые кнопки: "Новый договор", "Новый акт", "Новый счёт"
2. Раздел "Шаблоны":
   - Карточки шаблонов: Договор на услуги, Акт выполненных работ, Счёт на оплату
   - Кнопка "Создать документ" на каждой карточке
3. Форма создания документа:
   - Поля зависят от типа шаблона
   - Общие поля: номер, дата, контрагент
   - Автонумерация: [ПРЕФИКС]-[ГОД]-[НОМЕР]
   - Выбор контрагента из справочника или ввод нового
4. Предпросмотр документа — оформление в стиле официального документа
5. Кнопка "Скачать PDF"
6. Адаптивный дизайн (мобильные + десктоп)
7. Данные сохраняются в localStorage
8. Используй Tailwind CSS и shadcn/ui

Нажмите Enter. Lovable создаст систему за 2–3 минуты.

Важно

Не закрывайте вкладку во время генерации. Если закроете — процесс прервётся и придётся начинать заново.

Ожидаемый результат: Дашборд с быстрыми кнопками. Карточки шаблонов. Форма создания с предпросмотром.

Шаг 3: Справочник контрагентов

Одни и те же клиенты — в каждом документе. Заполнять реквизиты каждый раз — потеря времени.

Добавь справочник контрагентов:

1. Раздел "Контрагенты" в боковом меню
2. Таблица контрагентов:
   - Наименование (ООО, ИП, физлицо)
   - ИНН
   - ОГРН / ОГРНИП
   - Юридический адрес
   - Банковские реквизиты: банк, БИК, расчётный счёт, корр. счёт
   - Контактное лицо, email, телефон
3. Форма добавления / редактирования контрагента
4. Поиск по названию или ИНН
5. При создании документа — выпадающий список контрагентов
6. Выбрал контрагента — все реквизиты подставились автоматически
7. Кнопка "Добавить нового" прямо из формы документа
Про справочник

Справочник — это ваша адресная книга для бизнеса. Добавьте клиента один раз — и его данные подставляются во все документы. Ошибки в реквизитах исчезнут.

Ожидаемый результат: Список контрагентов с реквизитами. Автоподстановка при создании документа.

Шаг 4: Шаблоны документов

Каждый тип документа — свой набор полей и своё оформление.

Настрой шаблоны документов:

1. Договор на услуги:
   - Поля: номер, дата, контрагент, предмет договора, перечень услуг (таблица: услуга, количество, цена, сумма), общая сумма, срок выполнения, условия оплаты
   - Оформление: шапка с реквизитами обеих сторон, преамбула, пункты договора, подписи сторон
2. Акт выполненных работ:
   - Поля: номер, дата, контрагент, ссылка на договор, перечень работ (таблица), общая сумма
   - Оформление: шапка, таблица работ, подписи, печати
3. Счёт на оплату:
   - Поля: номер, дата, контрагент, перечень услуг (таблица), итого, НДС (если применимо), срок оплаты
   - Оформление: банковские реквизиты вверху, таблица услуг, итого
4. Реквизиты моей компании — заполняются один раз в настройках:
   - Название, ИНН, ОГРН, адрес, банк, БИК, расчётный счёт
Про шаблоны

Шаблон — это форма для выпечки. Тесто (данные) каждый раз разное. Но форма (структура документа) одна. Заливаете — получаете одинаково ровный результат. Без формы — каждый пирог разный.

Ожидаемый результат: Три настроенных шаблона. Каждый со своими полями и оформлением.

Шаг 5: Журнал и поиск документов

Документов будет много. Нужен порядок и быстрый поиск.

Добавь журнал документов и поиск:

1. Раздел "Журнал" — таблица всех созданных документов:
   - Номер, тип, дата, контрагент, сумма, статус
2. Статусы документа:
   - Черновик (серый) — не отправлен
   - Отправлен (синий) — передан клиенту
   - Подписан (зелёный) — обе стороны подписали
   - Отменён (красный) — аннулирован
3. Поиск по номеру, контрагенту или сумме
4. Фильтры: по типу документа, по статусу, по периоду
5. Связка документов: к договору привязаны акты и счета
   - В карточке договора — список связанных актов и счетов
   - Кнопка "Создать акт к договору" — данные подставляются
6. KPI на дашборде:
   - Документов за месяц
   - Сумма по договорам
   - Неподписанные документы
7. Экспорт журнала в CSV
Совет про связки

Связка «договор — акт — счёт» экономит больше всего времени. Создали договор. Работа сделана — нажали «Создать акт». Данные подставились. 30 секунд вместо 20 минут.

Ожидаемый результат: Журнал с поиском и фильтрами. Связка документов. Создание акта из договора в один клик.

Шаг 6: Деплой (публикация)

Доработаем дизайн и опубликуем систему.

Улучши дизайн системы документооборота:

1. Профессиональная тема:
   - Фон: #f8fafc, карточки: белые с тенью
   - Акцент: синий #2563eb для кнопок и ссылок
2. Боковая панель: "Обзор", "Документы", "Шаблоны", "Контрагенты", "Настройки"
3. Шапка: "[НАЗВАНИЕ КОМПАНИИ] — Документы", счётчик новых
4. Карточки шаблонов с иконками: договор (рукопожатие), акт (галочка), счёт (монета)
5. Предпросмотр документа на всю ширину экрана
6. Мобильная версия: табы вместо боковой панели

Вариант 1 — бесплатный поддомен: в Lovable нажмите Share — Publish. Получите ссылку вроде docs-generator-abc.lovable.app.

Вариант 2 — свой домен: нажмите Deploy — Deploy to Vercel. Подключите GitHub. Получите ссылку https://my-docs.vercel.app. Привяжите свой домен в настройках Vercel.

Про хранение данных

Данные хранятся в localStorage браузера. Для командной работы нужна база данных — Supabase или Firebase. На старте для одного рабочего места localStorage достаточно.

Что дальше

Базовая система готова. Вот как её развивать:

1. Электронная подпись — отправка документа на подпись через email. Клиент нажимает «Подписать» — статус меняется.

2. База данных — подключите Supabase для командной работы. Все сотрудники видят одну базу документов.

3. Telegram-уведомления — новый документ, смена статуса, напоминание о неподписанных.

4. Новые шаблоны — доверенности, дополнительные соглашения, письма. Каждый шаблон — один промпт в Lovable.

5. Интеграция с бухгалтерией — экспорт данных в формате для «1С» или «Моё дело».

6. Автозаполнение по ИНН — ввёл ИНН контрагента, система подтянула название и реквизиты из открытых данных.

Итог

Вы создали систему документооборота за 2–4 часа. Шаблоны заполняются автоматически. Реквизиты подставляются из справочника. Документы связаны между собой. 10 минут на пакет документов вместо двух часов.

✏️Упражнение: Создайте комплект документов для клиента

Добавьте одного реального контрагента. Создайте для него договор, акт и счёт. Ответьте: сколько времени ушло на три документа? Все ли реквизиты подставились? Что нужно поправить в шаблонах?

  1. Время — если больше 15 минут на три документа, проверьте справочник контрагентов. Все реквизиты должны подставляться автоматически. Ручной ввод — только уникальные поля (сумма, перечень услуг).
  2. Реквизиты — откройте предпросмотр. Сравните с образцом вашего обычного документа. Если чего-то не хватает (КПП, ОКПО) — добавьте поле в настройках.
  3. Шаблоны — покажите готовый документ бухгалтеру или юристу. Они скажут, какие пункты добавить. Один промпт в Lovable — и шаблон обновлён.

Всё работает? Добавьте остальных постоянных контрагентов. Чем больше данных в справочнике — тем быстрее создание документов.

Готово! Ваша система документооборота работает. Договоры, акты и счета генерируются за минуты. Реквизиты не теряются, нумерация не сбивается, ошибки не проскакивают. Документы — на автопилоте.
