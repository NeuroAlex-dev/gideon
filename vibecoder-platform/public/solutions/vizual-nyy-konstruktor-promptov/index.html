Визуальный конструктор промптов

Каждый раз угадываете формулировку промпта. Результат нестабильный. Что сработало вчера — не работает сегодня. Создадим конструктор, где промпты собираются из блоков. Стабильный результат каждый раз.

Что понадобится

  • Lovable — AI-конструктор сайтов и приложений
  • Опыт работы с нейросетями — базовое понимание промптов
  • Домен (по желанию) — если хотите свой адрес вместо бесплатного

Код писать не нужно. Lovable создаёт приложения по описанию.

Аналогия

Обычный промпт — это рецепт, написанный от руки. Каждый раз получается по-разному. Конструктор промптов — это кулинарная форма. Кладёте ингредиенты в ячейки, получаете одинаковый результат. Без угадывания.

Шаг 1: Подготовка

Прежде чем открыть Lovable, определите структуру промптов для вашей задачи.

Запишите в блокнот:

  1. Для чего промпты — тексты, код, изображения, анализ данных
  2. Частые элементы — какие блоки повторяются в каждом промпте
  3. Переменные — что меняется от запроса к запросу
  4. Примеры хороших промптов — 3-5 промптов, которые давали отличный результат
  5. Типичные ошибки — что обычно идёт не так

Пример для задачи «Генерация текстов для соцсетей»:

  • Блоки: роль, контекст, задача, формат, тон, ограничения
  • Переменные: тема поста, целевая аудитория, платформа
  • Ошибки: слишком общий промпт, нет примеров, не указан формат
Совет

Разберите 5 лучших промптов на части. Найдите повторяющиеся элементы. Они станут блоками вашего конструктора.

✏️Упражнение: Разберите промпт на блоки

Возьмите любой промпт, который вы часто используете. Разделите его на смысловые части: роль, задача, контекст, формат, ограничения.

Пример разбора промпта для генерации постов:

  • Роль: «Ты — SMM-специалист с 5 годами опыта»
  • Контекст: «Канал про маркетинг, 5000 подписчиков»
  • Задача: «Напиши пост про email-маркетинг»
  • Формат: «3-5 абзацев, с примером из практики»
  • Тон: «Дружеский, без канцеляризмов»
  • Ограничения: «До 1500 символов, без эмодзи»

Каждая часть — это отдельный блок в конструкторе.

Шаг 2: Создание проекта в Lovable

Откройте Lovable, создайте новый проект и вставьте этот промпт:

Создай веб-приложение "Конструктор промптов" на React.

Требования:
1. Шапка: логотип "Prompt Builder", навигация: "Конструктор",
   "Шаблоны", "Мои промпты", "Справка"

2. Главная страница — конструктор промптов:
   - Левая панель "Блоки" — категории блоков:
     Роль, Контекст, Задача, Формат, Тон, Примеры, Ограничения
   - Каждая категория раскрывается и содержит 4-6 готовых блоков
   - Блоки перетаскиваются drag-and-drop в правую панель

   - Правая панель "Ваш промпт":
     - Область для перетаскивания блоков (вертикальный список)
     - Каждый блок можно редактировать, удалить, перемещать
     - Внизу: "Итоговый промпт" — собранный текст
     - Кнопки: "Копировать", "Сохранить", "Очистить"

3. Дизайн: светлая тема, акцент — фиолетовый #7C3AED.
   Tailwind CSS, shadcn/ui. Адаптивный макет.
4. Блоки — карточки с иконками и кратким описанием

Нажмите Enter. Lovable создаст приложение за 2-3 минуты.

Важно

Если drag-and-drop не заработал с первого раза, попросите Lovable: «Исправь drag-and-drop. Блоки должны перетаскиваться из левой панели в правую.»

Ожидаемый результат: Двухпанельный интерфейс. Слева — блоки по категориям. Справа — область сборки промпта. Блоки перетаскиваются и редактируются.

Шаг 3: Библиотека блоков

Наполним конструктор готовыми блоками для каждой категории.

Наполни конструктор промптов готовыми блоками:

1. Категория "Роль" (6 блоков):
   - "Эксперт" → "Ты — эксперт в [ТЕМА] с 10+ годами опыта"
   - "Копирайтер" → "Ты — копирайтер для [АУДИТОРИЯ]"
   - "Аналитик" → "Ты — аналитик данных в сфере [СФЕРА]"
   - "Учитель" → "Ты — учитель, объясняющий просто"
   - "Маркетолог" → "Ты — маркетолог с фокусом на [КАНАЛ]"
   - "Программист" → "Ты — разработчик на [ЯЗЫК]"

2. Категория "Задача" (6 блоков):
   - "Написать" → "Напиши [ТИП ТЕКСТА] на тему [ТЕМА]"
   - "Проанализировать" → "Проанализируй [ОБЪЕКТ]"
   - "Создать план" → "Создай пошаговый план для [ЦЕЛЬ]"
   - "Переписать" → "Перепиши текст в стиле [СТИЛЬ]"
   - "Сравнить" → "Сравни [A] и [B] по критериям: [КРИТЕРИИ]"
   - "Ответить" → "Ответь на вопрос: [ВОПРОС]"

3. Категория "Формат" (5 блоков):
   - "Список", "Таблица", "Пошаговая инструкция", "JSON", "Абзацы"

4. Категории "Контекст", "Тон", "Примеры", "Ограничения" —
   по 4-5 блоков в каждой

5. Переменные в скобках [ТЕМА] подсвечиваются цветом.
   При клике — поле для ввода значения.
Совет

Переменные в квадратных скобках — это сила конструктора. Один шаблон с переменными заменяет десятки ручных промптов. Заполните переменные — получите готовый промпт.

Ожидаемый результат: 30+ готовых блоков по 7 категориям. Переменные подсвечиваются. При клике на переменную открывается поле ввода.

Шаг 4: Шаблоны промптов

Добавим готовые шаблоны — промпты, собранные из блоков.

Добавь страницу "Шаблоны" с готовыми промптами:

1. Сетка карточек (3 в ряд, адаптивная):
   - "Пост для соцсетей" — роль + контекст + задача + тон + формат
   - "Анализ конкурентов" — роль + задача + формат + ограничения
   - "Email-рассылка" — роль + задача + тон + ограничения
   - "Описание продукта" — роль + контекст + задача + формат
   - "Контент-план" — роль + задача + формат + ограничения
   - "Код функции" — роль + задача + формат + ограничения

2. Каждая карточка: иконка, название, описание,
   кнопки "Открыть в конструкторе" и "Копировать промпт"
3. Фильтр по тегам и поиск по названию
Зачем шаблоны

Шаблоны экономят время. Вместо сборки промпта с нуля — выберите шаблон, измените переменные, скопируйте. 30 секунд вместо 5 минут.

Ожидаемый результат: 6 готовых шаблонов с фильтрами. Каждый открывается в конструкторе или копируется одной кнопкой.

Шаг 5: Сохранение и история

Добавим страницу «Мои промпты» для сохранения и повторного использования.

Добавь страницу "Мои промпты" с сохранением и историей:

1. Список сохранённых промптов:
   - Карточки: название, дата, категория, первые 100 символов
   - Кнопки: "Открыть", "Копировать", "Удалить", "Редактировать"

2. История последних 20 промптов:
   - Автосохранение каждого скопированного промпта
   - Дата, время, текст, кнопка "Повторить"

3. Папки: "Тексты", "Код", "Маркетинг", "Анализ" + создание новых

4. Экспорт/импорт промптов в JSON. Данные в localStorage.
Аналогия

Сохранённые промпты — это ваш кулинарный блокнот. Записали рецепт один раз — пользуетесь годами. Не нужно каждый раз вспоминать пропорции. Открыли, скопировали, готово.

Ожидаемый результат: Библиотека сохранённых промптов с папками. Автоматическая история. Экспорт и импорт через JSON.

Шаг 6: Деплой (публикация)

Опубликуем конструктор, чтобы пользоваться с любого устройства.

Вариант 1 — бесплатный поддомен Lovable:

  1. В Lovable нажмите «Deploy» (вверху справа)
  2. Выберите «Deploy to Vercel»
  3. Получите ссылку вроде: https://prompt-builder-abc.vercel.app
  4. Готово — конструктор доступен по ссылке

Вариант 2 — свой домен:

  1. Купите домен (например, на reg.ru или Timeweb)
  2. В настройках Vercel добавьте свой домен
  3. Обновите DNS-записи у регистратора
  4. Через 5-10 минут сайт доступен по вашему адресу
Итог

Вы создали визуальный конструктор промптов за 2-4 часа. Блоки, шаблоны, переменные, история. Промпты собираются за 30 секунд. Результат стабильный каждый раз.

Что дальше

Базовый конструктор промптов готов. Вот как его развивать:

  • Оценка качества — AI анализирует промпт и даёт советы по улучшению
  • Совместная работа — делитесь шаблонами с командой по ссылке
  • Маркетплейс шаблонов — пользователи публикуют и продают свои шаблоны
  • Интеграция с AI — кнопка «Тестировать» отправляет промпт в нейросеть прямо из конструктора
  • Версионирование — история изменений каждого промпта, откат к предыдущей версии
  • Подписка — бесплатная версия с 10 шаблонами, платная — безлимит

✏️Упражнение: Соберите промпт из блоков

Соберите промпт для задачи: «Написать пост для Telegram-канала про продуктивность». Используйте минимум 4 блока: роль, задача, формат, ограничения.

Пример собранного промпта:

  • Роль: «Ты — копирайтер для предпринимателей»
  • Контекст: «Telegram-канал про продуктивность, 3000 подписчиков»
  • Задача: «Напиши пост: как планировать день без выгорания»
  • Формат: «3-4 абзаца, с примером из практики»
  • Ограничения: «До 1200 символов. Без списков.»

Все блоки склеиваются в один текст. Скопировали — вставили в нейросеть — получили результат.

Готово! Ваш визуальный конструктор промптов работает. Блоки вместо угадывания, шаблоны вместо пустого поля, история вместо потерянных промптов. Собирайте промпты за секунды — получайте стабильный результат.
