Автоматическая email-воронка

Подписчики есть, а продаж мало. Люди подписываются — и забывают о вас через день. Создайте автоматическую воронку писем, которая прогревает подписчика и доводит до покупки. Без ручной работы.

Что понадобится

  • Lovable — AI-помощник для создания веб-приложений. Пишет код за вас.
  • Продукт или услуга — то, что вы продаёте через воронку.
  • 5–7 писем — тексты или хотя бы тезисы для каждого письма.
  • Домен (по желанию) — свой адрес вместо бесплатного .vercel.app.

Время на создание: 3–4 часа. Код писать не нужно.

Аналогия

Email-воронка — это продавец, который лично ведёт каждого клиента. Пока вы спите — он знакомит с продуктом, отвечает на возражения и предлагает купить. Десять подписчиков или десять тысяч — работает одинаково.

Шаг 1: Подготовка — структура воронки и тексты писем

Воронка — это последовательность. Каждое письмо ведёт к следующему шагу.

Классическая структура (7 писем):

ДеньПисьмоЗадача
0ПриветствиеПознакомиться, дать обещанный бонус
1ИсторияРассказать свою историю, вызвать доверие
3ПользаДать ценный контент бесплатно
5КейсПоказать результат клиента
7ОфферМягкое предложение купить
10ВозраженияЗакрыть сомнения и страхи
14ДедлайнОграничение по времени или количеству
Совет

Начните с 5 писем. Идеальная воронка — не та, где 20 писем. А та, где каждое письмо делает одно действие. Остальные добавите после запуска.

Формула каждого письма: тема (5–7 слов) + зацепка + одна мысль + призыв к действию.

Шаг 2: Создание конструктора воронки в Lovable

  1. Откройте lovable.dev
  2. Создайте новый проект → "Blank project"
  3. Вставьте промпт:
Создай конструктор email-воронки на React.

Главная страница — визуальный конструктор цепочки писем:

1. Визуальная цепочка (вертикальная):
   - Блоки писем, соединённые стрелками (сверху вниз)
   - Каждый блок: номер, название письма, день отправки
   - Цвет блока: зелёный (активно), серый (черновик)
   - Кнопки: "Добавить письмо" между блоками, drag & drop
   - Кнопка "Добавить письмо в конец цепочки"

2. Панель редактирования письма (справа при клике на блок):
   - Поле "Тема письма" (текст)
   - Поле "День отправки" (число, от подписки)
   - Визуальный редактор текста письма (bold, italic, ссылки)
   - Кнопка "Добавить кнопку CTA" (текст + URL)
   - Превью письма (как будет выглядеть)

3. Триггеры:
   - Старт цепочки: "После подписки", "После покупки",
     "После регистрации" (выпадающий список)
   - Интервалы между письмами: задаются в каждом блоке

4. Дизайн: светлая тема, синий акцент, Tailwind CSS, shadcn/ui.
   Адаптивный макет. Боковая панель для навигации.
Важно

Замените [ПЛЕЙСХОЛДЕРЫ] на свои данные перед отправкой. Lovable создаст рабочий интерфейс за 2–3 минуты.

Ожидаемый результат: визуальный конструктор цепочки писем с редактором и превью.

Шаг 3: Редактор шаблонов и превью

Добавим полноценный редактор каждого письма с предпросмотром.

Добавь редактор шаблонов писем:

1. Шаблоны — готовые макеты писем:
   - "Приветствие" — заголовок + текст + кнопка
   - "История" — фото + длинный текст
   - "Кейс" — цитата клиента + результат + CTA
   - "Оффер" — заголовок + буллиты + цена + кнопка
   - "Дедлайн" — таймер + текст + кнопка
   При создании письма — выбор шаблона.

2. Редактор письма:
   - Тема (обязательно, до 60 символов)
   - Прехедер (текст после темы, до 100 символов)
   - Текст письма: заголовок H1, абзацы, жирный, курсив
   - Вставка кнопки: текст + URL + цвет
   - Вставка изображения: URL картинки
   - Вставка разделителя

3. Превью (переключатель "Десктоп / Мобильный"):
   - Рамка 600px для десктопа, 375px для мобильного
   - Показывает тему, прехедер и тело письма
   - Кнопка "Отправить тестовое" (сохраняет в лог)

4. Переменные в тексте:
   - {{имя}} — подставляет имя подписчика
   - {{дата}} — дата отправки
   - {{ссылка_отписки}} — ссылка для отписки
Аналогия с конвейером

Шаблоны — как формочки для печенья. Каждое письмо разное по начинке, но одинаково аккуратное по форме. Не нужно каждый раз верстать с нуля.

Про переменные

Переменные вроде персонализируют письмо. «Привет, Анна!» работает лучше, чем «Привет!». Конверсия выше на 20–30%.

Ожидаемый результат: редактор с шаблонами, переменными и превью на десктопе и мобильном.

Шаг 4: Управление подписчиками и аналитика

Нужно видеть, кто подписан, на каком шаге воронки и как реагирует.

Добавь управление подписчиками и аналитику:

1. Страница "Подписчики" — таблица:
   - Имя, email, дата подписки, текущий шаг воронки
   - Статус: "Активный", "Завершил", "Отписался"
   - Фильтры: по статусу, по шагу воронки, по дате
   - Поиск по имени и email
   - Кнопка "Добавить подписчика" (вручную)
   - Импорт CSV (email, имя)

2. Страница "Аналитика" — дашборд:
   - Воронка: подписались → открыли → кликнули → купили
     (горизонтальные полоски с процентами)
   - Таблица по каждому письму: отправлено, открытий (%),
     кликов (%), отписок (%)
   - График: подписки за 30 дней (линейный)
   - Лучшее письмо (по открытиям) и худшее (по отпискам)

3. Страница "Лог отправок" — что, кому, когда:
   - Каждая запись: email, письмо, дата, статус (доставлено,
     открыто, клик, отписка)
   - Фильтр по письму и по статусу

Данные в localStorage. Демо-данные для 20 подписчиков.
Совет

Следите за процентом открытий. Если ниже 20% — переписывайте тему письма. Тема — это 80% успеха. Плохая тема — письмо не откроют.

Ожидаемый результат: управление подписчиками, лог отправок и аналитика по каждому письму.

Шаг 5: Дизайн и настройка внешнего вида

Воронка должна выглядеть профессионально.

Улучши дизайн:

1. Конструктор: плавные анимации, пунктирные стрелки между
   письмами, мини-превью при наведении. Цвета: зелёный
   (отправлено), синий (запланировано), серый (черновик).

2. Редактор: белый фон, тонкие рамки, тулбар форматирования
   сверху. Превью обновляется в реальном времени.

3. Аналитика: воронка с градиентом, цветные бейджи,
   графики с подсказками (Recharts).

4. Общее: боковая панель ("Цепочка", "Подписчики",
   "Аналитика", "Лог"). На мобильном — бургер-меню.
   Шрифт Inter, акцент — [ВАШ_ЦВЕТ].

Ожидаемый результат: аккуратный и профессиональный интерфейс.

Шаг 6: Деплой

Публикуем воронку в интернете.

  1. В Lovable нажмите "Deploy" вверху справа
  2. Выберите "Deploy to Vercel"
  3. Подключите GitHub аккаунт (если нет — создайте бесплатно)
  4. Получите ссылку: https://ваша-воронка.vercel.app

Хотите свой домен? Купите на reg.ru и подключите в настройках Vercel.

Про реальную отправку писем

Этот инструмент — конструктор и аналитика воронки. Для реальной отправки подключите Resend, SendPulse или Unisender через API. Попросите Lovable добавить интеграцию.

Ожидаемый результат: воронка опубликована и готова к подключению email-сервиса.

Что дальше

Базовая воронка работает. Вот как её развить.

1. A/B тесты тем. Два варианта темы — выбирайте тот, что открывают чаще.

2. Сегментация. Разные воронки для разных аудиторий: новички и продвинутые.

3. Условные ветки. Открыл письмо — одна ветка. Не открыл — другая.

4. Интеграция с CRM. Подписчик купил — автоматически попадает в базу клиентов.

Результат

Автоматическая воронка прогревает подписчиков без вашего участия. Каждый новый подписчик получает серию из 5–7 писем. Вы видите аналитику: кто открыл, кто кликнул, кто купил. Продажи растут — а ручной работы нет.

✏️Упражнение: Создайте свою воронку из 5 писем

Напишите тему и главную мысль для каждого из 5 писем вашей воронки.

Пример для фитнес-тренера (продукт: курс «Тело за 90 дней»):

Письмо 1 (День 0): Тема: «Ваш план тренировок — внутри» Мысль: отдать бесплатный план на 7 дней. Представиться.

Письмо 2 (День 1): Тема: «Почему я бросил офис ради фитнеса» Мысль: личная история. Я был таким же. Показать путь.

Письмо 3 (День 3): Тема: «3 упражнения, которые сжигают жир быстрее бега» Мысль: полезный контент. Дать ценность бесплатно.

Письмо 4 (День 5): Тема: «Марина похудела на 12 кг за 2 месяца» Мысль: кейс ученицы. Фото до/после. Цитата.

Письмо 5 (День 7): Тема: «Набор закрыт через 48 часов» Мысль: оффер курса + скидка + ограничение по времени.

Готово! Автоматическая email-воронка создана. Подписчики прогреваются на автомате. Каждое письмо ведёт к следующему шагу. Вы видите, кто открыл, кто кликнул и кто готов купить. Продажи идут — а вы спите.
