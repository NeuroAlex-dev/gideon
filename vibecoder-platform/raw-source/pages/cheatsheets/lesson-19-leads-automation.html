Заявки с сайта → Telegram + Google Sheets

Полный гайд: форма захвата → уведомление в бот → строка в таблице → деплой на Vercel

Что строим

Схема потока данных:

📝 Форма на сайте → ⚡ API-обработчик → ✈️ Telegram + 📊 Google Sheets

  • Форма на сайте — многошаговая анкета с прогресс-баром
  • API-обработчик — сервер принимает данные формы
  • Telegram — моментальное уведомление в бот
  • Google Sheets — строка в таблице через Apps Script

Пошаговый алгоритм

12 шагов от нуля до работающей автоматизации.

  1. Открыть проект в VS Code. Команда: code . в терминале или File → Open Folder. Запустить проект локально.

  2. Дать задачу Claude Code [Telegram] — «Оцени структуру сайта и сделай выгрузку лидов с формы захвата сразу в Telegram. Скажи, что мне нужно для этого.»

  3. Получить токен бота [Telegram] — Найти своего бота через @BotFather → скопировать токен. Нет бота — создать: /newbot

  4. Получить свой Chat ID [Telegram] — Написать боту @userinfobot или переслать ему любое сообщение → в ответе будет ваш ID.

  5. Передать токен и Chat ID в Claude — Claude автоматически настроит отправку. Ответит на вопрос: хочу ли я перезаписать файлы — ответить «Да».

  6. Протестировать Telegram локально — Открыть локальный сайт → заполнить форму → проверить, что в боте пришло уведомление с данными.

  7. Получить инструкцию по Google Sheets [Google] — Спросить Claude: «Как подключить Google Sheets? Дай инструкцию.» — он выдаст код для Apps Script.

  8. Создать Apps Script в Google Sheets [Google] — В таблице: Расширения → Apps Script → вставить код от Claude → Развертывание → Веб-приложение → от вашего имени, доступ: все → получить URL.

  9. Передать URL в проект [Код] — Скопировать URL из Apps Script → вставить в Claude Code → он пропишет в нужный файл. Протестировать.

  10. Финальный тест локально — Заполнить форму → проверить Telegram (пришло уведомление) и Google Sheets (появилась строка).

  11. Запушить на GitHub → деплой на Vercel [Деплой] — Промпт Claude: «Запушь всё на GitHub для автоматического деплоя» — Vercel подхватит автоматически.

  12. Добавить переменные в Vercel [Деплой] — В Vercel: Settings → Environment Variables → добавить токен бота, Chat ID и URL таблицы → Redeploy.

Ключевые промпты

Промпт 1 — Подключить Telegram

Оцени структуру сайта и сделай выгрузку лидов с формы захвата сразу в Telegram.

Мой токен бота: [ВСТАВИТЬ ТОКЕН]
Мой Chat ID: [ВСТАВИТЬ CHAT ID]

Пропиши всё так, чтобы при отправке формы мне в Telegram
моментально приходили данные: имя, контакт и все заполненные поля.
Перезапиши все нужные файлы.

Промпт 2 — Улучшить форму (многошаговая анкета)

Поправь форму заявки — добавь поля:
1. Имя и Telegram для связи
2. Какой у человека бизнес / чем занимается
3. Опыт в ИИ — сделай кликабельными кнопками (Новичок / Базовый / Продвинутый / Эксперт)
4. Цель — что хочет получить (можно мультиселект)

Форма должна быть лёгкой: не всё сразу, а шаг за шагом.
Сделай слайдер-анкету с прогресс-баром — каждый вопрос на отдельном шаге.
В конце — кнопка «Отправить» и экран «Заявка принята».

Промпт 3 — Деплой на GitHub / Vercel

Запушь всё на GitHub для автоматического деплоя на Vercel.

Список переменных окружения, которые нужно добавить в Vercel:
- TELEGRAM_BOT_TOKEN
- TELEGRAM_CHAT_ID
- GOOGLE_SHEET_URL

Напомни мне, куда их добавить в интерфейсе Vercel.

При изменении кода скрипта

Нужно создавать новое развертывание — старый URL не обновляется автоматически.

Деплой на Vercel

  1. Claude пушит проект на GitHub
  2. Vercel автоматически деплоит
  3. Settings → Environment Variables
  4. Добавить токен, Chat ID, URL таблицы
  5. Redeploy — сайт в проде
Переменные окружения

Токены и URL хранятся только в Vercel, не в коде. Они не попадают на GitHub и не видны пользователям.

Лайфхаки

  • 🗣️ Голос вместо текста — задачи в Claude удобнее диктовать голосом через Super Whisper или Handy — скорость работы растёт в разы

  • 🔄 Используй старого бота — не обязательно создавать нового — подойдёт любой старый Telegram-бот. Главное — взять его токен в @BotFather

  • ⚠️ Advanced → Proceed — при авторизации Apps Script Google предупреждает об «опасности» — нажми Advanced → перейти. Это безопасно для своих скриптов

  • 🔐 Переменные — только в Vercel — никогда не хардкодь токены в коде. Храни их в Vercel → Environment Variables → они скрыты и не попадут в GitHub

  • 🧪 Сначала тест локально — всегда проверяй форму на localhost перед деплоем. Telegram должен отвечать, Sheets — принимать данные

  • 🐛 Не работает на проде? — открой консоль браузера (F12) и посмотри ошибку. Часто проблема — переменная не добавлена в Vercel
