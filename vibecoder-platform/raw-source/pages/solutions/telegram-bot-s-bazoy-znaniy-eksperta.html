Telegram-бот с базой знаний эксперта

Вы отвечаете на одни и те же вопросы каждый день? Создайте бота, который будет делать это за вас, используя вашу базу знаний и искусственный интеллект. Как личный помощник, который изучил все ваши материалы.

Что понадобится

  • Claude Code — терминал в VS Code для создания проекта
  • Telegram аккаунт — для создания бота через BotFather
  • API ключ — OpenRouter или Google AI для подключения LLM
  • Node.js — runtime для запуска приложения
  • Git — для управления кодом и деплоя

Шаг 1: Создание бота в Telegram

Каждый бот в Telegram живёт в личном чате с BotFather. Это как регистрация нового сотрудника.

Что делать:

  1. Откройте Telegram и найдите @BotFather
  2. Нажмите /start, затем /newbot
  3. Придумайте имя бота (например, Expert FAQ Bot)
  4. Придумайте username (должен заканчиваться на _bot, например, my_expert_bot)
  5. Скопируйте полученный API Token — это ключ доступа к вашему боту
Сохраните токен безопасно

Токен — это пароль вашего бота. Никому его не показывайте. Сохраните в защищённом месте (например, в файле .env).

Ожидаемый результат: Вы получите сообщение вроде Use this token to access the HTTP API: 123456789:ABCdefGHIjklmnoPQRstuvWXYZ

Шаг 2: Подготовка базы знаний

База знаний — это ваш архив ответов на частые вопросы. Структурируем её так, чтобы бот её понимал.

Формат файла knowledge_base.txt:

ВОПРОС: Как начать работу с сервисом?
ОТВЕТ: Зарегистрируйтесь в личном кабинете, подтвердите email, и добавьте первый проект.

ВОПРОС: Какая цена подписки?
ОТВЕТ: Бесплатный план — 2 проекта. Pro — 499р/мес на 10 проектов.

Или более сложный формат (JSON):

[
  {"question": "...", "answer": "..."},
  {"question": "...", "answer": "..."}
]
База знаний — это учебник

Если бот — это ученик, то база знаний — его учебник. Чем лучше учебник, тем умнее ученик. Пишите простые, понятные ответы.

Создайте файл knowledge_base.txt с вашими вопросами и ответами на основные темы вашей работы.

Шаг 3: Создание проекта через Claude Code

Откройте Claude Code и введите этот промпт:

Создай Node.js проект для Telegram бота на базе библиотеки telegraf.
Структура:
- bot.js (основной файл)
- knowledge_base.txt (база знаний)
- .env (переменные окружения)
- package.json

Установи зависимости: telegraf, dotenv

В bot.js создай:
1. Подключение Telegram API с токеном из .env
2. Обработчик команды /start с приветствием
3. Обработчик текстовых сообщений (пока просто логирует)
4. Слушатель на порту 3000

Выведи код с объяснением каждого блока.
Копируй-вставляй

Claude Code покажет готовый код. Скопируй его в свой редактор и сразу запусти в терминале.

Ожидаемый результат: Работающий бот, который говорит "Hello" на любое сообщение.

Шаг 4: Подключение AI

Теперь научим бота думать. Подключим LLM (большую языковую модель).

Обнови bot.js:
1. Установи axios для API запросов (npm install axios)
2. Добавь переменную OPENROUTER_KEY из .env
3. При получении сообщения от юзера:
   - Отправь его в OpenRouter API (используй модель claude-3-haiku)
   - Отправь пользователю ответ от нейросети

Функция должна быть асинхронной. Обработай ошибки API.

Пример: юзер пишет "Привет" → бот отправляет в API → API возвращает ответ → бот пишет ответ юзеру.
API ключ — это деньги

Каждый запрос стоит денег (даже копейки). Установи лимит запросов, чтобы не разориться. Например, блокируй юзеров с 100+ запросами в день.

Шаг 5: Загрузка базы знаний

Теперь бот будет отвечать не просто умно, а умно по вашему сценарию.

Обнови bot.js:
1. При запуске приложения загрузи knowledge_base.txt в переменную
2. Распарси файл: каждый Q&A становится объектом
3. Перед запросом в AI:
   - Найди в базе знаний похожий вопрос (используй простой поиск по ключевым словам)
   - Если нашёл → отправь в промпт AI контекст: "Вот ответ из базы знаний: ..."
   - Если не нашёл → просто отправь вопрос пользователя

Это называется RAG (Retrieval-Augmented Generation).

Функция searchKnowledgeBase(query) должна вернуть релевантный ответ или null.
RAG — это советчик с книгой

Вместо того чтобы гадать, нейросеть сначала смотрит в вашу книгу (базу знаний), потом отвечает. Ответ более точный и соответствует именно вашему стилю.

Шаг 6: Тестирование и доработка

Протестируй своего бота в Telegram. Проверь, что он корректно отвечает на вопросы из базы знаний.

Добавь в bot.js:
1. Команду /help — список команд
2. Команду /reset — очистить историю диалога
3. Команду /about — информация о боте
4. Обработчик ошибок с сообщением "Извините, произошла ошибка. Попробуйте позже"
5. Логирование всех запросов в консоль (для отладки)

Протестируй:
- Что бот отвечает на вопросы?
- Что происходит при одновременных запросах 5 юзеров?
- Как бот реагирует на длинные сообщения (>4096 символов)?

Если долгий ответ → разбей его на несколько сообщений.
Тестируй сам себе

Создай второй Telegram аккаунт и общайся со своим ботом. Представь, что ты клиент, и проверь все сценарии.

Шаг 7: Деплой

Сейчас бот работает только на твоём компьютере. Нужно запустить его 24/7 на сервере.

Выбери платформу:

ВАРИАНТ 1: Railway (проще, бесплатно до 5$/мес)
1. Зарегистрируйся на railway.app
2. Подключи свой GitHub репозиторий
3. Railway автоматически деплоит app.js
4. Переменные окружения добавь в Dashboard
5. Бот будет работать постоянно

ВАРИАНТ 2: VPS (дешевле, но нужна настройка)
1. Возьми сервер на Hetzner или DigitalOcean
2. Установи Node.js
3. Загрузи код через git
4. Запусти "npm start" в фоновом процессе (используй pm2)
5. Настрой автозапуск при перезагрузке сервера

Какой вариант выбрать? Если ты новичок → Railway. Если много пользователей → VPS.
Вебхуки vs Polling

Railway лучше работает с вебхуками (бот сам получает обновления). Настрой в telegraf режим вебхука, не polling.

Упражнение: Создай свою базу знаний

Пример для консультанта по дизайну:

Q: Как долго разрабатывается сайт? A: Обычный сайт на WordPress — 2-3 недели. Сложный с кастомной функциональностью — 1-2 месяца. Зависит от требований.

Q: Какой бюджет нужен? A: Простой сайт от 30 тыс. Средний с интеграциями от 100 тыс. Все индивидуально, обсудим на встрече.

Q: Делаете ли вы мобильный дизайн? A: Да, все наши сайты адаптивные. Сначала проектируем для мобильных, потом расширяем.

Q: Помогаете с СЕО? A: Базовая оптимизация включена. Глубокую стратегию обсудим отдельно.

Q: Какие способы оплаты? A: Оплата частями: 30% аванс, 30% в процессе, 40% при запуске.

Что дальше?

Базовый бот готов. Вот что можно добавить:

  • Аналитика — отслеживай, на какие вопросы чаще спрашивают. Добавь счётчик в БД.
  • Админ-панель — редактируй базу знаний без перезагрузки бота (добавь /edit команду).
  • Многоязычность — если клиенты пишут на разных языках, подключи автопереводчик.
  • История диалогов — сохраняй беседы в БД (PostgreSQL) для анализа.
  • Платные фичи — ограничь бесплатных пользователей, чтобы монетизировать.
  • Интеграция с CRM — автоматически создавай лиды в твоей системе.
