Умная CRM с предсказанием продаж

Менеджеры тратят время на «мёртвые» сделки. А горячие клиенты уходят к конкурентам. Потому что нет системы, которая подскажет — кто готов купить прямо сейчас.

Сейчас мы построим CRM, которая сама анализирует сделки. ИИ покажет: кого дожать, кого не трогать, а кто купит сам.

Что понадобится

  • Lovable — AI-конструктор для создания интерфейса CRM
  • Google AI Studio (опционально) — для продвинутой ИИ-аналитики скоринга сделок
Аналогия

Обычная CRM — это записная книжка. Ты записываешь контакты, но сам решаешь, кому звонить. Умная CRM — это навигатор для продавца. Она говорит: «Поверни к этому клиенту — он готов купить. А вон тот — подождёт».

Шаг 1: Подготовка

Перед созданием определите:

  1. Этапы воронки — например: Новый → Квалификация → Предложение → Переговоры → Сделка
  2. Данные для скоринга — что влияет на вероятность покупки:
    • Размер сделки
    • Количество касаний (звонков, писем)
    • Время на этапе
    • Источник лида
  3. Критерии «горячего» клиента — какой балл = готов купить
Совет

Начните с 5-7 параметров скоринга. Больше — запутаетесь. Меньше — неточный прогноз.

Шаг 2: Создание CRM в Lovable

  1. Откройте Lovable
  2. Создайте новый проект
  3. Вставьте промпт:
Создай умную CRM-систему для отдела продаж на React с Tailwind CSS.

Главный экран — Kanban-доска с колонками этапов воронки:
«Новый», «Квалификация», «Предложение», «Переговоры», «Сделка закрыта».

Каждая карточка сделки показывает:
— Имя клиента и компанию
— Сумму сделки
— ИИ-скоринг от 0 до 100 (цветовой индикатор: зелёный >70, жёлтый 40-70, красный <40)
— Количество дней на текущем этапе
— Иконки последних действий (звонок, письмо, встреча)

Перетаскивание карточек между колонками (drag & drop).

Верхняя панель: общая сумма воронки, количество сделок, средний скоринг, прогноз закрытия на месяц.

Боковая панель при клике на сделку: полная информация, история касаний, рекомендации ИИ (что делать дальше).

Дизайн: профессиональный, тёмная тема, акцентный синий. Шрифт Inter.

Ожидаемый результат: Kanban-доска с карточками сделок, скорингом и аналитикой.

Шаг 3: ИИ-скоринг и рекомендации

Добавьте интеллектуальный блок анализа:

Добавь систему ИИ-скоринга сделок:

1. Формула скоринга на основе параметров:
— Сумма сделки (больше = выше приоритет)
— Дней на этапе (больше = тревога, снижение скора)
— Количество касаний за неделю (больше = выше вовлечённость)
— Источник лида (рекомендация = +20, холодный = +5)
— Открытие последнего письма (да = +15)

2. Блок рекомендаций для каждой сделки:
— Если скор > 70: «Горячий лид — предложите встречу или демо»
— Если скор 40-70: «Нужен прогрев — отправьте кейс или отзыв»
— Если скор < 40: «Заморожен — переведите в nurturing или закройте»
— Если дней на этапе > 14: «Внимание: сделка застряла»

3. Цветовая индикация на карточках: пульсирующая зелёная точка для горячих.
Продвинутый вариант

Если хотите настоящий ИИ-скоринг — подключите Google AI Studio. Загрузите историю сделок, и модель найдёт паттерны, которые формула не увидит. Но для старта формула работает отлично.

Шаг 4: Аналитика и прогнозы

Добавь страницу «Аналитика» с графиками:

1. Воронка продаж — горизонтальная диаграмма: сколько сделок на каждом этапе и их сумма.

2. Прогноз выручки на месяц:
— Считай: сумма сделки × вероятность закрытия (скоринг/100) для каждой сделки
— Покажи итог: «Ожидаемая выручка: 2.4М₽»
— График по неделям: план vs факт

3. Активность команды:
— Таблица менеджеров: имя, количество сделок, сумма воронки, средний скоринг
— Топ-3 менеджера с лучшей конверсией

4. Проблемные зоны:
— Список сделок, которые застряли > 7 дней
— Сделки с падающим скорингом (тренд вниз)
Важно

Прогноз — не гарантия. Это инструмент для приоритизации. Менеджер всё равно принимает решение сам.

Шаг 5: История касаний и задачи

Добавь в боковую панель сделки:

1. Таймлайн касаний — вертикальная лента:
— Иконка типа (📞 звонок, ✉️ письмо, 🤝 встреча, 💬 чат)
— Дата и краткое описание
— Кнопка «+ Добавить касание» с быстрой формой

2. Блок задач по сделке:
— Список задач с дедлайнами
— Автозадачи от ИИ: «Скоринг упал — позвоните клиенту»
— Чекбоксы выполнения

3. Заметки — текстовое поле для менеджера.

4. Кнопка «Следующее действие» — ИИ предлагает конкретный шаг на основе истории.

Ожидаемый результат: полная карточка сделки с историей, задачами и рекомендациями ИИ.

Шаг 6: Деплой (публикация)

Финальный штрих — приведите дизайн в порядок:

Проверь весь дизайн CRM:
— Единый стиль карточек и отступов
— Адаптивность для планшета (менеджеры часто работают с iPad)
— Плавные анимации при перетаскивании карточек
— Пустые состояния: «Нет сделок на этом этапе — создайте первую»

Два варианта публикации:

  1. Бесплатно — нажмите Share → опубликуйте на поддомене Lovable
  2. Свой домен — подключите домен в настройках проекта (например, crm.вашбизнес.ru)
Готово!

У вас есть умная CRM с предсказанием продаж. Менеджеры видят приоритеты, руководитель — прогноз выручки. Больше никаких «забытых» сделок.

Что дальше

  • Подключите базу данных (Supabase) для хранения реальных сделок
  • Добавьте импорт из Excel — загрузить существующую базу клиентов
  • Настройте уведомления в Telegram — «Сделка X застряла 7 дней»
  • Интегрируйте с почтой — автоматический учёт писем
  • Добавьте роли: менеджер видит свои сделки, руководитель — все
  • Подключите Google AI Studio для продвинутого скоринга на основе истории

✏️Упражнение: Практика: первые сделки

Добавьте 5 реальных сделок из вашей воронки. Для каждой укажите: клиент, сумма, этап, последнее касание. Посмотрите, какой скоринг присвоит система.

Пример для digital-агентства:

  1. ООО «Рост» — 450К₽, этап «Предложение», звонок вчера → Скоринг: 78 (горячий)
  2. ИП Смирнова — 80К₽, этап «Квалификация», письмо 5 дней назад → Скоринг: 45 (прогрев)
  3. «ТехноПарк» — 1.2М₽, этап «Переговоры», встреча 2 дня назад → Скоринг: 85 (горячий)
  4. «СтартАп» — 150К₽, этап «Новый», холодный лид → Скоринг: 25 (nurturing)
  5. «МедиаГрупп» — 300К₽, этап «Предложение», без касаний 12 дней → Скоринг: 32 (застрял)

Рекомендация системы: сначала звоните «ТехноПарк» и «ООО Рост».
