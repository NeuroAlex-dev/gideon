Лендинг с A/B-тестированием

Какой заголовок сработает? Синяя кнопка или зелёная? Хватит гадать. Создадим лендинг, который сам тестирует два варианта и показывает, какой конвертит лучше. Маркетинг на данных, а не на интуиции.

Что понадобится

  • Lovable — AI-помощник, который создаёт веб-приложения за вас
  • Два варианта оффера — заголовки, тексты, картинки для теста
  • Цель конверсии — что считать успехом (клик, заявка, покупка)
  • Домен (по желанию) — свой адрес вместо бесплатного .vercel.app

Код писать не нужно. Lovable сделает всё по описанию.

Аналогия

A/B-тестирование — это научный эксперимент для маркетинга. Вы не спорите, какой заголовок лучше. Вы показываете оба варианта реальным людям и считаете результат. Гипотеза, эксперимент, данные, решение. Как в лаборатории, только вместо пробирок — лендинги.

Шаг 1: Подготовка — сформулируйте гипотезу

Прежде чем открыть Lovable, определите, что именно тестируете.

Выпишите в блокнот:

  1. Гипотеза — что хотите проверить (заголовок, CTA, цвет, картинка)
  2. Вариант A — текущий вариант (контроль)
  3. Вариант B — новый вариант (тест)
  4. Метрика успеха — что измеряете (клики, заявки, конверсия)

Примеры гипотез:

Что тестируемВариант AВариант B
Заголовок«Научим зарабатывать на фрилансе»«Первый заказ за 7 дней»
CTA-кнопка«Оставить заявку»«Получить бесплатную консультацию»
Цвет кнопкиСинийОранжевый
Совет

Тестируйте одну вещь за раз. Если поменяете заголовок и кнопку одновременно — не поймёте, что повлияло на результат.

Шаг 2: Создание лендинга в Lovable

Откройте lovable.dev, создайте новый проект и вставьте этот промпт:

Создай лендинг с встроенной системой A/B-тестирования на React.

Требования к лендингу:
1. Hero-секция: заголовок, подзаголовок, CTA-кнопка
2. Секция преимуществ: 3-4 карточки с иконками
3. Секция отзывов: 2-3 карточки с цитатами
4. Форма заявки: имя, телефон/Telegram, кнопка отправки
5. Футер с контактами

Требования к A/B-тестированию:
1. При загрузке случайно выбирается вариант A или B (50/50)
2. Выбранный вариант сохраняется в localStorage
3. Вариант A и B отличаются: заголовком, текстом и цветом CTA-кнопки
4. Отслеживаются: просмотры, клики на CTA, отправки формы
5. Все события сохраняются в localStorage

Дизайн: современный, минималистичный, Tailwind CSS, shadcn/ui,
адаптивный (мобильный + десктоп)

Вариант A:
- Заголовок: "[ЗАГОЛОВОК_A]"
- CTA: "[ТЕКСТ_КНОПКИ_A]"
- Цвет кнопки: [ЦВЕТ_A]

Вариант B:
- Заголовок: "[ЗАГОЛОВОК_B]"
- CTA: "[ТЕКСТ_КНОПКИ_B]"
- Цвет кнопки: [ЦВЕТ_B]

Замените плейсхолдеры на свои варианты из Шага 1. Нажмите Enter.

Важно

Не закрывайте вкладку во время генерации. Если закроете — процесс прервётся и придётся начинать заново.

Ожидаемый результат: Рабочий лендинг, который случайно показывает вариант A или B. Каждый визит фиксируется.

Шаг 3: Дашборд результатов

Без дашборда вы не увидите, кто побеждает. Добавим панель аналитики.

Добавь дашборд результатов A/B-теста:

1. Скрытая страница /admin (доступ по URL)
2. Таблица сравнения: просмотры, клики на CTA, конверсия CTA,
   заявки, конверсия формы — для каждого варианта + разница в %
3. Индикатор победителя: зелёная плашка "Лидирует вариант [A/B]"
4. Столбчатая диаграмма: конверсии A vs B
5. Достоверность данных:
   - Менее 100 просмотров: "Мало данных для выводов"
   - 100-500: "Предварительные результаты"
   - 500+: "Достоверные результаты"
6. Кнопки "Сбросить статистику" и "Экспорт в CSV"
Зачем статистическая достоверность

Если 3 из 5 человек кликнули на вариант B — это ничего не значит. Выборка слишком мала. Нужно минимум 100–200 просмотров на вариант.

Ожидаемый результат: Страница /admin с таблицей, графиками и индикатором победителя.

Шаг 4: Редактор вариантов

Чтобы запускать новые тесты, не нужно переписывать код. Добавим редактор.

Добавь редактор вариантов на страницу /admin:

1. Для каждого варианта (A и B) — форма: заголовок,
   подзаголовок, текст CTA, цвет CTA (color picker)
2. Мини-превью hero-секции рядом с формой
3. Кнопки: "Сохранить" и "Сбросить статистику + новый тест"
4. Переключатель распределения трафика: 50/50, 70/30, 90/10
5. Возможность отключить тест (показывать только один вариант)
6. Все настройки в localStorage
Аналогия с пультом управления

Редактор вариантов — как пульт от телевизора. Не нужно лезть внутрь, чтобы переключить канал. Нажали кнопку — поменяли заголовок. Нажали другую — запустили новый тест.

Ожидаемый результат: Можно менять заголовки, кнопки, цвета без кода. Новые тесты запускаются в два клика.

Шаг 5: Улучшение лендинга

Базовый лендинг работает. Сделаем его убедительнее.

Улучши лендинг:

1. Hero: анимация fade-in, счётчик "Уже [N]+ клиентов",
   подпись "Бесплатно. Без обязательств."
2. Преимущества: 4 карточки с иконками Lucide, hover-эффект,
   тексты: [ПРЕИМУЩЕСТВО_1], [ПРЕИМУЩЕСТВО_2],
   [ПРЕИМУЩЕСТВО_3], [ПРЕИМУЩЕСТВО_4]
3. Отзывы: карточки с фото, именем, текстом, звёздочки рейтинга
4. Форма: имя, телефон/Telegram, email (необязательный),
   цвет кнопки = цвет CTA варианта, после отправки: "Спасибо!"
5. Мобильная версия: кнопки и поля на всю ширину экрана
Совет про отзывы

Отзывы конвертят лучше преимуществ. Если нет реальных — напишите 2–3 по образцу. Когда появятся настоящие — замените.

Ожидаемый результат: Лендинг с анимациями, отзывами и формой. Выглядит профессионально, готов принимать трафик.

Шаг 6: Дизайн и деплой

Финальная полировка и публикация в интернете.

Финальные улучшения: белый фон (#ffffff), тёмный текст (#1a1a2e),
акцент [ЦВЕТ_БРЕНДА], крупные заголовки, плавная прокрутка к форме,
мета-теги для соцсетей (og:title, og:description, og:image)

Деплой:

  1. В Lovable нажмите "Deploy" (вверху справа)
  2. Выберите "Deploy to Vercel"
  3. Подключите GitHub аккаунт (если нет — создайте бесплатно)
  4. Вы получите ссылку вроде: https://my-landing-abc.vercel.app
  5. По желанию — подключите свой домен в настройках Vercel
Про localStorage

Статистика хранится в браузере посетителя. Для тестов с малым трафиком (до 500 визитов) этого достаточно. Для серьёзных тестов подключите Supabase через Lovable — данные будут на сервере.

Как использовать:

  • Запустите рекламу на лендинг — трафик делится 50/50
  • Через 2–3 дня откройте /admin — посмотрите результаты
  • Когда набралось 200+ просмотров — выбирайте победителя
  • Запускайте новый тест с новой гипотезой

Что дальше

Базовый A/B-тест работает. Вот как развивать систему:

1. Multivariate-тесты — тестируйте четыре варианта одновременно. Больше гипотез за тот же период.

2. Серверная аналитика — подключите Supabase. Данные не потеряются при очистке кэша.

3. Тепловая карта кликов — покажет, куда нажимают. Может, CTA-кнопку просто не замечают.

4. Автовыбор победителя — система переключит весь трафик на лучший вариант автоматически.

5. Персонализация — разные варианты для мобильного и десктопа. Поведение отличается.

Итог

Вы создали лендинг с A/B-тестированием за 2–3 часа. Больше не нужно гадать, какой заголовок лучше. Система покажет победителя. Каждый тест делает ваш маркетинг эффективнее.

✏️Упражнение: Запустите свой первый A/B-тест

Придумайте два заголовка для продукта или услуги. Создайте лендинг с обоими вариантами. Отправьте ссылку 10 друзьям и проверьте /admin через день.

  1. Заголовок A — опишите продукт: «Курсы английского для взрослых»
  2. Заголовок B — опишите результат: «Заговорите на английском за 3 месяца»
  3. Создайте лендинг в Lovable с обоими вариантами
  4. Разошлите ссылку в чаты, соцсети, мессенджеры
  5. Через 24 часа зайдите на /admin и посмотрите конверсии

Обычно заголовок про результат побеждает заголовок про продукт. Но проверьте на своей аудитории — сюрпризы случаются.

Готово! Ваш лендинг с A/B-тестированием работает. Трафик делится автоматически, конверсии считаются, дашборд показывает победителя. Тестируйте гипотезы — пусть данные решают.
