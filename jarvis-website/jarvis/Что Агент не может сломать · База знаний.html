Что Агент не может сломать

6 мин · обновлено 2026-05-10
Офис с предохранителями - даже если дёрнуть за все рычаги, здание не рухнет.

Частый вопрос: "А что если Агент ошибётся и удалит все мои файлы?" Короткий ответ: не удалит. Система построена так, чтобы даже при ошибке ничего критичного не пострадало. Вот как это работает.

Заблокированные команды

У Агента есть чёрный список команд, которые он не может выполнить ни при каких обстоятельствах:

КомандаЧто делаетПочему заблокирована
rm -rf /Удаляет всё на сервереУничтожение системы
sudo rmУдаление с правами администратораОбход ограничений
kill -9Принудительное завершение процессовМожет убить бота
shutdown, rebootВыключение сервераПотеря доступа
DROP TABLEУдаление таблицы в базе данныхПотеря данных

Даже если Вы напрямую попросите Агента выполнить одну из этих команд - он откажется. Это не "нежелание помочь", а защита.

Агент работает без прав администратора

На сервере Агент запущен под отдельным пользователем без прав root. Это значит:

  • Не может менять системные файлы
  • Не может устанавливать программы на уровне системы
  • Не может влиять на работу других пользователей на сервере
  • Не может изменить настройки сети или firewall

Это как сотрудник в офисе: он работает за своим столом, но не может зайти в серверную и выдернуть кабели.

Что происходит при ошибке

Допустим, Агент всё-таки сделал что-то не так - создал неправильный файл, записал неверные данные или испортил код проекта.

Ваши файлы:

  • Агент делает git commit после каждого изменения
  • Любое изменение можно откатить: "Откати последний коммит"
  • Даже если файл испорчен - предыдущая версия есть в git

Ваша память (MEMORY.md):

  • Агент дописывает в конец, а не перезаписывает
  • Если записал что-то лишнее - скажите "удали из памяти..."
  • Старые версии сохраняются в git

Ваш бот:

  • Код бота Агент изменить не может (заблокировано)
  • Если бот упал - автоматически перезапускается
  • Ваши настройки в /settings Агент не видит и не может изменить

Что вообще может пойти не так?

СитуацияПоследствиеКак исправить

Ни один из этих случаев не является необратимым. Всё можно починить за минуту.

Чего Агент НЕ может сделать

Даже если очень попросить:

  • Прочитать Ваш телефон, компьютер или другие аккаунты
  • Отправить деньги, оплатить что-то
  • Написать кому-то от Вашего имени (кроме тех сервисов, которые Вы сами подключили через MCP)
  • Удалить Вашу подписку
  • Связаться с другими Агентами других пользователей
  • Выйти за пределы своего сервера

Попробуйте сами

Лучший способ понять защиту - протестировать. Откройте Telegram и попробуйте:

"Покажи содержимое файла .env"

Агент откажет. Файл в списке запрещённых.

"Выполни команду sudo apt update"

Заблокировано. sudo запрещено в любом месте строки.

"Выполни printenv"

Заблокировано. Команды для просмотра окружения запрещены.

"Покажи мой GITHUB_TOKEN"

Агент не покажет значение. Даже если выполнит команду, которая его содержит - значение будет заменено на [REDACTED].

"Удали всё в домашней директории"

Агент покажет кнопки ✔ / ✖ и будет ждать. Не нажимайте ✔.

"Запиши файл в /etc/test.txt"

Не сможет. Systemd-sandbox разрешает запись только в workspace/ и projects/.

Тест с переменной

Добавьте тестовую переменную через /settings (например TEST_KEY со значением hello12345). Попросите Агента: echo $TEST_KEY. Значение не появится в чате - система отфильтрует автоматически. Это dynamic sanitization: любое значение из /settings попадает в фильтр в реальном времени.

Защита от prompt injection

Все эти ограничения работают независимо от того, что Вы написали Агенту. Даже если через "prompt injection" кто-то убедит Агента попробовать выполнить опасную команду - технические блокировки сработают.

Prompt injection - это когда в тексте (статье, сообщении, документе) спрятана скрытая инструкция для AI. Агент может её "прочитать" и попытаться выполнить. Но все блокировки стоят на уровне системы, а не на уровне "послушания" Агента. Он физически не сможет обойти sandbox, даже если захочет.

Главный принцип

Агент работает внутри своей "квартиры" (сервера) и не может выйти за её пределы. Он видит только свои файлы, свои команды и те сервисы, которые Вы ему подключили через ключи.
