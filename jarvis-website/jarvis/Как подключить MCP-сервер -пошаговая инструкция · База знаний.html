Как подключить MCP-сервер -пошаговая инструкция

6 мин · обновлено 2026-05-10
Один JSON-файл, две строчки конфига -и Агент умеет новый фокус.

Вы решили подключить к Агенту новый инструмент через MCP. Вся конфигурация живёт в одном файле -~/.claude.json. Эта статья показывает как именно добавить MCP-сервер: вручную через файл, или попросив Агента.

Главное сообщение

Один файл -все подключения

Все MCP-серверы Агента записаны в ~/.claude.json → секция mcpServers. Добавить новый сервер = дописать блок в этот файл. Удалить = стереть блок. Никакого перезапуска -Агент подхватит изменения при следующем сообщении.

Способ 1: попросить Агента

Самый простой путь -скажите Агенту прямо в чате:

«Подключи MCP-сервер supadata, URL: https://mcp.supadata.dev/v1»

Агент сам прочитает ~/.claude.json, добавит новую запись в секцию mcpServers и запишет файл обратно. Вам не нужно открывать никакие файлы руками.

Это работает, потому что в правилах Агента (CLAUDE.md) есть инструкция:

Добавить MCP: прочитай ~/.claude.json через Read, добавь сервер в mcpServers, запиши ВСЁ обратно через Write.

Способ 2: через VS Code

Если у Вас подключён VS Code Tunnel (/connect), можно отредактировать файл вручную:

  1. Откройте VS Code
  2. Откройте файл ~/.claude.json
  3. Найдите секцию "mcpServers" (или создайте, если её нет)
  4. Добавьте блок нового сервера

Формат конфигурации

Файл ~/.claude.json выглядит так:

{
  "mcpServers": {
    "pencil": {
      "type": "stdio",
      "command": "/home/agent/.pencil/mcp/visual_studio_code/out/mcp-server-linux-x64",
      "args": ["--app", "visual_studio_code"],
      "env": {}
    },
    "supadata": {
      "type": "http",
      "url": "https://mcp.supadata.dev/v1"
    }
  }
}

Каждый MCP-сервер -это объект с именем (ключ) и настройками (значение).

Тип stdio -программа на сервере

Используется когда MCP-сервер -это программа, которая запускается на Вашем VPS.

{
  "type": "stdio",
  "command": "/путь/к/программе",
  "args": ["аргумент1", "аргумент2"],
  "env": {}
}
ПолеЧтоОбязательно
typeВсегда "stdio"Да
commandПуть к исполняемому файлу или команда (npx, node)Да
argsМассив аргументов командной строкиНет
envПеременные окружения для процесса (например, API-ключ)Нет

Пример с npx (Apify -автоматизация веб-задач):

{
  "apify": {
    "type": "stdio",
    "command": "npx",
    "args": ["@apify/mcp-server"],
    "env": {
      "APIFY_TOKEN": "ваш_токен"
    }
  }
}

При каждом запуске Claude Code CLI запускает эту программу и общается с ней через stdin/stdout. Когда сессия завершается -процесс останавливается.

Тип http -удалённый сервер

Используется когда MCP-сервер работает в облаке по URL.

{
  "type": "http",
  "url": "https://адрес-сервера/путь",
  "headers": {
    "Authorization": "Bearer ваш_токен"
  }
}
ПолеЧтоОбязательно
typeВсегда "http"Да
urlПолный URL сервераДа
headersHTTP-заголовки (для авторизации)Нет

Пример (Supadata -данные с сайтов):

{
  "supadata": {
    "type": "http",
    "url": "https://mcp.supadata.dev/v1"
  }
}

Как удалить MCP-сервер

Через Агента:

«Отключи MCP-сервер supadata»

Агент удалит запись из ~/.claude.json.

Вручную через VS Code:

Откройте ~/.claude.json, удалите блок нужного сервера из mcpServers, сохраните файл.

Как проверить, что подключено

Попросите Агента:

«Какие MCP-серверы у меня подключены?»

Агент прочитает ~/.claude.json и покажет список. Или откройте файл сами через VS Code.

Частые ошибки

MCP-сервер не появляется после добавления.

Агент читает ~/.claude.json при старте новой сессии. Если Вы добавили сервер во время активного разговора -сделайте /reset, и при следующем сообщении Агент подключится к новому серверу.

Ошибка «command not found» для stdio-типа.

Программа не установлена на сервере. Если MCP-сервер требует npx -проверьте что Node.js установлен (он ставится при установке Агента). Если это бинарный файл -проверьте что путь в command верный и файл имеет права на запуск (chmod +x).

Ошибка авторизации для http-типа.

Проверьте что URL и заголовки указаны правильно. Если сервер требует API-ключ -добавьте его в headers или в env.

Мой Вам совет

Начинайте с Способа 1 -просто попросите Агента. Он знает как редактировать ~/.claude.json и сделает это аккуратно. К ручному редактированию переходите, только если Агент почему-то не справился (такое бывает редко).

После подключения нового MCP -обязательно протестируйте: попросите Агента сделать что-нибудь простое через этот сервер. Если ответил -значит подключение работает.
