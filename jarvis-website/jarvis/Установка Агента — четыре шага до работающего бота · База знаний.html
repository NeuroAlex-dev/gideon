Установка Агента — четыре шага до работающего бота

10 мин · обновлено 2026-05-10
Это как переезд в новую квартиру — дом построен, мебель завезли, осталось получить ключи и заехать.

Это главная инструкция. Полный путь от пустых рук до момента, когда Ваш Агент поприветствует Вас лично и скажет «Здравствуйте, готов к работе». Шагов четыре, времени — тридцать-сорок минут с чашкой чая. Никакого чёрного окна терминала Вам открывать не придётся.

Сразу поставлю Вас в общую картину. Установка устроена так: Вы арендуете маленький сервер в интернете, заводите Telegram-бота через @BotFather, проверяете подписку Claude — а дальше Claude Code на Вашем ноутбуке сам заходит на сервер и разворачивает там всё необходимое. В конце Вы открываете сервер через VS Code и разворачиваете Конструктор Агентов одной командой.

Главная мысль: всю техническую работу делает Claude Code. От Вас — три «билета» (IP сервера + пароль, токен бота, подписка Claude) и нажатия Enter в нужных местах.

!

На этой странице нет ни одной команды для чёрного окна терминала. Всё, что выглядит как команда — это либо текст для чата с Claude Code (он сам поймёт, что с ним делать), либо команда, которую Claude Code продиктует серверу за Вас. Если в какой-то момент инструкция говорит «откройте терминал» — значит, вы смотрите не на ту страницу.

Что нужно подготовить ДО старта

  • VS Code и Claude Code на ноутбуке. Если Вы из выпускников курса — у Вас уже есть. Если пришли с эфира — поставьте сначала VS Code (бесплатно, code.visualstudio.com), потом расширение Claude Code (через маркет VS Code, ищите «Claude Code» от Anthropic). Один раз поставили — забыли. Это Ваш единственный «инструмент» в этой установке.
  • Подписка Claude — Pro или Max. Pro подходит, Max — комфортнее по лимитам, но не обязателен. Главное — что подписка есть и работает у Вас на ноутбуке, в обычном чате Claude.
  • Telegram под рукой. Через него заведём бота и в конце заберём промпт у @jarchitect_bot.
  • VPS — арендованный сервер. Минимальный тариф доступен у Beget, Timeweb или Digital Ocean. Принимают российские карты.
💬

VS Code — это бесплатный редактор от Microsoft, в который встраивается Claude Code расширением. Думайте про него как про окно, в котором живёт Claude. Сам по себе VS Code ничего не делает, он просто даёт Claude дом на Вашем ноутбуке.

Claude Code — это Ваш умный помощник внутри VS Code. Тот, кому Вы говорите задачи человеческим языком: «зайди на сервер, поставь то-то, перенеси такие-то файлы». Он сам разбирается как и что выполнить. Ему не нужно объяснять что такое SSH или какая команда чем отличается — он сам всё знает.

Маршрут установки

🏁 Старт
   │
   ├── 1️⃣ Арендуем VPS                   (~5 мин — заходите на сайт хостинга)
   ├── 2️⃣ Создаём бота в @BotFather      (~3 мин — два вопроса, один токен)
   ├── 3️⃣ Проверяем подписку Claude       (~1 мин)
   └── 4️⃣ Запускаем установку через Claude Code (~30 мин, Claude Code делает сам)
          ├── Часть А. Установка сервера (локальный Claude Code + IP + токен бота)
          └── Часть Б. Конструктор Агентов (@jarchitect_bot → персональный промпт)
   │
   ↓
🎉 Готово — пишите своему Агенту в Telegram!

Шаг 1. Арендуете сервер (~5 минут)

Идём на любой хостинг и заводим VPS. Я покажу на примере Beget — самый простой для новичка из России, поддержка на русском, оплата привычная, интерфейс понятный. Если Вам удобнее Timeweb или Digital Ocean — берите там, шаги похожие.

Заходите в раздел «Облачные серверы» и нажимаете «Создать сервер».

Скриншот — раздел «Облачные серверы» на Beget

В параметрах выбираете: 2 ядра CPU, 4 ГБ RAM, 40 ГБ диск, ОС Ubuntu 24.04. Этих параметров хватит и на Агента, и есть запас на рост памяти и проектов на годы вперёд.

Скриншот — конфигуратор тарифа: 2 ядра, 4 ГБ, 40 ГБ, Ubuntu 24.04

💬

VPS — это арендованный маленький компьютер в интернете, который включён 24/7. Стоит в дата-центре, никогда не выключается, имеет свой адрес. Вы арендуете его, как квартиру — заплатили, получили ключи, заехали. Нужен, чтобы Ваш Агент работал круглосуточно, даже когда Ваш ноутбук закрыт.

Через ~30 секунд после оплаты на почту от хостинга придёт письмо с тремя строчками: IP-адрес сервера (вида 123.45.67.89), логин (root) и пароль. Сохраните это в надёжном месте — это ключи от новой квартиры Агента.

Скриншот — письмо с IP, логином и паролем сервера

!

Тариф берите с запасом по диску — Агент со временем накопит память, дневники, knowledge-базу, и место уйдёт быстрее чем кажется. 40 ГБ хватит на несколько лет спокойной работы.

Один Агент — один сервер

Под одного Агента лучше арендовать отдельный VPS. Если со временем захочется завести ещё одного (например, рабочего и личного) — поселите его на другой сервер. Так каждый Агент работает стабильнее, и память не перепутается.

Шаг 2. Создаёте бота в @BotFather (~3 минуты)

В Telegram открываете @BotFather, нажимаете Start и пишете команду /newbot. Он спросит две вещи по очереди:

  1. Имя бота — то, что увидят пользователи. Например: «Агент Анны» или любое красивое имя.
  2. Username — техническое имя. Должно заканчиваться на bot или _bot, должно быть уникальным. Например: anna_agent_bot.

После этого @BotFather пришлёт токен — длинную строку вида 1234567890:AAEh-xxxx.... Скопируйте его и сохраните рядом с IP-адресом сервера.

Скриншот — диалог с @BotFather: создание нового бота и получение токена

💬

BotFather — это официальный бот Telegram, через который создаются все остальные боты. Он выдаёт каждому новому боту токен — длинную строку, как электронный ключ. По этому токену сервер потом сможет писать в Telegram от имени Вашего бота.

!

Токен бота — это пароль от Вашего бота. Не показывайте его в чатах, не выкладывайте в открытые места. Если случайно засветили — попросите у @BotFather команду /revoke, выберите бота, получите новый токен.

Бот пока не отвечает на сообщения — это нормально. Он оживёт после Шага 4, когда сервер с ним подружится.

Шаг 3. Проверяете подписку Claude (~1 минута)

Откройте claude.ai в браузере, войдите своим аккаунтом. Если в верхнем меню видите статус Pro или Max — всё, идёте дальше. Если статус Free — оформите Pro в настройках профиля.

💬

Подписка Claude — это Ваш аккаунт на claude.ai, которым Вы пользуетесь в обычном чате. Агент будет работать через эту же подписку — лимиты идут оттуда, отдельно за его работу ничего платить не нужно.

Убедитесь, что Вы помните логин/пароль от аккаунта Claude — они понадобятся в момент авторизации на сервере, в Шаге 4.

Шаг 4. Запускаете установку через Claude Code (~30 минут)

Самый большой шаг, но самый интересный. Здесь Claude Code на ноутбуке сам сходит на Ваш сервер и всё установит.

Делается в две части: сначала ставим сервер (Часть А, ~20 минут), потом устанавливаем Конструктор Агентов через персональный промпт (Часть Б, ~2 минуты).

Часть А. Установка сервера локальным Claude Code

Откройте VS Code на ноутбуке. Создайте любую папку на рабочем столе — например, agent-setup — и откройте её в VS Code (File → Open Folder). Запустите Claude Code.

Скопируйте в Claude Code этот установочный промпт, замените три значения на свои и нажмите Enter:

Скачай инструкцию с
https://raw.githubusercontent.com/Ntmib/jarvis-architect/main/server/INSTALL-SERVER.md
и выполни всё, что в ней написано.

IP: 123.45.67.89
Пароль: ваш-пароль-от-сервера
Telegram-бот: 1234567890:AAEh-xxxxxxxxxxxxxxxxxxxxxxxx

Замените:

  • 123.45.67.89 → Ваш реальный IP из письма хостинга
  • ваш-пароль-от-сервера → Ваш пароль root
  • 1234567890:AAEh-xxxx... → Ваш токен от @BotFather
!

Пароль и токен Вы пишете прямо в чат Claude Code на ноутбуке. Они никуда не уходят дальше — Claude Code хранит их только в этой переписке. Сразу после установки можете поменять пароль через панель хостинга, а токен — через /revoke у @BotFather. Оба они уже сделают своё дело.

Дальше Claude Code ведёт Вас за руку. Сам подключается к серверу, разворачивает программы, поднимает Telegram-бота как службу. Перед каждым шагом одной фразой объясняет, что сейчас будет делать, после — что получилось.

Что Claude Code сделает по шагам:

  1. Подключится к Вашему серверу по SSH
  2. Установит базовые программы — Node.js, Claude Code CLI
  3. Создаст на сервере отдельного пользователя agent
  4. Поставит базовые скиллы (готовые роли: писатель, разведчик, разработчик, дизайнер)
  5. Поднимет Telegram-бота как службу 24/7
  6. Попросит Вас один раз авторизовать Claude на сервере (через ссылку)
  7. Настроит VS Code Tunnel, чтобы Вы могли открывать сервер из обычного VS Code

Авторизация Claude на сервере

В какой-то момент Claude Code напишет: «откройте эту ссылку, войдите своим аккаунтом Claude и нажмите Authorize». Это нужно один раз — чтобы Агент на сервере мог работать через Вашу подписку. Откройте ссылку в браузере, войдите как обычно, нажмите большую кнопку Authorize.

Скриншот — кнопка «Authorize» для подтверждения доступа Claude Code к серверу

Если выдало ошибку в браузере — не пугайтесь

Это известная мелкая особенность Safari. Просто откройте ту же ссылку в Chrome или Firefox — заработает с первого раза. Ничего у Вас не сломалось.

Скриншот — типичная ошибка браузера; решается переключением на Chrome или Firefox

После авторизации вернитесь в Claude Code и скажите «готово» — он продолжит.

Когда установка завершится, Claude Code напишет короткий отчёт: что поставлено, что сделать дальше. Telegram-бот к этому моменту уже жив — попробуйте написать Вашему боту в Telegram /start, он должен ответить.

Часть Б. Устанавливаем Конструктор Агентов

Когда Telegram-бот ответил на /start — сервер готов. Теперь устанавливаем Конструктор.

Откройте @jarchitect_bot в Telegram и напишите /get. Бот пришлёт Ваш персональный промпт и пошаговую инструкцию что с ним делать.

!

Промпт персональный и работает один раз — это система защиты. Если что-то пошло не так или промпт сгорел — напишите боту /new, он выдаст свежий.

Готово — Конструктор Агентов установлен

Когда Конструктор установлен, Ваш Агент автоматически прочитает опорные файлы (CLAUDE.md, SOUL.md, MEMORY.md, GOALS.md) и поприветствует Вас. Напишите что-нибудь — это уже первый разговор.

Что у Вас сейчас есть:

  • Сервер в облаке, работает 24/7
  • Telegram-бот отвечает на Ваши сообщения через Вашу подписку Claude
  • Конструктор Агентов установлен и ждёт первого разговора
  • VS Code Tunnel — можно заходить на сервер с любого компьютера, где есть VS Code

Если что-то пошло не так

Промпт от @jarchitect_bot уже использован или сгорел → напишите боту /new, он выдаст свежий. Один промпт работает один раз — это специально, для безопасности.

Claude Code на ноутбуке говорит «подключение не прошло» → вероятно SSH-порт у провайдера блокируется. Это редкий случай — напишите в @aibasis_bot, разберёмся. Не пытайтесь чинить руками, не залезайте в веб-панель хостинга (там paste из буфера часто сломан и Вы только устанете).

Все остальные типовые ситуации разобраны в разделе Troubleshooting — заходите, ищите свой случай.

Мой Вам совет

Не торопитесь во время установки. Это случается один раз, и сделать это спокойно важнее, чем быстро. Прежде чем нажать Enter в Шаге 4 с паролем и токеном — перепроверьте, что заменили все три плейсхолдера. Это самая частая мелкая ошибка: «забыл подставить IP» или «забыл заменить пароль».

И пока локальный Claude Code ставит сервер — не сидите над ним, не обновляйте чат каждые 10 секунд. Это нормальный процесс, он молча работает. Заварите чай, отойдите. Через 15-20 минут вернётесь — сервер и бот будут готовы, останется только часть Б.
