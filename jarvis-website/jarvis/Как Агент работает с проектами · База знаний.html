Как Агент работает с проектами

5 мин · обновлено 2026-05-10
У Агента есть квартира (~/workspace/) и офисное здание (~/projects/). В квартире -личность. В офисе -рабочие проекты. Каждый этаж -отдельный проект.

Ваш Агент умеет вести несколько проектов одновременно: сайт, бот, контент-план, исследование -каждый в своей папке, со своими заметками, не мешая друг другу. Эта статья объясняет как устроена структура и как переключаться между проектами.

Главное сообщение

Два мира -личность и работа

~/workspace/ -это личность Агента: память, характер, цели, скиллы. Она одна и общая для всех проектов.

~/projects/ -это рабочие проекты: каждый в своей папке, со своими файлами и заметками. Проектов может быть сколько угодно.

Структура на сервере

🤖 Сервер Агента
│
├── 📂 ~/workspace/                Личность (общая)
│      ├── SOUL.md                 характер
│      ├── MEMORY.md               долгосрочная память
│      ├── GOALS.md                текущие цели
│      ├── USER.md                 профиль владельца
│      ├── memory/                 дневники
│      ├── knowledge/              справочники
│      └── .claude/skills/         скиллы (14 штук)
│
└── 📂 ~/projects/                 Рабочие проекты
       ├── my-website/             сайт-визитка
       │     ├── memory/           заметки проекта
       │     ├── knowledge/        решения проекта
       │     └── ...               код, файлы
       ├── telegram-bot/           Telegram-бот
       └── content-plan/           контент-план

Как создать новый проект

Просто скажите Агенту:

«Создай новый проект landing-page»

Агент создаст папку ~/projects/landing-page/ с подпапками memory/ и knowledge/ внутри. Дальше все файлы этого проекта будут жить здесь.

Или создайте проект через git:

«Склонируй репозиторий https://github.com/user/repo в projects»

Агент выполнит git clone в ~/projects/repo/.

Как переключаться между проектами

«Работаем над проектом my-website»

Агент переключит фокус на папку ~/projects/my-website/. Теперь все правки, поиск, чтение файлов -в контексте этого проекта.

Но личность Агента (SOUL.md, MEMORY.md, USER.md) остаётся из ~/workspace/. Агент всегда знает кто Вы, что Вам важно, в каком стиле общаться -независимо от проекта.

Изоляция проектов

Каждый проект изолирован от других:

  • Заметки memory/ в одном проекте не видны из другого
  • Решения knowledge/ хранятся отдельно
  • Git-репозитории независимы

Это значит, что Агент не перепутает контекст сайта-визитки с контекстом Telegram-бота. Каждый проект -отдельное пространство.

Два интерфейса -один диск

Если у Вас подключён VS Code Tunnel (/connect), оба интерфейса смотрят на одни и те же файлы:

  • Telegram -через Агента. Удобно для задач, вопросов, генерации.
  • VS Code -через браузер или приложение. Удобно для правки кода, просмотра структуры, загрузки файлов.

Правило: ~/workspace/ -территория Агента (через Telegram). ~/projects/ -территория VS Code (через туннель). Агент не трогает проекты без Вашей просьбы.

Если Вы правите файл в VS Code, а Агент одновременно тоже хочет его править -он увидит это через git status и спросит, прежде чем что-то менять.

Мой Вам совет

Не держите все файлы в ~/workspace/. Туда идёт только личность Агента и общие знания. Рабочие задачи -в ~/projects/, каждая в своей папке.

Когда начинаете новую тему -создайте проект. Когда заканчиваете -проект остаётся на диске как архив. Через месяц можно вернуться и Агент вспомнит контекст из memory/ и knowledge/ этого проекта.
