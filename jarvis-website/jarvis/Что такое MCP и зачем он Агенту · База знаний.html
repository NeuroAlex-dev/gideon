Что такое MCP и зачем он Агенту

4 мин · обновлено 2026-05-10
MCP - это универсальная розетка. Агент втыкает штепсель - и получает доступ к новому сервису, без программирования.

Ваш Агент умеет работать с файлами, искать в интернете, писать код. Но иногда нужно больше: открыть макет в дизайн-редакторе, вытянуть данные из сайта, запустить автоматизацию. Для этого существует MCP -протокол, который позволяет Агенту подключаться к внешним инструментам.

Главное сообщение

Универсальная розетка

MCP (Model Context Protocol) -это стандартный разъём между Агентом и внешними сервисами. Один протокол, одна точка подключения, десятки возможностей. Вам не нужно разбираться в API каждого сервиса -достаточно «воткнуть штепсель» в конфигурацию.

Как это работает

Без MCP Агент может искать в интернете и работать с файлами на сервере. Этого достаточно для многих задач, но не для всех.

С MCP Агент получает прямой доступ к внешним сервисам: календарь, почта, таблицы, дизайн-редактор. Он видит возможности сервиса и сам вызывает нужные действия, без Вашей помощи.

🤖 Агент (Claude Code CLI)
│
├── 📁 Файлы на сервере     (встроено)
├── 🌐 Интернет и curl       (встроено)
│
└── 🔌 MCP-серверы           (~/.claude.json)
       ├── pencil            дизайн-редактор
       ├── supadata          данные с сайтов
       └── ...               любой MCP-сервер

Что уже подключено из коробки

При установке Агента через @jarchitect_bot автоматически ставится один MCP-сервер:

Pencil -инструмент для работы с дизайн-макетами (файлы .pen). Агент может открывать макеты, читать их содержимое, вносить правки, делать скриншоты и экспортировать элементы. Всё это -прямо из Telegram, без открытия отдельной программы.

Pencil работает локально на Вашем сервере, не отправляет данные наружу.

Два типа подключений

MCP-серверы подключаются двумя способами:

ТипКак работаетПример
ЛокальныйПрограмма запускается прямо на Вашем сервереPencil
ОблачныйАгент обращается к сервису в интернетеSupadata, Google Calendar

Локальный - данные не уходят с Вашего сервера. Безопаснее.

Облачный - данные передаются по сети. Подходит для работы с внешними сервисами (почта, календарь, таблицы).

Где хранится конфигурация

Все подключённые MCP-серверы записаны в одном файле:

Путь к конфигурации~/.claude.json

Внутри -секция mcpServers. Claude Code CLI читает этот файл при каждом запуске и подключается ко всем перечисленным серверам. Добавили новый сервер в файл -при следующем сообщении Агент уже его видит.

Чем MCP отличается от API-ключей

В /settings Вы добавляете ключи к сервисам (Deepgram, GitHub, Vercel). Агент использует их вручную - сам формирует запросы к каждому сервису.

MCP работает проще: Агент подключается к сервису и видит список готовых действий. Дальше вызывает нужное - сервис сам разбирается с деталями.

💬 Аналогия: API-ключ -это как номер телефона. Вы сами звоните и говорите что нужно. MCP -это как секретарь: Вы говорите задачу, а он сам знает кому звонить и что говорить.
