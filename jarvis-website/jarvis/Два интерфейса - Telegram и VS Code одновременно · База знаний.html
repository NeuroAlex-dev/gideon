Два интерфейса - Telegram и VS Code одновременно

3 мин · обновлено 2026-05-10
Двое в одном офисе - один говорит задачи, второй проверяет результат.

Telegram и VS Code смотрят на один и тот же сервер. Вы можете работать через оба интерфейса одновременно - давать задачи в Telegram и сразу видеть результат в VS Code.

Как это работает

Представьте: Вы сидите в Telegram и говорите Агенту "сделай лендинг". Параллельно открыт VS Code. Как только Агент создаст файл - Вы тут же увидите его в дереве файлов VS Code.

Действие в TelegramЧто видите в VS Code
"Создай файл index.html"Файл появляется в дереве
"Напиши в MEMORY.md что я люблю кофе"MEMORY.md обновляется
"Сделай git commit"Появляется новый коммит

И наоборот: если Вы отредактируете файл в VS Code - Агент увидит изменения при следующем обращении к этому файлу.

Правила совместной работы

Один диск - два входа

Все файлы - общие. Telegram и VS Code работают с одними и теми же папками. Изменения одного видны другому.

Что важно помнить:

  1. Не правьте файл одновременно. Если Агент работает над файлом - не редактируйте его в VS Code в этот момент. Дождитесь, пока Агент закончит.
  1. Агент видит Ваши правки. Если Вы что-то изменили в VS Code, а потом попросили Агента "прочитай этот файл" - он увидит Вашу версию.
  1. Git-изменения общие. Если Вы сделали git commit в VS Code - Агент увидит это в git log. Если Агент закоммитил - Вы увидите в VS Code.

Можно ли работать только через Telegram?

Да. VS Code - дополнительный инструмент, не обязательный. Всё, что можно сделать в VS Code, можно попросить Агента сделать через Telegram:

  • "Покажи содержимое MEMORY.md"
  • "Отправь мне файл index.html"
  • "Какие файлы есть в папке projects?"

VS Code просто быстрее для некоторых задач - особенно если нужно пролистать много файлов или поправить текст вручную.
